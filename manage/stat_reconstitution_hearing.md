# ステータス振り直し機能 — 仕様ヒアリング用チェックリスト

実装前に **docs/092_stat_reconstitution_design.md** と **spec/092_stat_reconstitution.md** を参照し、以下を決定すること。決定内容は設計 doc または本ファイルに反映する。

---

## 1. 採用する方式

- [x] **案A** と **案B** のどちらを実装するか
  - **案A（部分再構築）**: 1アイテムで 1レベル分（18pt）のステータスを振り戻し。使用個数選択 → 振り戻し量指定 → 実行 → 再配分。
  - **案B（完全再構築）**: 5レベルダウンを代償に全ポイントを振り戻し（下限10%のみ残して再配分）。
  - **決定**: 両方実装。施術名称は 案A＝**部分再構築**、案B＝**完全再構築**。  
  - **仕様追加**: 第三の施術 **完全再構築β**（アイテム `reconstitution_ampoule_beta` 1個、レベルダウンなしで全ポイント振り戻し）を追加。

---

## 2. 案A を採用する場合

- [x] 「1レベル分」のポイント数は **18**（60×0.30、自由配分分）でよいか。それとも **60**（1レベル分のCAP全量）を振り戻せるようにするか。
  - **決定**: 18pt でよい。
- [x] 振り戻しと再配分は **1回の送信でまとめて実行** するか、**振り戻し実行** と **配分確定** の 2 段階でよいか。
  - **決定**: 1回の送信にまとめる。状態を持たない（再配分後の 7 ステを送信）。

---

## 3. 案B を採用する場合

- [x] **再構築アンプルαは消費するか**。消費する場合、何個か（例: 1個）。
  - **決定**: 消費する。1個（部分再構築と同様）。
- [x] 現在レベルが **5 以下** のときの扱い
  - 「実行不可」とするか
  - 「レベル1まで下げる」とするか
  - **決定**: 使用不可。表示は「身体への負担が大きく施術不可能」。
- [x] レベルダウンに伴い **経験値は newLevel の累計にリセット**（戻したレベル分の経験値は消失）でよいか。
  - **決定**: 消失でよい。実行時に注意を表示する。

---

## 4. 共通

- [x] 再構築実行後、**未割り振りポイントをそのまま保存**（合計 &lt; CAP の状態でOK）とするか、必ず「配分確定」まで完了させるか。  
  （spec/048 では合計 ≤ CAP であれば未割り振りを残して保存可能）
  - **決定**: 未割り振りのまま保存でよい。

---

## 決定メモ（記入例）

- **施術名称**: 案A＝部分再構築、案B＝完全再構築。両方実装。
- **案A**: 1レベル分＝18pt。送信＝1回にまとめる（状態を持たない）。再配分後の 7 ステを送信。
- **案B**: アイテム 1 個消費。Lv≤5＝使用不可（「身体への負担が大きく施術不可能」と表示）。経験値＝リセットでよい、実行時に注意を表示。
- **共通**: 未割り振り保存＝可。
- **完全再構築β**: アンプルβ 1個消費。レベル・経験値は変更せず、7ステを下限のみにリセット。レベル制限なし。
