# 管理画面：同一装備種別を出力とするレシピが複数ある場合の注意表示

**参照**: `spec/093_equipment_dismantle.md` §6・§7.1

工房の**鍛錬**と**装備解体**では、装備種別ごとに「本来のレシピ」を **CraftRecipe の code 昇順で先頭 1 件** として扱う。同一装備種別を出力とするレシピが複数あると、運用者が意図しないレシピが使われる可能性があるため、管理画面で注意を表示する。

---

## 1. 表示する画面と内容

### 1.1 装備種別（EquipmentType）の編集・新規画面

- **対象**: `src/app/dashboard/admin/equipment-types/` の編集フォーム・新規フォーム（存在する場合）。
- **表示内容**: 常時または「この装備を出力とするレシピ」を表示しているとき、
  - 「この装備種別を出力とするレシピが複数ある場合、工房の鍛錬・解体では **code が辞書順で先のレシピ** が使われます。運用時は同一装備を出力するレシピを 1 件に揃えるか、code の順序を意識してください。」
- 実装: 当該装備種別を `outputEquipmentTypeId` に持つ CraftRecipe の件数を取得し、2 件以上なら上記の注意文を表示する。

### 1.2 クラフトレシピの編集・新規画面（出力＝装備のとき）

- **対象**: `src/app/dashboard/admin/craft-recipes/[id]`（編集）、`/craft-recipes/new`（新規）。出力種別が equipment のときのみ。
- **表示内容**: 選択中の出力装備種別について、**同じ装備種別を出力とするレシピが他に 1 件以上ある場合**、
  - 「同一装備を出力するレシピが他に **○ 件** あります。鍛錬・解体では **code 昇順の先頭** が採用されます。」
- 実装: 編集中のレシピを除き、同じ `outputEquipmentTypeId` を持つ CraftRecipe の件数を取得。1 件以上なら警告または注意を表示する。新規作成時は、選択中の装備種別に対して既存のレシピ件数を表示する。

---

## 2. 実装のタイミング

- 装備解体機能の実装と同時でなくてもよい。管理画面の注意表示は後追いで追加可能。
- 実装時は `getAdminCraftRecipeOptions` や装備種別取得 API に「同一装備を出力するレシピ数」を返すフィールドを追加するか、クライアントで一覧から集計するかはプロジェクトの都合でよい。
