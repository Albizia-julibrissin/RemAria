# spec/094 チャット拡張 結合・確認チェックリスト（Phase 4）

実装完了後の手動確認用。実施したらチェックを入れる。

---

## 前提

- 開発サーバー起動: `npm run dev`
- ログイン可能なテストアカウント（例: シードの test2 / 管理人）でログイン済みであること
- 開拓任務が 1 件以上あり、**notifyChatOnClear = true** にした任務が 1 件以上あること（管理画面で開拓任務編集 → 「クリア報告時に全体チャットに…を投稿する」にチェック → 保存）

---

## 1. 任務クリア報告 → チャットに quest_clear が 1 件出る

- [ ] 開拓任務画面で、**notifyChatOnClear が ON の任務**を 1 つ選ぶ（未達成のもの、または達成済みで未報告のもの）
- [ ] 達成条件を満たしている状態で「クリア報告」を実行する
- [ ] チャットパネルを開く（右下「💬 チャット」）
- [ ] **「〇〇が任務「△△」を達成しました。」** のシステムメッセージが 1 件表示されること
- [ ] その行に **旗アイコン** と **主人公アイコン**（自分のアイコン）が表示されること
- [ ] **プレイヤー名がリンク**になっており、クリックで `/dashboard/profile/[自分のuserId]`（開拓者証）に遷移すること
- [ ] **通常のユーザー投稿**でも、送信者名が開拓者証へのリンクになっていること（他ユーザーまたは自分の投稿で確認）
- [ ] notifyChatOnClear が **OFF** の任務でクリア報告した場合は、**チャットに同じメッセージは出ない**こと（既存の「報告」動作のみ）

---

## 2. チャット表示設定の反映

### 2.1 パネルサイズ

- [ ] ヘッダー「設定」→「チャット設定」で設定ページを開く
- [ ] パネルサイズを **小(S)** に変更して保存する
- [ ] チャットパネルを開く → 幅・高さが **小さく**（280px × 30vh 相当）なっていること
- [ ] パネルサイズを **大(L)** に変更して保存する
- [ ] チャットを開き直す（または設定保存後その場で開いている場合は反映されていること）→ 幅・高さが **大きく**（440px × 55vh 相当）なっていること
- [ ] パネルサイズを **カスタム** にし、幅・高さのスライダーで任意の値（例: 幅 50、高さ 50）に変更して保存する
- [ ] チャットパネルが、指定した割合（vw/vh）に近いサイズになっていること

### 2.2 システムメッセージの表示 ON/OFF

- [ ] 設定で **「システムメッセージを表示する」を OFF** にして保存する
- [ ] チャットパネルを開く（または開き直す）
- [ ] 先ほど出した **任務達成のシステムメッセージ（quest_clear）が一覧に表示されない**こと（ユーザー投稿のみ表示される）
- [ ] 設定で **「システムメッセージを表示する」を ON** に戻して保存する
- [ ] チャットを開き直す → 任務達成メッセージが再び表示されること

### 2.3 その他設定（任意）

- [ ] **起動時にチャットを開く** を ON にし、ページをリロード → 初期表示でチャットパネルが開いていること
- [ ] **メッセージの文字サイズ** を「小」または「大」に変更し、保存 → チャット内のメッセージ文字サイズが変わること

- 不具合があれば issue または doc に記録し、修正後に再確認する。
