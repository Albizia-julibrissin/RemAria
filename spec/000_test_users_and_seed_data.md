# Spec: テストユーザーと初期データ

この Spec は、`npm run db:seed` で投入される**テストユーザーおよび初期キャラ構成**の方針を固定する。

---

## 1. テストユーザー一覧

- `test1@example.com`（accountId=`test_user_1`）
- `test2@example.com`（accountId=`test_user_2`）

本 Spec では **test1** のみ詳細を定義する。test2 は将来の検証用に空のアカウントとして扱う。

---

## 2. test1 の初期キャラ構成

### 2.1 キャラ一覧

- 主人公（category=`protagonist`） … 1 体
- 仲間（category=`companion`） … 1 体（「初期仲間」）
- メカ（category=`mech`） … 1 体

### 2.2 レベル・CAP

- 主人公 / 初期仲間は **レベル 50** とする。
- CAP は `docs/09_character_implementation_notes.md` のルールに従う。
  - Lv1 の CAP 初期値 = 560
  - LvN の CAP = 560 + 60 × (N - 1)
  - よって **Lv50 の CAP = 560 + 60 × 49 = 3,500**
- メカはレベルによる CAP 増加・割り振りは行わないため、**レベルは 1（デフォルト）・CAP は 60（初期値）**のままとする（実際の戦闘性能はパーツから算出）。

### 2.3 ステータス初期値（テスト用 Lv50）

- 主人公 / 初期仲間については、レベル 50 時点の**ステータス**を基準とする（spec/048: 各ステ 10〜30%、合計＝CAP）。
- CAP=3,500 のとき、下限 10% = 350/ステ。自由分 30% を均等に振った値とする。

CAP=3,500 のとき：

- 各ステの下限 = `floor( CAP * 0.10 ) = 350`
- 自由分 30% = 1,050 を 7 ステに均等 → 各 +150。350 + 150 = 500。

したがって、主人公 / 初期仲間 の初期値は次のとおりとする。

- `STR = 500`, `INT = 500`, `VIT = 500`, `WIS = 500`, `DEX = 500`, `AGI = 500`, `LUK = 500`
- `CAP = 3,500`

メカについては、基礎ステータスは最低値＋パーツ補正で表現する方針とし、Seed 時点では次の値とする（基礎7ステすべて10なので CAP=70）。

- `STR = 10`
- `INT = 10`
- `VIT = 10`
- `WIS = 10`
- `DEX = 10`
- `AGI = 10`
- `LUK = 10`
- `CAP = 70`

> 実装上は「未使用ポイント」を専用カラムで持たず、常に `CAP - (STR+INT+VIT+WIS+DEX+AGI+LUK)` から導出する（主人公・仲間のみ）。

## 3. 将来の拡張

- 将来的に「物理型」「魔法型」などのビルドテンプレートをテストユーザーに反映したい場合でも、
  - **レベル（とそこから決まる CAP）** と
  - **各ステの 5〜30% 制約**

  の 2 点は本 Spec のルールとして維持する。
- テンプレートを適用する場合は、本ファイルに「test1 主人公は物理型A 配分」などの小節を追加し、そこから seed 実装を更新する。
