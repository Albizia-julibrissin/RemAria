# Spec Template

このテンプレートは、すべての機能仕様書の基準とする。
1機能につき1ファイル作成すること。

------------------------------------------------------------------------

## 0. 依存・横断

### 0.1 依存する spec

-   この spec が前提とする他の spec を列挙する（例：`010_auth` のセッションが有効であること）。
-   該当なしの場合は「なし」と記載。

### 0.2 提供する API / 利用者（該当する場合）

-   この spec が**横断的に提供**する API や関数がある場合に記載する。
-   表形式で「API 名」「用途」「呼び出し元」を明記する。
-   横断 spec（認証等）の場合は、他 spec が参照すべき**横断前提**をここに書く。
-   該当しない場合は「該当なし」または本セクションを省略。

## 11. 画面仕様（該当する場合）

ユーザー向け画面がある場合に記載する。該当しない場合は「該当なし」または省略。

各画面ごとに以下を定義する：

| 項目 | 内容 |
|------|------|
| **URL** | 画面のパス（例：`/login`） |
| **前提** | ログイン要否、依存する spec の状態 |
| **要素** | フォーム項目、ボタン、リンク等 |
| **呼び出す API** | 使用する Server Action / API |
| **成功時** | 遷移先・表示内容 |
| **エラー時** | 表示内容・遷移先 |

保護画面の場合は、`010_auth` のセッション確認に依存する旨を「前提」に明記する。
