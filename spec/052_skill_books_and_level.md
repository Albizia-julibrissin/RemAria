# Spec: スキル分析書・スキルレベル

`docs/026_user_inventory_and_items.md` §2.6・§4.1 に基づき、**スキル分析書**（Item category=skill_book）の所持・消費と、**キャラのスキルレベル**（CharacterSkill.level）およびレベルアップに必要な冊数ルールを定義する。

------------------------------------------------------------------------

## 0. 依存・横断

### 0.1 依存する spec

- **010_auth**：所持・消費 API はセッション前提。
- **045_inventory_and_items**：Item.category=skill_book、UserInventory のスタック型所持。
- **038_battle_skills_and_effects**：Skill マスタ（戦闘スキルは category=battle_active）。
- **025_character_list**：Character（主人公・仲間）の所有権。

### 0.2 提供する API / 利用者

| API | 用途 | 呼び出し元 |
|-----|------|-----------|
| consumeSkillBook | スキル分析書を 1 冊消費し、指定キャラに習得 or レベルアップする。 | 物資庫（習得ダイアログ）、キャラ詳細（レベルアップモーダル） |
| getCharactersForSkillBook(skillId) | 指定スキルをまだ習得していないキャラ一覧（主人公＋仲間）を返す。 | 物資庫（習得ダイアログ表示時） |
| getSkillBookLevelUpInfo(characterId, skillId) | 指定キャラ・スキルの現在Lv・次Lv必要冊数・所持数・分析書itemId を返す。 | キャラ詳細（レベルアップモーダル表示時） |

------------------------------------------------------------------------

## 1. 目的

- **スキル分析書**：探索ドロップで入手。同一スキル用はスタック型で所持。消費時に「習得」または「レベルアップ」を行う。
- **スキルレベル**：キャラが習得したスキルにレベル（0, 1, 2, …）を持たせる。初回消費で習得（Lv0）、以降は必要冊数を消費してレベルアップ。
- **レベルアップ必要冊数**：Lv N → Lv N+1 に **(N+1) 冊**を消費する。  
  - Lv0→1: 1 冊、Lv1→2: 2 冊、Lv2→3: 3 冊、…

------------------------------------------------------------------------

## 2. 用語

- **スキル分析書**：Item の一種。category=skill_book。item.skillId で紐づく 1 スキルに対してのみ使用可能。
- **習得**：そのキャラがまだそのスキルを持っていない場合、CharacterSkill を level=0 で作成する。
- **レベルアップ**：既に習得済みの場合、現在レベルを L とすると、(L+1) 冊を消費して level を L+1 にする。

------------------------------------------------------------------------

## 3. 永続化データ

### 3.1 Item（拡張）

spec/045 の Item に、スキル分析書用の参照を追加する。

| カラム | 型 | 説明 |
|--------|-----|------|
| （既存） | … | id, code, name, category, … |
| **skillId** | **String?, FK→Skill** | category=skill_book のとき必須。この分析書で習得/レベルアップできるスキル。 |

- category=skill_book の Item は、必ず skillId を設定する。seed で「スキル 1 種につき 1 アイテム」を登録する想定。

### 3.2 CharacterSkill（拡張）

spec/038・既存 CharacterSkill に **level** を追加する。

| カラム | 型 | 説明 |
|--------|-----|------|
| （既存） | … | id, characterId, skillId, createdAt |
| **level** | **Int, default 0** | スキルレベル。0=習得直後。レベルアップで 1, 2, 3, … に増加。 |

- 既存データは level=0 として扱う（マイグレーションで default 0 を付与）。

------------------------------------------------------------------------

## 4. ルール

### 4.1 レベルアップ必要冊数

- 現在レベル **L** から **L+1** になるために消費する分析書の数は **L+1** 冊。
- 例：Lv0→1: 1 冊、Lv1→2: 2 冊、Lv2→3: 3 冊。

### 4.2 消費時の挙動

1. **未習得**：そのキャラに CharacterSkill(characterId, skillId, level=0) を 1 件作成し、分析書を **1 冊**消費する。
2. **習得済み**：現在の level を L とする。所持冊数が **L+1 以上**なら、(L+1) 冊を消費して level を L+1 に更新。所持が足りない場合はエラー。

### 4.3 スキルレベルの戦闘への反映

戦闘計算では、発動キャラのそのスキルに対する **CharacterSkill.level**（以下 L）を用いて、**スキル倍率**と**消費 MP** に補正を掛ける。補正は **sqrt(L / L_max)** の線形で効かせ、設計上の最大レベル **L_max = 99** で上限を満たす。

スキル種別は spec/038 の **Skill.battleSkillType**（physical / magic / support）で判定する。

#### 4.3.1 物理・魔法（battleSkillType = physical / magic）

| 項目 | L=0 | L=99 | 式 |
|------|-----|------|-----|
| **スキル倍率** | 1.00 倍 | 1.10 倍 | `1 + 0.10 * sqrt(L / 99)` |
| **消費 MP** | 1.0 倍（そのまま） | 0.9 倍 | `1 - 0.1 * sqrt(L / 99)` |

- ダメージ計算時のスキル倍率（effectType の power や倍率）に、上記スキル倍率を乗算する。
- 消費 MP は、マスタまたは効果で定義された基礎 MP に上記消費 MP 倍率を乗算した値を切り捨てで適用する（最小 1 などは既存ルールに従う）。

#### 4.3.2 補助（battleSkillType = support）

| 項目 | L=0 | L=99 | 式 |
|------|-----|------|-----|
| **スキル倍率** | 適用しない | — | 補助スキルはダメージ倍率補正なし |
| **消費 MP** | 1.0 倍 | 0.8 倍 | `1 - 0.2 * sqrt(L / 99)` |

- 補助スキルはダメージ計算の倍率補正は行わない。消費 MP のみ上記で軽減する。

#### 4.3.3 共通

- **設計上の最大レベルは 99**。L > 99 の場合は 99 として計算する（cap）。
- **未習得**（CharacterSkill が存在しない）の場合は L=0 として扱い、補正は 1.0 倍・消費 MP はそのまま。
- **レベルアップ**：現在 L=99 のスキルには分析書でレベルアップできない。consumeSkillBook は L≥99 のときエラーを返す。キャラ詳細のスキル一覧では L=99 のスキルに「レベルアップ」ボタンを出さず「最大レベル」と表示する。
- 実装場所：戦闘ループでスキル発動時に、発動者キャラの CharacterSkill.level を参照し、上記式で倍率・消費 MP を算出して適用する。参照は spec/038・040 に従う戦闘実行（run-battle-with-party 等）内で行う。

## 5. API

### 5.1 consumeSkillBook

- **入力**：itemId（UserInventory に存在するアイテムID）、characterId（習得/レベルアップ対象のキャラ）。
- **前提**：セッションから userId を取得。item は category=skill_book かつ skillId が設定されていること。character はその userId に属し、category は protagonist または companion であること。
- **処理**：
  1. Item を取得。category !== 'skill_book' または skillId が null ならエラー。
  2. Character を取得。userId が一致しない、または category が protagonist/companion でないならエラー。
  3. UserInventory で (userId, itemId) の quantity を取得。0 以下ならエラー。
  4. CharacterSkill で (characterId, skillId) を検索。
  5. 存在しない場合：CharacterSkill を level=0 で作成。UserInventory を 1 減算。
  6. 存在する場合：現在 level を L とする。必要冊数 = L+1。quantity >= L+1 なら level を L+1 に更新し、UserInventory を (L+1) 減算。そうでなければエラー。
- **出力**：成功時 { success: true, newLevel: number }。失敗時 { success: false, error: string }。
- **トランザクション**：在庫減算と CharacterSkill の作成/更新を同一トランザクションで行う。

------------------------------------------------------------------------

## 6. エラー条件

- 未ログイン：401 相当。
- itemId がスキル分析書でない、または skillId が無い：不正なアイテム。
- characterId が自分のキャラでない、またはメカである：対象不可。
- 所持数不足（未習得時は 1 冊未満、習得済み時は必要冊数未満）：在庫不足。

------------------------------------------------------------------------

## 7. 画面・UX

- **物資庫（スキル分析書タブ）**：スタック型で一覧表示。各アイテムに「習得」ボタンでダイアログを開き、**そのスキルをまだ習得していないキャラのみ**を一覧から選んで consumeSkillBook を呼ぶ。成功時は refresh で一覧を更新。getCharactersForSkillBook(skillId) で未習得キャラ一覧を取得する。
- **キャラ詳細**：習得スキル一覧に **レベル**を表示する（例：「メテオ Lv2」）。各スキルに「レベルアップ」ボタンを配置。押下でモーダルを開き、タイトル「{スキル名}の分析」・現在Lv・次のLvまでの必要冊数・所持数・個数入力・「分析」ボタンを表示。分析で consumeSkillBook を呼びレベルアップ。getSkillBookLevelUpInfo(characterId, skillId) でモーダル用情報を取得。MVP ではレベル効果は戦闘に反映しないが、表示は行う。

------------------------------------------------------------------------

## 8. 探索ドロップとの連携

- 探索エリアの **技能ドロップテーブル**（ExplorationArea.skillDropTableId）に、スキル分析書アイテムを DropTableEntry で登録する。
- 探索報酬付与（既存の exploration 完了処理）で UserInventory に加算されるため、本 spec の「消費」のみ追加すればよい。
