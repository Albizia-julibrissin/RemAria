# Spec: 仮戦闘（テスト用）

戦闘計算式（`docs/10_battle_calc_formulas.md`・`10_battle_status.csv`）が意図どおり動くか検証するための**テスト用の仮戦闘**を定義する。

**本 spec の位置づけ**：ここで少しずつ戦闘ロジックを実装し、動作確認しながら**実際に本番で使う戦闘システムへフィードバックする**想定である。仮戦闘で検証した計算式・行動順・ターゲット抽選・表示仕様は、探索戦闘やイベント戦闘など本番戦闘にそのまま流用する。

本番の探索・報酬・敵マスタ・結果の永続化は本 spec のスコープ外とする。

------------------------------------------------------------------------

## 0. 依存・横断

### 0.1 依存する spec

- **010_auth**：セッションが有効であること（ログイン済み）。仮戦闘の実行・画面は保護画面として getSession を前提とする。
- **015_protagonist_creation**：主人公が作成済みであること。主人公のステータスを戦闘入力として用いる。

### 0.2 提供する API / 利用者

| API | 用途 | 呼び出し元 |
|-----|------|-----------|
| runTestBattle | 主人公 vs 固定敵（スライム3体）で 1v3 戦闘を実行し、ログ・勝敗・配置を返す | テスト用画面 `/battle/test` |

------------------------------------------------------------------------

## 1. 目的

- 戦闘計算（派生ステータス・行動順・**ターゲット抽選（列ウェイト）**・命中・直撃/致命・ダメージ・疲労・MP 回復）が仕様どおり動くことを、**仮の 1v3 戦闘**で確認できるようにする。
- **3x3 マス表現**（味方左・敵右、行/列・前列/中列/後列）と、1 ターンごとの状態表示（HP/MP バー・配置グリッド・ログ）で挙動を確認する。
- テスト用であるため、スキル・装備・結果の永続化・敵マスタは扱わない。実装したロジックは本番戦闘へフィードバックする。

------------------------------------------------------------------------

## 2. 用語

- **仮戦闘**：本 spec で定義する「主人公 1 体 vs 固定敵 3 体（スライム）」の戦闘。通常攻撃のみ。結果は保存しない。
- **サイクル**：生存者が全員 1 回ずつ行動したまとまり。1v3 では 1 サイクルあたり最大 4 ターン（主人公 + 敵 3 体の行動順に従う）。
- **派生ステータス（戦闘用ステータス）**：基礎ステータス（STR, INT, VIT, WIS, DEX, AGI, LUK）を **`10_battle_status.csv` の係数で二次解釈**した値。HP, MP, 物理攻撃, 魔法攻撃, 物理防御, 魔法防御, 命中力, 速度（回避）, 運。**AGI は速度そのものではなく**、係数計算を経て派生値「速度（回避）」になる。
- **固定敵**：コードまたは定数で定義された敵。本 spec ではスライム 1 種の基礎ステータスを 3 体分に共通適用し、**配置のみ**（中列・後列）を定数で持つ。
- **列ウェイト**：ターゲット抽選で使用。前列 1.5／中列 1.0／後列 0.5。`P(target i) = weight(col_i) / Σ(weight)`（生存者のみ対象）。
- **3x3 マス**：行 1～3（上から下）、列 1＝前列・2＝中列・3＝後列（左から右）。味方グリッドは左、敵グリッドは右に配置して向き合せる。表示は □＝空き、■＝ユニット存在。

## 3. 入力（Input）

### 3.1 runTestBattle の入力

- **明示的な引数なし**。Server Action 内で以下を取得・決定する。
  - **主人公**：現在ログインユーザーの Character（category=protagonist）1 体（getSession → getProtagonist）。いなければエラー。
  - **主人公の配置**：定数（例：1 行目・後列）。1v3 では変更しない。
  - **敵**：スライム 3 体。同一の基礎ステータス（定数）。配置は定数で **中列・後列** に 3 体を割り当てる（例：1 行目中列、2 行目中列、3 行目後列）。

### 3.2 主人公の入力元

- Character のカラム：`STR`, `INT`, `VIT`, `WIS`, `DEX`, `AGI`, `LUK`, `CAP`（10_battle_status.csv 準拠）。装備・バフは適用しない。

### 3.3 敵の定義（固定 3 体）

- 名前：スライム。3 体を「スライム1」「スライム2」「スライム3」のように識別する。
- 基礎ステータス：3 体とも同一の定数オブジェクト（STR, INT, VIT, WIS, DEX, AGI, LUK, CAP）。派生は主人公と**同じ計算式**（5.1）。
- 配置（定数）：例として 1 体目＝1 行目・中列、2 体目＝2 行目・中列、3 体目＝3 行目・後列。前列は使わない。

## 5. ルール

### 5.1 派生ステータス（戦闘用ステータス）

- **前提**：永続するのは基礎ステータスのみ。戦闘時はこれを係数表で**二次解釈**し、戦闘用ステータス（物理攻撃力・速度（回避）など）として使用する。工業配備時は基礎ステータスがそのまま影響する（二次解釈しない）。
- **算出式**：`DerivedStat = Σ( BaseStat[s] * Coef[DerivedStat][s] )`。係数は **`docs/10_battle_status.csv`** の「ベース表」行（2～10 行目）に従う。
- **CSV の列**：STR, INT, VIT, WIS, DEX, AGI, LUK。DB と一致（docs/16 A 案）。キャラ・敵とも同じ 7 種＋CAP で派生値を算出。**AGI は係数計算の入力であり、結果として「速度（回避）」が得られる**（AGI ＝ 速度ではない）。
- **運ポイント**：`LuckPoint = 12 * LUK`（基礎 LUK）。直撃・致命率の計算に使用。
- 本 spec では **物理攻撃のみ**使用する（通常攻撃は物理）。

### 5.2 戦闘フロー（1v3・通常攻撃のみ）

1. **初期化**：主人公・敵 3 体それぞれで派生ステータスを算出。現在 HP = MaxHP、現在 MP = MaxMP。疲労はサイクルごとに 5.4 に従い更新。
2. **サイクルループ**（最大サイクル数 30）：
   - **行動順**：生存者（主人公 + 生存している敵）について、重み `W_turn = sqrt(EVA) * (1 + α * (LUK/7))` で**重み付き抽選**し、1 人ずつ順番を確定（確定した者は除外して次を抽選）。
   - **各ターン**：そのターンの攻撃者が「主人公」か「敵 i」かを判定。
     - **主人公のターン**：ターゲットを**生存している敵のうち列ウェイトで 1 体抽選**（5.2.1）。その敵に通常攻撃。
     - **敵のターン**：ターゲットは**主人公固定**。その敵が主人公に通常攻撃。
   - **攻撃処理**（共通）：ターゲット決定後、命中判定 → 命中時は直撃／致命判定 → DEF_effective（致命時は DEF×0.5）→ 乱数（通常攻撃は攻撃側 1.0、防御側 0.8～1.0）→ Mitigation → ダメージ計算 → 直撃/致命倍率（×1.2）→ 対象 HP 減算、攻撃側 MP 回復（MaxMP の 2％＋6～12）。ログにそのターン終了時点の主人公・敵 3 体の HP/MP を記録。
   - 主人公 HP≤0 で敗北、敵 3 体すべて HP≤0 で勝利。どちらかでループ脱出。最大サイクルまでに決着しなければ **draw**。
3. **ログ**：各ターンの結果（上記 4.1 の log 項目）を配列に積み、返す。

#### 5.2.1 ターゲット抽選（列ウェイト）

- 主人公が攻撃する際、**生存している敵**の中から 1 体を選ぶ。
- **列ウェイト**：前列 1.5、中列 1.0、後列 0.5（`docs/10_battle_calc_formulas.md` 2. ターゲット選択に準拠）。
- **確率**：`P(target i) = weight(col_i) / Σ( weight(col_j) )`（j は生存している敵のみ）。敵 3 体が中列・中列・後列なら、重み 1.0, 1.0, 0.5 → 確率はおおよそ 40％, 40％, 20％。

### 5.3 直撃率・致命率（10 の式準拠）

- **K**：`K = CAP * 0.3 * 12`（攻撃側の CAP）。
- **Δ**：`LuckPoint_attacker - LuckPoint_defender`。LuckPoint = 12 * LUK（基礎）。
- **P(direct)**：`clamp( 0.25 + D * ( Δ / ( |Δ| + K ) ), 0, 0.95 )`。**D = 0.5**。
- **P(fatal | direct)**：直撃時のみ判定。`clamp( 0.25 + ( P(direct) - 0.25 ) / 3, 0, 0.50 )`。

### 5.4 定数一覧（docs/10 準拠）

| 項目 | 値 |
|------|-----|
| 行動順 α | 0.2 |
| 命中 β | 0.5 |
| 直撃/致命 D | 0.5 |
| K | CAP * 0.3 * 12 |
| 列ウェイト | 前列 1.5、中列 1.0、後列 0.5 |
| 防御減衰 | 1000 / (1000 + DEF_effective) |
| 疲労 | 1～5 サイクル 1.0、6 以降 1 + 0.04*(cycle-5) |
| 通常攻撃乱数 | 攻撃側 1.0、防御側 0.8～1.0 |
| MP 回復 | floor(MaxMP*0.02) + Random(6,12) |
| 最大サイクル | 30 |

## 6. 処理フロー（概要）

1. セッション取得。未ログインならエラー。
2. 主人公取得（getProtagonist）。存在しなければ NO_PROTAGONIST でエラー。
3. 固定敵 3 体の基礎ステータスと配置（中列・後列）を定数から取得。
4. 主人公・敵 3 体それぞれで派生ステータスを算出。現在 HP/MP をセット。
5. サイクル = 1 から最大までループ：
   - 生存者で行動順を重み付き抽選（4 者→3 者→… と 1 人ずつ確定）。
   - 各ターン：攻撃者が主人公なら列ウェイトでターゲット敵を抽選、敵ならターゲットは主人公。命中→直撃/致命→ダメージ→HP/MP 更新。ログにそのターン終了時の全員の HP/MP を追加。
   - 主人公 HP≤0 または敵 3 体全滅でループ脱出、勝者決定。
6. 結果（result）、log、summary、protagonistPosition、enemyPositions を返す。

------------------------------------------------------------------------

## 7. 永続化データ / 一時データ

### 7.1 永続化するデータ

- **なし**。仮戦闘の結果は保存しない。

### 7.2 保存しないデータ

- 戦闘ログ、各ターンの中間値（命中率・直撃率・乱数・Mitigation 等）、勝敗・配置。すべて実行時のみ保持し、レスポンスで返す。

------------------------------------------------------------------------

## 8. 例（最低 3 ケース）

### ケース 1：主人公が勝つ

- 入力：主人公（例：物理型A）vs スライム 3 体（中列・後列に配置）。
- 期待：`result: "player"`、log で主人公の攻撃が敵を撃破していく。`summary.winner === "player"`、`enemyHpFinals` が [0,0,0] または全滅に至る。

### ケース 2：敵が勝つ

- 入力：主人公（弱め）vs スライム 3 体。
- 期待：`result: "enemy"`、`summary.winner === "enemy"`。主人公の HP が 0 になる。

### ケース 3：主人公未作成

- 入力：ログイン済みだが主人公がまだ作成されていない状態で runTestBattle を呼ぶ。
- 期待：`success: false`, `error: "NO_PROTAGONIST"`。

## 10. テスト観点

- 派生ステータスが CSV 係数と一致するか（既知の基礎値で手計算と照合）。
- 行動順が SPD/LUK の重みどおりになるか（4 者での抽選）。
- **ターゲット抽選**が列ウェイトどおりか（中列 2 体・後列 1 体なら、中列が選ばれやすく後列が選ばれにくい）。
- 命中・直撃・致命の確率が式の範囲内か。ダメージが Mitigation・疲労・倍率と一致するか。
- 最大サイクルで draw になるケースがあるか。

## 12. スコープ（本 spec で扱わないもの）

- スキル・消費 MP・スキル倍率。
- 戦闘結果の DB 保存・報酬・履歴。
- 敵の種類複数・敵マスタ・探索との連携。
- 味方パーティ 3 体・仲間・メカ（現状は主人公 1 体のみ）。

**本 spec で実装済み**：3 体以上の敵・列・ターゲット抽選（列ウェイト）、3x3 マス表現、1 ターンごとの状態表示。これらは本番戦闘へフィードバックする。
