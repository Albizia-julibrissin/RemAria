# Spec: キャラ一覧・詳細

ログインユーザーが**所有キャラ（主人公・仲間・メカ）を一覧**し、**クリックで詳細（基礎ステータス表示）**に遷移できるようにする。  
ダッシュボードから「宿舎」カードでキャラ一覧へ入れるようにする。

------------------------------------------------------------------------

## 0. 依存・横断

### 0.1 依存する spec

- **010_auth**：セッションが有効であること（ログイン済み）。一覧・詳細は保護画面とする。
- **015_protagonist_creation**：主人公の有無判定（getProtagonist）。主人公未作成時は作成画面へ強制遷移する。

### 0.2 提供する API / 利用者

| API | 用途 | 呼び出し元 |
|-----|------|-----------|
| getCharactersByUserId（または同等） | ユーザー所有キャラ一覧取得 | キャラ一覧画面 |
| getCharacterById（または同等） | 1体のキャラ詳細取得（基礎ステータス含む） | キャラ詳細画面 |

**横断前提**：キャラ一覧・詳細画面でも「主人公未作成なら主人公作成画面へ強制遷移」する（万が一の直接アクセス対策）。

------------------------------------------------------------------------

## 1. 目的

- ログインユーザーが**自分が持つキャラ（主人公・仲間・メカ）を一覧**で確認できること。
- 一覧のカードを**クリックすると詳細画面**に遷移し、**基礎ステータス（STR, INT, VIT, WIS, DEX, AGI, LUK, CAP）**を確認できること。
- ダッシュボードの**「宿舎」カード**からキャラ一覧（`/dashboard/characters`）へ遷移できること。
- 主人公が未作成の状態で一覧・詳細に侵入された場合は、**主人公作成画面へ強制遷移**すること。

------------------------------------------------------------------------

## 2. 用語

- **キャラ一覧**：そのユーザーが持つ全 Character（category = protagonist / companion / mech）の一覧。
- **宿舎**：ダッシュボード上のカードのラベル。クリックで `/dashboard/characters` へ遷移する。
- **キャラ詳細**：1体の Character の詳細画面。本 spec では**基礎ステータスの表示のみ**（編集・装備・スキルはスコープ外）。

------------------------------------------------------------------------

## 3. 入力（Input）

### 3.1 一覧取得

- **入力**：セッションの `userId`（Server Action または layout で getSession から取得）。
- **取得対象**：`Character` のうち `userId` が一致する行をすべて取得。category の指定はしない（主人公・仲間・メカをまとめて表示）。

### 3.2 詳細取得

- **入力**：キャラの `id`（URL パラメータ例：`/dashboard/characters/[id]`）。
- **制約**：その `id` の Character の `userId` が、セッションの `userId` と一致すること。一致しなければ 404 または権限エラー。

------------------------------------------------------------------------

## 4. 出力（Output）

### 4.1 一覧用の1件

```json
{
  "id": "clx...",
  "category": "protagonist",
  "displayName": "主人公",
  "iconFilename": "1.gif"
}
```

- `category`：`protagonist` / `companion` / `mech`。表示ではカードの背景色で区別する。
- `iconFilename`：NULL の場合は表示用にデフォルト画像またはプレースホルダを用意してよい。

### 4.2 詳細用（基礎ステータス）

```json
{
  "id": "clx...",
  "category": "protagonist",
  "displayName": "主人公",
  "iconFilename": "1.gif",
  "STR": 10,
  "INT": 10,
  "VIT": 10,
  "WIS": 10,
  "DEX": 10,
  "AGI": 10,
  "LUK": 10,
  "CAP": 60
}
```

- 詳細画面では上記の**基礎ステータスを表示するだけ**（編集・装備・スキルは本 spec では扱わない）。

------------------------------------------------------------------------

## 5. ルール

### 5.1 表示対象

- 一覧に表示するのは**ログインユーザーが所有する Character のみ**（`Character.userId === session.userId`）。
- 主人公・仲間・メカを**同じ一覧に表示**する。種別のフィルタ（タブ等）は**今回のスコープ外**（後で検討）。

### 5.2 カードの見た目

- 一覧の各キャラは**カード**で表示する。
- カードに表示する情報：**表示名（displayName）** と **アイコン（iconFilename）** のみ。
- **主人公・仲間・メカはカードの背景色で区別**する（例：主人公＝色A、仲間＝色B、メカ＝色C）。具体的な色は実装で決定し、docs/07 等に合わせる。

### 5.3 詳細画面

- カードを**クリック**すると、そのキャラの**詳細画面**（`/dashboard/characters/[id]`）に遷移する。
- 詳細画面では**基礎ステータス（STR, INT, VIT, WIS, DEX, AGI, LUK, CAP）が見られるだけ**とする。編集・装備・スキルは別 spec または後拡張。

### 5.4 主人公未作成時の強制遷移

- キャラ一覧・詳細のいずれかにアクセスしたとき、**主人公が未作成**（getProtagonist が null、または User.protagonistCharacterId が NULL）なら、**主人公作成画面（`/character/create`）へ強制遷移**する。
- 通常は dashboard の layout で未作成時に作成画面へリダイレクトするが、万が一直接 URL で侵入された場合に備え、一覧・詳細ページ側でも同様の判定を行う。

### 5.5 詳細の権限

- 詳細画面で表示するキャラは、**そのキャラの userId がログインユーザーの id と一致する場合のみ**表示する。一致しない場合は 404 または「見つかりません」等で返す。

------------------------------------------------------------------------

## 6. 処理フロー（概要）

### 6.1 キャラ一覧表示

1. セッション取得（未ログインならログイン画面へ）
2. getProtagonist で主人公の有無を確認 → いなければ `/character/create` へリダイレクト
3. ユーザーに紐づく全 Character を取得（getCharactersByUserId 相当）
4. 一覧を表示（カード：表示名・アイコン、背景色は category で区別）
5. 各カードは `/dashboard/characters/[id]` へのリンク

### 6.2 キャラ詳細表示

1. セッション取得
2. getProtagonist で主人公の有無を確認 → いなければ `/character/create` へリダイレクト
3. URL の [id] で Character を取得
4. character.userId === session.userId でなければ 404
5. 基礎ステータス（STR, INT, VIT, WIS, DEX, AGI, LUK, CAP）を表示

------------------------------------------------------------------------

## 7. 永続化データ / 一時データ

### 7.1 永続化するデータ

- **参照のみ**。キャラ一覧・詳細では Character の**読取のみ**行い、新規作成・更新・削除は行わない（一覧・詳細は参照画面）。

### 7.2 保存しないデータ

- 一覧の並び順をユーザーが変更する等は本 spec では扱わない。表示順は実装で決めてよい（例：主人公を先頭、あとは createdAt 順など）。

------------------------------------------------------------------------

## 8. 例（最低3ケース）

### ケース1：正常系（一覧表示）

- 前提：ログイン済み、主人公作成済み、キャラが1体以上いる
- 操作：ダッシュボードの「宿舎」カードをクリック
- 期待：`/dashboard/characters` が表示され、所有キャラがカードで一覧される。表示名・アイコンが表示され、主人公・仲間・メカで背景色が異なる。

### ケース2：正常系（詳細表示）

- 前提：ログイン済み、そのユーザーが持つキャラの id を指定
- 操作：一覧のカードをクリック → `/dashboard/characters/[id]`
- 期待：そのキャラの表示名・アイコンと基礎ステータス（STR, INT, VIT, WIS, DEX, AGI, LUK, CAP）が表示される。

### ケース3：主人公未作成で一覧に侵入した場合

- 前提：何らかの理由で主人公未作成の状態で `/dashboard/characters` にアクセスした（通常は layout でリダイレクトされる想定）
- 期待：一覧画面側でも主人公未作成を検知し、**主人公作成画面（`/character/create`）へ強制遷移**する。

### ケース4：他ユーザーのキャラ詳細へアクセス

- 前提：ログイン済み。URL に他ユーザーが持つキャラの id を直接指定
- 期待：そのキャラの userId がセッションの userId と一致しないため、**404 または「見つかりません」**を返す。データは表示しない。

------------------------------------------------------------------------

## 9. エラー条件 / NG

| 条件 | 挙動 |
|------|------|
| 未ログインで一覧・詳細にアクセス | 認証によりログイン画面へリダイレクト（既存 middleware） |
| 主人公未作成で一覧・詳細にアクセス | 主人公作成画面（`/character/create`）へ強制遷移 |
| 他ユーザーのキャラ id で詳細にアクセス | 404 または「見つかりません」 |
| 存在しないキャラ id で詳細にアクセス | 404 |

------------------------------------------------------------------------

## 10. 画面仕様

### 10.1 ダッシュボード（変更点）

| 項目 | 内容 |
|------|------|
| **変更** | **「宿舎」カード**を追加する。クリックで `/dashboard/characters` へ遷移する。 |
| **場所** | ダッシュボード上。仮戦闘ボタン等と同様、カードとして配置（実装でレイアウト決定）。 |
| **ラベル** | 「宿舎」（日本語表示でよい）。 |

### 10.2 キャラ一覧画面

| 項目 | 内容 |
|------|------|
| **URL** | `/dashboard/characters` |
| **前提** | ログイン済み。主人公作成済み（未作成なら `/character/create` へリダイレクト）。 |
| **見出し** | 「キャラクター」や「マイキャラ」等（実装で決定）。 |
| **要素** | キャラカードの一覧。各カード：表示名、アイコン。カードの背景色は category（主人公／仲間／メカ）で差をつける。カードクリックで詳細へ。 |
| **呼び出す API** | getCharactersByUserId 相当（ユーザーに紐づく全 Character 取得）。 |
| **空のとき** | キャラが0体の場合は空の一覧または「キャラがいません」等のメッセージを表示してよい（通常は主人公は必ず1体いる想定）。 |

### 10.3 キャラ詳細画面

| 項目 | 内容 |
|------|------|
| **URL** | `/dashboard/characters/[id]` |
| **前提** | ログイン済み。主人公作成済み。指定 id のキャラが存在し、かつそのキャラの userId がログインユーザーと一致すること。 |
| **見出し** | キャラの表示名等。 |
| **要素** | アイコン、表示名、**基礎ステータス**（STR, INT, VIT, WIS, DEX, AGI, LUK, CAP）の表示のみ。一覧へ戻るリンク。 |
| **呼び出す API** | getCharacterById 相当（id と userId で取得し、userId が一致する場合のみ表示）。 |
| **成功時** | 上記の内容を表示。 |
| **エラー時** | 他ユーザーのキャラ or 存在しない id の場合は 404。 |

------------------------------------------------------------------------

## 11. スコープ（本 spec で扱わないもの）

- 一覧の**フィルタ**（主人公のみ／仲間のみ／メカのみ）→ 後で検討。
- キャラの**編集**（名前・アイコン変更）。
- **仲間・メカの追加**（作成・入手）。
- 詳細画面での**装備・スキル**表示・編集。
- 並び順のユーザー変更。

---

## 12. 実装メモ

- **リポジトリ**：`character-repository` に「ユーザーに紐づく全 Character 一覧取得」と「id + userId で1件取得」を追加する。
- **保護**：一覧ページ・詳細ページのいずれでも、getProtagonist が null なら `redirect("/character/create")` する。
- **詳細の 404**：`character.userId !== session.userId` または該当 Character が存在しない場合は `notFound()` でよい。
- **カード背景色**：`docs/07_ui_guidelines.md` や既存テーマに合わせて、protagonist / companion / mech で異なる背景色（または枠色）を指定する。
