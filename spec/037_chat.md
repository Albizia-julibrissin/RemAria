# Spec: チャット（全体チャット・常駐UI）

`docs/00_overview.md` および `docs/022_chat_ui_design.md` に基づき、**全画面共通の常駐チャット**（開閉可能）を実装する。MVP では全体チャットのみ。永続ログは最小限とし、直近ログのみ保持する。

------------------------------------------------------------------------

## 0. 依存・横断

### 0.1 依存する spec

- **010_auth**：セッションが有効であること。送信・取得 API は getSession を前提とする。未ログイン時はチャット UI を非表示または「ログインすると利用できます」表示とする。

### 0.2 提供する API / 利用者

| API | 用途 | 呼び出し元 |
|-----|------|-----------|
| sendChatMessage | 全体チャットに 1 件送信する | チャットパネル内の送信ボタン |
| getRecentChatMessages | 直近 N 件のメッセージを取得する（表示用） | チャットパネル（開いたとき・ポーリング／SSE 等） |

------------------------------------------------------------------------

## 1. 目的

- プレイヤーがどの画面でも「世界が同時に動いている感覚」を得られるよう、**全画面共通の常駐チャット UI**（開閉可能）を提供する。
- **全体チャット**に短文を送信し、直近のメッセージ一覧を表示する。
- 永続ログは最小限に抑え、**直近ログのみ保持**する（保持件数または保持期間は 5. ルールで定義）。

------------------------------------------------------------------------

## 2. 用語

- **全体チャット**：全プレイヤーが参加する 1 チャンネル。MVP ではこれのみ。
- **チャットメッセージ**：1 件の投稿。送信者（User）、本文（body）、送信日時（createdAt）を持つ。
- **直近ログ**：取得 API が返す「直近 N 件」のメッセージ。永続化はこの範囲を超える古いデータを削除してよい（方針は 5. ルール）。

------------------------------------------------------------------------

## 3. 入力・出力

### 3.1 sendChatMessage の入力

- **セッション**：getSession() から得た userId。
- **body**（string）：本文。必須。最大長は 5. ルールで定義。

```json
{
  "body": "こんにちは、探索から帰還しました"
}
```

### 3.2 sendChatMessage の出力

- **成功時**：作成されたメッセージの id, userId, body, createdAt、および送信者表示名（User.name または id）を返してよい。クライアントは一覧に追加表示する。
- **失敗時**：バリデーションエラー・未ログイン・レート制限等の理由を返す。

### 3.3 getRecentChatMessages の入力

- **セッション**：getSession()。未ログインでも「直近 N 件」は返してよい（表示のみ）。未ログインで返すかは実装方針に任せる。
- **limit**（任意）：取得件数。未指定時はデフォルト値（例：50 件）。

### 3.4 getRecentChatMessages の出力

- **成功時**：直近 N 件のメッセージの配列。各要素は id, userId, body, createdAt、および送信者表示名。**createdAt の降順**（新しい順）。

```json
{
  "messages": [
    {
      "id": "cuid",
      "userId": "cuid",
      "senderName": "冒険者",
      "body": "本文",
      "createdAt": "2025-02-26T12:00:00.000Z"
    }
  ]
}
```

------------------------------------------------------------------------

## 4. 永続化データ（本 spec で利用するもの）

### 4.1 ChatMessage（新規）

| カラム | 型 | 制約 | 説明 |
|--------|-----|------|------|
| id | String (cuid) | PK | 主キー |
| userId | String | NOT NULL, FK→User.id | 送信者 |
| body | String | NOT NULL | 本文（最大長は 5. ルール） |
| createdAt | DateTime | NOT NULL, default now() | 送信日時 |

- **リレーション**：User に 1 対多（User.chatMessages）。
- **インデックス**：createdAt 降順で取得するため `@@index([createdAt])` を推奨。
- 永続ログは最小限のため、**古いメッセージの削除**（件数上限または期間上限）をバッチで行う設計とする（MVP では削除なしでも可。上限を設ける場合は 5. ルールに従う）。

------------------------------------------------------------------------

## 5. ルール

- **本文（body）**：必須。最大 **500 文字**（UTF-8）。空文字・空白のみは不可。
- **取得件数**：getRecentChatMessages のデフォルト limit は **50 件**。最大 100 件まで許容する。
- **永続化方針**：直近ログのみ保持する。例：直近 1000 件を保持し、それを超える古いメッセージは定期削除する。MVP では「削除しない」または「1000 件を超えた分を削除」のいずれでも可。
- **ミュート・ブロック・通報**：docs/00 で要件にあるが、**MVP では省略可**（022 に記載のとおり）。将来 spec で拡張する。

------------------------------------------------------------------------

## 6. 処理フロー

### 6.1 sendChatMessage

1. セッション検証（未ログインならエラー）。
2. body のバリデーション（必須・長さ 1〜500・空白のみ不可）。
3. （任意）レート制限チェック（例：同一ユーザー 1 分あたり 10 件まで）。MVP では省略可。
4. ChatMessage を 1 件作成（userId, body）。createdAt は default now()。
5. 作成したメッセージ（id, userId, body, createdAt）および送信者表示名を返す。

### 6.2 getRecentChatMessages

1. limit を決定（未指定なら 50、指定ありなら min(指定値, 100)）。
2. ChatMessage を createdAt 降順で limit 件取得。User を join して表示名を取得。
3. 配列をそのまま返す。

------------------------------------------------------------------------

## 7. 画面仕様（022 準拠）

チャット UI は**ルートレイアウト**（`app/layout.tsx`）に配置し、全画面で表示する。`docs/022_chat_ui_design.md` に従う。

| 項目 | 内容 |
|------|------|
| **配置** | ルート layout の children の外。ヘッダーと同じ階層にチャット用クライアントコンポーネント（例：`ChatFloating`）を配置。 |
| **前提** | ログイン時は送信・表示を有効。未ログイン時は非表示または「ログインすると利用できます」のみ表示。 |
| **閉じた状態** | 右下にフロートボタン（例：「💬 チャット」）。スタイル：`bg-base-elevated border border-base-border text-text-primary hover:border-brass hover:text-brass`。クリックで開く。 |
| **開いた状態** | 右下から上方向にパネル（幅 320〜400px、高さ 40〜50vh）。ヘッダー（「全体チャット」＋閉じるボタン）、メッセージリスト（縦スクロール）、入力欄＋送信ボタン。 |
| **呼び出す API** | 開いたとき・一定間隔：`getRecentChatMessages`。送信ボタン：`sendChatMessage`。 |
| **開閉の保持** | クライアント state（useState）。任意で localStorage（例：`remaeria-chat-open`）に保存し再読み込み時に復元。 |

ミュート／ブロック／通報のメニューは MVP では省略可（022 記載どおり）。

------------------------------------------------------------------------

## 8. エラー条件

- **未ログインで sendChatMessage**：401 またはエラーメッセージを返し、送信しない。
- **body が空・空白のみ・501 文字以上**：400 とバリデーションエラーメッセージを返す。
- **getRecentChatMessages**：未ログイン時は空配列を返すか、表示用に直近 N 件を返すかは実装方針に任せる（認証必須にするなら 401）。

------------------------------------------------------------------------

## 9. テスト観点

- sendChatMessage：未ログインで 401（または同等）。body 空で 400。body 500 文字で成功、501 文字で 400。
- getRecentChatMessages：createdAt 降順で返ること。limit 未指定でデフォルト件数。limit 100 超で 100 件までにクリップ。
- UI：ルート layout に配置されていること。開閉でパネルが表示／非表示になること。
