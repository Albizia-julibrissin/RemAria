# 018: 設備配置エリアと進行解放・画面想定

設備を「どこに」「いくつ」置けるかを決める**設置エリア**と、進行に応じてエリアが解放されていく流れ、および画面まわりの想定をまとめる。  
017（設備の型・希少部品）と組み合わせて、設置時には「設置エリアを選ぶ → コスト・枠の制限内で設備を配置する」形を想定する。

---

## 1. 目的

- **進行**：クリアするごとに設置エリアが増え、配置の幅が広がる。
- **制約**：各設置エリアに「最大コスト」「最大配置数」があり、その範囲で設備を配置する。
- **画面**：設置エリアの選択 → そのエリア内での設備配置（建設・解体・型の選択）を扱う UI の前提を定義する。

---

## 2. 設置エリアの概念

### 2.1 設置エリアとは

- 設備を**配置する枠**の単位。プレイヤーは「どの設置エリアに設備を置くか」を選ぶ。
- 1 つの設置エリアには次の**制限**がある：
  - **最大コスト**（例：100）… そのエリアに配置した設備のコスト合計の上限。
  - **最大配置数**（例：3）… そのエリアに置ける設備の個数上限。
- 設置エリアは**進行に応じて解放**され、最初は 0 または 1 エリアから始まり、クリアするごとに増えていく想定。

### 2.2 進行と解放の流れ（例）

- **探索エリア**が複数ある（例：草原なんちゃら、山なんちゃら、…）。
- 各探索エリアには**ノード**（行き先）が複数ある。  
  例：**草原なんちゃら** → 「洞窟」「遺跡」など。  
  「草原なんちゃら→洞窟」「草原なんちゃら→遺跡」を**すべてクリア**すると、**草原エリア用の施設（設置）エリア**が解放される、といったイメージ。
- 別の探索エリア（例：山なんちゃら）も同様に、そのエリア配下のノードを全部クリアすると、**山エリア用の設置エリア**が解放される。
- 進行ごとに**設置エリアがどんどん増えていく**。

### 2.3 用語の整理

| 用語 | 説明 |
|------|------|
| **探索エリア** | プレイヤーが探索する「場所」の大枠（例：草原なんちゃら）。行先ノードの親。 |
| **ノード** | 探索エリア内の具体的な行き先（例：洞窟、遺跡）。クリア条件・報酬テーブルを持つ想定。 |
| **設置エリア** | 設備を配置する枠。探索エリアのノードをすべてクリアすると解放される（上記の例では「草原エリアの施設エリア」など）。最大コスト・最大配置数を持つ。 |
| **設備** | 017 で定義した「設備」（FacilityType）と「型」（基本型／派生型）。1 設備 = 1 つのコストを持つ。 |

### 2.4 生産の前提

- **設備は仲間キャラの配備がなくても、基礎生産力で稼働する。**  
  配備がある場合は 017 のとおり配備キャラのステータスで倍率等が変動するが、配備がなくても基礎的な生産は実行される。

---

## 3. 初期エリア（強制配置）

### 3.1 初期エリアの仕様

- ゲーム開始時から解放されている設置エリアを**初期エリア**とする。
- 初期エリアの制限：**最大コスト 200、最大配置数 5**。
- 初期エリアには次の 5 設備を**あらかじめ配置した状態**でスタートする（強制配置）。

| # | 設備 | 役割（入出力） |
|---|------|----------------|
| 1 | 川探索 | 水を取得（資源探索・output のみ） |
| 2 | 浄水施設 | 水 → 飲料水 |
| 3 | 小麦畑 | 小麦を取得（資源探索・output のみ） |
| 4 | 小麦製粉器 | 小麦 → 小麦粉 |
| 5 | 携帯食料包装 | 飲料水 ＋ 小麦粉 → 携帯食料 |

### 3.2 生産チェーン

- **川探索**で **水** を取得。
- **浄水施設**に水を input → **飲料水** を output。
- **小麦畑**で **小麦** を取得。
- **小麦製粉器**に小麦を input → **小麦粉** を output。
- **携帯食料包装**に飲料水と小麦粉を input → **携帯食料** を output。

→ この**携帯食料**を、初期エリアの探索で消費する。初期エリアで探索を行うためには、上記 5 設備による携帯食料の生産が前提となる。

### 3.3 強制配置の扱い

- 上記 5 設備は**強制配置**とする。プレイヤーは解体・入れ替えできない。
- 初期エリアのコスト・配置数制限（200／5）のうち、強制配置 5 つで既に 5 枠を使用するため、初期エリアでは**追加の設備は配置しない**想定とする。  
  （あるいは強制配置分を「枠・コストを消費しない」特別枠として扱い、同一エリアに追加で 0〜N 設備を置けるようにするかは、実装時にどちらかで定義する。本 doc では「初期エリアは 5 設備で固定」とする。）

### 3.4 初期エリアの仮数値（コスト・生産量）

以下は**仮の数値**。コスト合計 200 ぴったり、かつ 1 時間あたり携帯食料 100 個がロスなく生産されるように設定する。

#### コスト（合計 200）

| 設備 | コスト（仮） |
|------|--------------|
| 川探索 | 40 |
| 浄水施設 | 40 |
| 小麦畑 | 40 |
| 小麦製粉器 | 40 |
| 携帯食料包装 | 40 |
| **合計** | **200** |

#### 生産仕様（仮）

| 設備 | 周期 | 入力 | 出力 | 1時間あたり |
|------|------|------|------|-------------|
| 川探索 | 60分 | — | 水 600 | 水 600 |
| 浄水施設 | 20分 | 水 200 | 飲料水 100 | 飲料水 300 |
| 小麦畑 | 120分 | — | 小麦 200 | 小麦 100 |
| 小麦製粉器 | 30分 | 小麦 50 | 小麦粉 150 | 小麦粉 300 |
| 携帯食料包装 | 15分 | 飲料水 75 ＋ 小麦粉 75 | 携帯食料 25 | 携帯食料 100 |

#### 1時間あたりの試算（ロスなし）

- **水**：川 600/時 → 浄水が 20分×3 回で 水 200×3 = 600 消費・飲料水 100×3 = **300/時**。✓ 過不足なし。
- **飲料水**：浄水 300/時 → 携帯食料包装が 15分×4 回で 飲料水 75×4 = 300 消費。✓ 過不足なし。
- **小麦**：小麦畑 200/120分 = 100/時 → 製粉が 30分×2 回で 小麦 50×2 = 100 消費・小麦粉 150×2 = **300/時**。✓ 過不足なし。
- **小麦粉**：製粉 300/時 → 携帯食料包装が 15分×4 回で 小麦粉 75×4 = 300 消費。✓ 過不足なし。
- **携帯食料**：15分×4 回で 25×4 = **100/時**。✓

→ 上記仮数値で、**1時間に携帯食料 100 個**がロスなく生産される。

## 4. コストと配置数制限（追加解放エリア）

追加解放される設置エリアでは以下が適用される。初期エリアの強制配置は除く。

### 4.1 設備のコスト

- **設備（種類＋型）ごとにコスト**が決まる。  
  例：川探索設備・ベース型 = 20、山探索設備・ベース型 = 50。
- 設置エリアに設備を 1 つ置くたびに、その設備のコストがエリアの**使用コスト**に加算される。
- 同一設備を複数置く場合も、1 つあたりのコスト × 個数で計算する。

### 4.2 制約の満たし方

- 設置エリアには**最大コスト**と**最大配置数**がある。
- **配置数**：そのエリアに置いている設備の**個数**が最大配置数以下であること。
- **コスト**：そのエリアに置いている設備の**コスト合計**が最大コスト以下であること。
- 両方を満たす範囲で、設備の組み合わせは自由（同じ設備を複数可、異なる設備の混在も可）。

### 4.3 例

- 設置エリアの制限：**最大コスト 100、最大配置数 3**。
- 川探索設備・ベース型：コスト **20**。山探索設備・ベース型：コスト **50**。

| 配置例 | コスト合計 | 配置数 | 備考 |
|--------|------------|--------|------|
| 川 3 つ | 20×3 = 60 | 3 | コスト・枠ともに余裕あり。 |
| 山 2 つ | 50×2 = 100 | 2 | コスト上限いっぱい、枠は 1 つ余り。 |
| 川 2 つ + 山 1 つ | 20×2 + 50 = 90 | 3 | コスト・枠ともに上限内。 |
| 山 3 つ | 50×3 = 150 | 3 | コスト超過のため不可。 |

→ 「同じの 3 つが限界」「山は 2 個が限界」「川と山を組み合わせれば川 2・山 1 で最大」といった運用になる。

## 5. 設置の流れ（操作・画面想定）

### 5.1 設置時の選択順

1. **設置エリアを選ぶ**  
   解放済みの設置エリア一覧から、配置したいエリアを選択する。
2. **そのエリアの制限を確認する**  
   最大コスト（例：100）と最大配置数（例：3）、および現在の使用コスト・現在の配置数を表示する。
3. **設備を配置する**  
   - 建設：研究で解放済みの設備の「基本型」、または希少部品を持つ場合は「派生型」を選び、そのエリアに 1 つ配置する。  
   - 派生型で建てる場合は希少部品を 1 個消費する（017 の消耗仕様）。
   - 配置すると、エリアの使用コスト＋その設備のコスト、配置数＋1 となる。
4. **制限を超えない範囲**で、追加配置・入れ替え・解体を行う。  
   解体した設備の希少部品は返却されない（017 準拠）。

### 5.2 画面・UI で想定する要素

- **設置エリア一覧（またはマップ）**  
  - 解放済みの設置エリアが一覧または地図上で選べる。  
  - 未解放の設置エリアは「〇〇をすべてクリアで解放」などと表示してもよい。
- **設置エリア詳細（配置画面）**  
  - 選択した設置エリアの名前。  
  - **最大コスト / 現在の使用コスト**（例：50 / 100）。  
  - **最大配置数 / 現在の配置数**（例：2 / 3）。  
  - そのエリアに並んでいる**設備の一覧**（設備名・型・コスト・必要に応じて配備キャラなど）。  
  - 操作：**建設**（設備＋型を選ぶ）、**解体**（配置した設備を減らす）。  
  - 建設時：設備一覧から「どの設備のどの型で建てるか」を選び、コスト・枠に収まるかチェックして確定。
- **設備選択（建設時）**  
  - 研究で解放済みの設備は「基本型」で選択可能。  
  - 希少部品を所持している設備は「派生型」（α型・β型など）も選択可能。選択すると部品 1 個消費と明示する。

上記は**画面仕様の前提**として記載。実際のワイヤー・画面数・遷移は別途 UI 仕様で詰める。

（初期エリアは強制配置のため、建設・解体の操作対象外とする。）

## 6. データ・スキーマ上の対応（方針）

- **設置エリア**  
  - マスタ：設置エリア ID、名前、最大コスト、最大配置数、解放条件（例：探索エリア ID またはノードクリア条件）など。  
  - 解放状態：プレイヤーごとに「どの設置エリアが解放済みか」を保持する。
- **設置済み設備（FacilityInstance）**  
  - どの**設置エリア**に属するか（設置エリア ID）。  
  - どの**設備**（FacilityType）で、どの**型**（基本型／派生型）か。  
  - 017 で想定した facilityTypeId と variantId（または variantCode）に加え、設置エリア ID を持たせる。
- **設備のコスト**  
  - 設備＋型ごとのコストは、FacilityType と型（FacilityVariant 等）に持たせるか、別テーブルで「設備＋型 → コスト」を定義する。  
  詳細は 017 実装時の DB 設計（08 または 017 用 spec）で定義する。

探索エリア・ノード・クリア条件・報酬テーブルは探索コンテンツ側の設計に委ねる。本 doc では「ノードをすべてクリア → その探索エリアに紐づく設置エリアが解放される」という**関係**だけを前提とする。

---

## 7. 017 との関係

- 設備の**種類**と**型**（基本型／派生型）は 017 の定義に従う。
- 設置時には「設置エリアを選ぶ」→「そのエリアのコスト・配置数制限内で、設備（＋型）を選んで建設」となる。
- 派生型で建てる場合は希少部品を消費し、取り外し・返却はなし（017 と同じ）。
- 設備ごとの**コスト**は、017 には含めず本 doc（018）で定義する。実装時は設備マスタまたは型マスタにコストを持たせる。

---

## 8. まとめ

- **設置エリア** = 設備を配置する枠。進行（探索ノード全クリアなど）で解放され、数が増えていく。
- 各設置エリアには**最大コスト**と**最大配置数**があり、その範囲で設備を配置する（初期エリアは強制配置 5 で固定）。
- **設備は配備がなくても基礎生産力で稼働する**。配備があると 017 のとおりステータスで倍率等が変動する。
- **初期エリア**はコスト 200・配置 5。川探索・浄水施設・小麦畑・小麦製粉器・携帯食料包装の 5 設備を強制配置。水→飲料水、小麦→小麦粉、飲料水＋小麦粉→携帯食料の生産チェーンで携帯食料を生産し、初期エリアの探索で消費する。強制配置は解体・変更不可。
- **設備（種類＋型）ごとにコスト**があり、追加エリアでは同じ設備を複数置く場合はコスト×個数で合計する。
- 画面では「設置エリア選択 → エリアごとの配置画面（コスト・枠表示、建設・解体）」を想定。初期エリアは配置変更不可。詳細は UI 仕様で拡張する。
- 探索エリア・ノード・クリア条件は探索設計に委ね、本 doc では「ノード全クリア → 設置エリア解放」のつながりだけを前提とする。

以上を設備配置エリアと画面想定の基準とする。
