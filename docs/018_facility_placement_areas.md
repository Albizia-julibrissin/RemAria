# 018: 設備配置と進行解放（設置枠・コスト上限）

設備を「いくつ」「どれだけコストまで」置けるかを決める**設置枠数**と**コスト上限**、および進行に応じてそれらが拡張されていく流れをまとめる。  
**エリア制は採用しない**。工業は 1 つのプールで、受け取りも一括。進行は「設置枠数アップ」「コスト上限アップ」で表現する。世界観上の演出は将来実装の**称号**（例：「〇〇工業エリア解放！」）で補う。

017（設備の型・希少部品）と組み合わせて、設置時には「設置枠・コスト制限内で設備を配置する」形を想定する。

---

## 1. 目的

- **進行**：クリアするごとに**設置枠数**または**コスト上限**が増え、配置できる設備の幅が広がる。
- **制約**：**最大コスト**と**最大配置数（設置枠）**を 1 セットで持ち、その範囲で設備を配置する（エリアの選択はない）。
- **画面**：工業画面で、枠・コストの制限内での設備配置（建設・解体・型の選択）を扱う。

---

## 2. 設置枠とコスト上限の概念

### 2.1 単一プール

- 設備は**エリアに分けず、1 つの工業プール**に配置する。プレイヤーが「どのエリアに置くか」を選ぶことはない。
- そのプールに対して次の**制限**がある：
  - **最大コスト**（例：200 → 進行で 300, 400 … に増加）
  - **最大配置数（設置枠）**（例：5 → 進行で 6, 7 … に増加）
- 進行（探索テーマ攻略完了など）に応じて、**最大コスト**や**設置枠数**が上がっていく想定。

### 2.2 進行と拡張の流れ（例）

- **探索テーマ**を攻略完了すると、報酬として「**工業拡張**」が付くテーマがある（020 と接続）。
- 工業拡張の内容例：
  - **設置枠 +1**（最大配置数が 1 増える）
  - **コスト上限 +100**（最大コストが 100 増える）
- テーマごとに「工業拡張あり／なし」「枠アップ／コストアップ／両方」をマスタで設定する。
- **世界観・演出**：システム上は「枠が増えた」「コストが増えた」だが、将来実装する**称号**で「〇〇工業エリア解放！」などを表示し、雰囲気を出す。

### 2.3 用語の整理

| 用語 | 説明 |
|------|------|
| **設置枠（最大配置数）** | その時点で配置できる設備の**個数上限**。進行で増加する。 |
| **コスト上限（最大コスト）** | 配置した設備の**コスト合計の上限**。進行で増加する。 |
| **工業拡張** | 進行報酬による「設置枠数アップ」または「コスト上限アップ」。エリア解放ではない。 |
| **設備** | 017 で定義した「設備」（FacilityType）と「型」（基本型／派生型）。1 設備 = 1 つのコストを持つ。 |

### 2.4 生産の前提

- **設備は仲間キャラの配備がなくても、基礎生産力で稼働する。**  
  配備がある場合は 017 のとおり配備キャラのステータスで倍率等が変動するが、配備がなくても基礎的な生産は実行される。

---

## 3. 初期状態（強制配置）

### 3.1 初期の仕様

- ゲーム開始時点で**設置枠 5・コスト上限 200**とする。
- 次の 5 設備を**あらかじめ配置した状態**でスタートする（**強制配置**）。

| # | 設備 | 役割（入出力） |
|---|------|----------------|
| 1 | 川探索 | 水を取得（資源探索・output のみ） |
| 2 | 浄水施設 | 水 → 飲料水 |
| 3 | 小麦畑 | 小麦を取得（資源探索・output のみ） |
| 4 | 小麦製粉器 | 小麦 → 小麦粉 |
| 5 | 携帯食料包装 | 飲料水 ＋ 小麦粉 → 携帯食料 |

### 3.2 生産チェーン

- **川探索**で **水** を取得。
- **浄水施設**に水を input → **飲料水** を output。
- **小麦畑**で **小麦** を取得。
- **小麦製粉器**に小麦を input → **小麦粉** を output。
- **携帯食料包装**に飲料水と小麦粉を input → **携帯食料** を output。

→ この**携帯食料**を、探索で消費する。探索を行うためには、上記 5 設備による携帯食料の生産が前提となる。

### 3.3 強制配置の扱い

- 上記 5 設備は**強制配置**とする。プレイヤーは解体・入れ替えできない。
- 初期の枠 5 を強制配置 5 つで使用するため、**追加の設備は配置しない**想定でスタートする。  
  進行で設置枠が 6 以上になれば、6 枠目以降にプレイヤーが設備を追加配置できる。
- （強制配置分を「枠・コストを消費しない」特別枠として扱い、初期から追加で 0〜N 設備を置けるようにするかは、実装時にどちらかで定義する。本 doc では「初期は 5 設備で固定、枠アップで追加可能」とする。）

### 3.4 初期の仮数値（コスト・生産量）

以下は**仮の数値**。コスト合計 200 ぴったり、かつ 1 時間あたり携帯食料 100 個がロスなく生産されるように設定する。

#### コスト（合計 200）

| 設備 | コスト（仮） |
|------|--------------|
| 川探索 | 40 |
| 浄水施設 | 40 |
| 小麦畑 | 40 |
| 小麦製粉器 | 40 |
| 携帯食料包装 | 40 |
| **合計** | **200** |

#### 生産仕様（仮）

| 設備 | 周期 | 入力 | 出力 | 1時間あたり |
|------|------|------|------|-------------|
| 川探索 | 60分 | — | 水 600 | 水 600 |
| 浄水施設 | 20分 | 水 200 | 飲料水 100 | 飲料水 300 |
| 小麦畑 | 120分 | — | 小麦 200 | 小麦 100 |
| 小麦製粉器 | 30分 | 小麦 50 | 小麦粉 150 | 小麦粉 300 |
| 携帯食料包装 | 15分 | 飲料水 75 ＋ 小麦粉 75 | 携帯食料 25 | 携帯食料 100 |

#### 1時間あたりの試算（ロスなし）

- **水**：川 600/時 → 浄水が 20分×3 回で 水 200×3 = 600 消費・飲料水 100×3 = **300/時**。✓ 過不足なし。
- **飲料水**：浄水 300/時 → 携帯食料包装が 15分×4 回で 飲料水 75×4 = 300 消費。✓ 過不足なし。
- **小麦**：小麦畑 200/120分 = 100/時 → 製粉が 30分×2 回で 小麦 50×2 = 100 消費・小麦粉 150×2 = **300/時**。✓ 過不足なし。
- **小麦粉**：製粉 300/時 → 携帯食料包装が 15分×4 回で 小麦粉 75×4 = 300 消費。✓ 過不足なし。
- **携帯食料**：15分×4 回で 25×4 = **100/時**。✓

→ 上記仮数値で、**1時間に携帯食料 100 個**がロスなく生産される。

## 4. コストと配置数制限（拡張後）

進行で設置枠・コスト上限が増えたあと、強制配置以外の枠では以下が適用される。

### 4.1 設備のコスト

- **設備（種類＋型）ごとにコスト**が決まる。  
  例：川探索設備・ベース型 = 20、山探索設備・ベース型 = 50。
- 設備を 1 つ置くたびに、その設備のコストが**使用コスト**に加算される。
- 同一設備を複数置く場合も、1 つあたりのコスト × 個数で計算する。

### 4.2 制約の満たし方

- **配置数**：配置している設備の**個数**が最大配置数（設置枠）以下であること。
- **コスト**：配置している設備の**コスト合計**が最大コスト以下であること。
- 両方を満たす範囲で、設備の組み合わせは自由（同じ設備を複数可、異なる設備の混在も可）。

### 4.3 例

- 進行後：**最大コスト 300、設置枠 7**（強制 5 で 5 使用、あと 2 枠＋コスト余裕）。
- 川探索設備・ベース型：コスト **20**。山探索設備・ベース型：コスト **50**。

| 配置例 | コスト合計 | 配置数 | 備考 |
|--------|------------|--------|------|
| 強制 5 ＋ 川 2 つ | 200＋40 = 240 | 7 | 枠いっぱい。 |
| 強制 5 ＋ 山 2 つ | 200＋100 = 300 | 7 | コスト上限いっぱい。 |

## 5. 設置の流れ（操作・画面想定）

### 5.1 設置時の流れ

1. **工業画面を開く**  
   現在の設置枠・コスト上限、使用中枠・使用コスト、配置済み設備一覧を表示する。
2. **制限を確認する**  
   最大コスト（例：300）と設置枠（例：7）、および現在の使用コスト・現在の配置数を表示する。
3. **設備を配置する**（枠に余裕がある場合）  
   - 建設：研究で解放済みの設備の「基本型」、または希少部品を持つ場合は「派生型」を選び、1 つ配置する。  
   - 派生型で建てる場合は希少部品を 1 個消費する（017 の消耗仕様）。
   - 配置すると、使用コスト＋その設備のコスト、配置数＋1 となる。
4. **制限を超えない範囲**で、追加配置・入れ替え・解体を行う（強制配置 5 は除く）。  
  解体した設備の希少部品は返却されない（017 準拠）。

### 5.2 画面・UI で想定する要素

- **工業（設備配置）画面**  
  - **最大コスト / 現在の使用コスト**（例：200 / 200）。  
  - **設置枠 / 現在の配置数**（例：5 / 5）。  
  - 配置済み**設備の一覧**（設備名・型・コスト・必要に応じて配備キャラなど）。  
  - 操作：**建設**（枠に余裕があるとき、設備＋型を選ぶ）、**解体**（強制配置以外を減らす）。  
  - 建設時：設備一覧から「どの設備のどの型で建てるか」を選び、コスト・枠に収まるかチェックして確定。
- **設備選択（建設時）**  
  - 研究で解放済みの設備は「基本型」で選択可能。  
  - 希少部品を所持している設備は「派生型」も選択可能。選択すると部品 1 個消費と明示する。

上記は**画面仕様の前提**として記載。実際のワイヤー・画面数・遷移は別途 UI 仕様で詰める。

（強制配置 5 は建設・解体の操作対象外とする。）

## 6. データ・スキーマ上の対応（方針）

- **設置枠・コスト上限**  
  - ユーザー（または Factory 等）ごとに **maxSlots**（最大配置数）と **maxCost**（最大コスト）を持つ。  
  - 進行（テーマ攻略完了など）でこれらを増やす。解放済み「設置エリア」の管理は行わない。
- **設置済み設備（FacilityInstance）**  
  - **userId** で所有者を紐づける。**設置エリア ID は持たない**（単一プールのため）。  
  - どの**設備**（FacilityType）で、どの**型**（基本型／派生型）か。  
  - 017 の facilityTypeId と variantId（または variantCode）を持つ。表示順用に displayOrder 等を持ってもよい。
- **設備のコスト**  
  - 設備＋型ごとのコストは、FacilityType と型（FacilityVariant 等）に持たせるか、別テーブルで「設備＋型 → コスト」を定義する。  
  詳細は 017 実装時の DB 設計（08 または 017 用 spec）で定義する。

探索テーマ・攻略完了条件は探索設計に委ねる。本 doc では「テーマ攻略完了 → 工業拡張（枠アップ／コストアップ）」のつながりだけを前提とする。

---

## 7. 017 との関係

- 設備の**種類**と**型**（基本型／派生型）は 017 の定義に従う。
- 設置時には「設置枠・コスト制限内で、設備（＋型）を選んで建設」となる。
- 派生型で建てる場合は希少部品を消費し、取り外し・返却はなし（017 と同じ）。
- 設備ごとの**コスト**は、017 には含めず本 doc（018）で定義する。実装時は設備マスタまたは型マスタにコストを持たせる。

---

## 8. まとめ

- **エリア制は廃止**。設備は**単一の工業プール**に配置し、**設置枠数**と**コスト上限**で制限する。
- **進行**は「設置枠数アップ」「コスト上限アップ」（工業拡張）。探索テーマ攻略完了などで付与する。
- **世界観・演出**は、将来実装の**称号**（例：「〇〇工業エリア解放！」）で補う。
- **設備は配備がなくても基礎生産力で稼働する**。配備があると 017 のとおりステータスで倍率等が変動する。
- **初期**は設置枠 5・コスト 200。川探索・浄水施設・小麦畑・小麦製粉器・携帯食料包装の 5 設備を強制配置。強制配置は解体・変更不可。枠・コストが増えれば追加配置可能。
- **設備（種類＋型）ごとにコスト**があり、同一設備を複数置く場合はコスト×個数で合計する。
- 画面では「工業画面で枠・コスト表示と設備一覧、建設・解体」を想定。詳細は UI 仕様で拡張する。
- 探索テーマ・攻略完了は探索設計に委ね、本 doc では「テーマ攻略完了 → 工業拡張」のつながりだけを前提とする。

以上を設備配置と進行解放の基準とする。
