# 020: 探索機能 設計（案）

探索の構造・パーティ・戦闘/技能判定・ドロップ・目標（テーマ攻略完了条件）を定義する。
018（工業拡張）とは「テーマ攻略完了時、そのテーマが工業拡張付きなら設置枠数・コスト上限をアップ」で接続。世界観の演出は将来実装の称号（例：「〇〇機工区解放！」）で補う。周回前提のハクスラ／型ドロップがメインループ。

**UX 目標**：手軽に始められ、かつ飽きづらい（飽きないとは言わない）体験。出撃までのステップは少なく、本編の「イベント→選択→結果」のリズムと消耗品・ドロップの裁量で毎回の違いを出す。

---

## 1. 探索の階層と言葉の使い方

| 階層 | 説明 | 例 |
|------|------|-----|
| **テーマ** | 世界観・ストーリー上の地域の大枠。この単位で「攻略完了」を判定する。 | 錆びれた森林地区（元環境保護区サイジアナー） |
| **エリア** | テーマに属する、実際にプレイヤーが選んで探索する場所（＝ゲーム的な「マップ」）。1 テーマあたり複数。 | 森林監視施設、森林奥地、過成長植物園 |

- テーマの**最終エリア**にはマナコアが存在する（世界観：999 のマナコア裏設定）。
- 例：錆の森林テーマの最終エリア = **過成長植物園**（マナコアがある場所）。

### 1.1 用語の統一

- 以降の仕様では、**上位を「テーマ」・下位を「エリア」** で統一する。
- UI 上で「マップ」という文言を使う場合も、仕様上は「エリア（＝マップ）」として扱う。

---

## 2. パーティ編成

- **最大3枠**：主人公（アカウントキャラ）・仲間・メカ。各枠は**0 or 1**（出さない選択可）。
- **最低1人**いれば探索出撃可能（ソロでも可）。
- 01_features の「パーティ3体固定」から「1〜3体で選択可能」に変更する案。

---

## 3. 探索フロー（1回の探索＝1エリア選択）

### 3.1 出撃前（準備）

1. **テーマを選ぶ** → そのテーマ内の**エリア**を1つ選ぶ。
2. **パーティプリセットを選ぶ**：作戦室（039）で編集・保存済みのプリセットから1つ選択。編成（1〜3枠）と作戦スロットがそのまま探索戦闘に使われる。
3. **持ち込む消耗品を選ぶ**：所持している探索用消耗品のうち、今回の探索に持ち込む種類・数を選択（上限は別途マスタまたは定数で定義。例：最大 N 種×M 個）。
4. **出撃**：上記で確定し、探索開始。

### 3.2 探索中（イベントループ）

5. 各エリアには「**戦闘イベントの必要回数**」（雑魚戦の回数＋強敵戦1回）が設定されており、その回数ぶんの戦闘をこなすと最低限の探索が完了する。
6. 戦闘イベントの合間や前後で、**一定確率で技能判定イベントが挟まる**：
   - 「運が良い」周回では技能イベントが多く発生し、成功すればそのぶんドロップ枠が増える。
   - **発生判定**：**各イベント（戦闘・技能など）が終了するたびに**、次に「技能イベントか戦闘イベントか」を抽選する（事前にキューを積まない方式）。発生確率はエリアごとにマスタで定義（`baseSkillEventRate` 等）。
   - **拡張**：技能に限らず、将来的に他の探索イベント（技能強制成功・ハイアンドローで連勝すると枠増等の射幸要素）を追加できるよう、イベント種別は汎用化しておく。
7. イベント1回分の流れ：
   - **技能判定イベント**：発生時、プレイヤーが**どの基礎ステで解決するか**を1つ選択。判定後、**イベント終了時に消耗品を使うか**を選択可能（後述）。
   - **戦闘イベント**：エリアごとに設定された敵と、既存の戦闘ロジック（020 仮戦闘・038/039/040）で戦闘。敵の選出・配置は **spec/050** に従い、探索では **row は選出順で上から固定**（1体目＝上段、2体目＝中段、3体目＝下段）、**列（col）はマスタの設定**。戦闘終了後、同様に**消耗品を使うか**を選択可能。
8. 規定の雑魚戦回数をこなし終えると**強敵**に到達。強敵勝利後に**領域主出現の有無を抽選**。領域主が出れば戦闘、出なければその時点で探索終了。
9. **報酬確定**：エリアのドロップテーブルと探索結果（クリア数・技能成功数など）に応じてドロップ数を算出し、報酬を受け取って1サイクル終了。

### 3.3 消耗品の使用タイミング

- **技能イベント／戦闘イベントの直後**に、持ち込み済みの消耗品から「使う／使わない」を選択できる。
- 想定効果の例：HP 回復・MP 回復・ドロップ枠アップ（次のドロップ数加算）・次のイベント時ステアップ（一時バフ）など。効果の正本は 026 および探索用消耗品 spec で定義する。

## 4. 技能判定イベント（詳細）

### 4.1 基準値と補正

- **エリア**ごとに、**全基礎ステの基準値**を持つ。例：STR 100、VIT 150、DEX 80 …（STR, INT, VIT, WIS, DEX, AGI, LUK の各値）。
- **イベントの性質**ごとに、**全基礎ステの補正値（倍率）**を持つ。例：STR 130％、VIT 80％ …。
- そのイベントでステータス X で挑んだ場合の**必要値** = そのエリアの「X の基準値」× そのイベントの「X の補正倍率」。

例：エリア基準が STR 100・VIT 150、イベント補正が STR 130％・VIT 80％ のとき  
→ STR で挑むなら必要値 130（100×130％）、VIT で挑むなら必要値 120（150×80％）。

### 4.2 プレイヤー選択と判定

- イベント発生時、プレイヤーは**好きな基礎ステータスを1つ選んで**解決に挑む。
- 判定に使う値は、**パーティ内で選択したステが最高のキャラ**を自動で選び、そのキャラのその基礎ステを用いる。
- **成功判定**：
  - 挑んだキャラのその基礎ステ **≥ 必要値** → **100％クリア**。
  - **＜ 必要値**のとき → 成功率 = **自分ステ／必要値**（小数点以下は実装で扱う）。例：VIT 100 で VIT を選択し必要値 120 なら、約 83.3％で成功。
- クリア時：ドロップ数への加算（後述）。

## 5. 戦闘の種類と発生ルール

- 各エリアに**雑魚戦の発生回数**を設定する。MVP の基準値は「**5 戦 ＋ 強敵 1 戦**」とし、全読み 10 分／流し読み 5 分／スキップ 1〜2 分程度のプレイ時間を想定する。
- **雑魚戦**が規定回数発生し、規定回数クリアした**次の戦闘**で**強敵戦**。
- 強敵戦の後、**領域主戦の抽選**を行い、領域主戦を行うかどうかを決定（領域主は**全エリアに存在**する想定）。
- 結果パターン：
  - **全滅** → その時点で探索終了、報酬は全滅時ルールで確定。
  - **強敵クリア** → 強敵まで報酬確定。
  - **領域主クリア** → 領域主まで報酬確定＋領域主専用ドロップ等。

## 6. ドロップ数ルール（案）

### 6.1 枠数（スロット）の考え方

- 各エリアごとに**標準ドロップ枠数（min〜max）**を設定する（例：3〜5枠）。
- 探索中の行動結果に応じて、**この枠数に「＋何枠」として加算」するだけ**で表現する。  
  （下限/上限を別々に操作する複雑なルールは採用しない）
- 誰がクリアしたか（主人公/仲間/メカ）ではなく、**パーティとしてのクリア回数・結果**で加算する。

### 6.2 枠数の加算ルール（MVP 案）

- **雑魚戦勝利**：+1 枠（勝利した戦闘ごとに）
- **技能判定成功**：+1 枠（成功した技能イベントごとに）
- **強敵クリア**：+2 枠
- **領域主クリア**：**通常枠は増やさず、「領域主専用ドロップ枠」を別に 1 枠追加**する。
  - 領域主専用枠は、**領域主専用ドロップテーブル**から抽選する。
  - 例：その領域主専用素材、特定アイテムの確定ドロップなど。

最終的な「通常ドロップ枠数」は「エリアごとの標準枠（min〜max からランダム）＋上記の加算枠」で決まり、  
領域主を倒した場合はこれに加えて「領域主専用枠」が 1 枠ロールされる。

### 6.3 枠の由来と視覚表現（UI メモ）

- 各ドロップ枠には、「**どの行動で増えた枠か**」を表す **origin 種別**を持たせる（例：`battle`, `skill`, `strong_enemy`, `area_lord_standard`, `area_lord_special` など）。
- UI では origin ごとに枠の色・装飾を変えることで、プレイヤーに「どの行動がどの報酬につながったか」を視覚的に伝える：
  - 戦闘勝利由来：銅枠（ブロンズ）
  - 技能イベント成功由来：銀枠（シルバー）
  - 強敵クリア由来：金枠（ゴールド）
  - 領域主専用枠：虹枠（レインボー）

## 7. 経験値（Exp）ルール（案）

### 7.1 獲得量の考え方

- **探索ごとの経験値量の差は、基本的に「ほとんど出さない」**方針とする。
- エリアやドロップ内容による経験値差は設けず、**「勝利した戦闘の数と種類」だけで決まる**。
- **出撃メンバー（その探索に参加したキャラ）全員が同じ経験値を獲得**する。

### 7.2 獲得量の例（MVP 案）

- 探索 1 回分の経験値 = 以下の合計（探索終了時にまとめて付与）：
  - **雑魚戦勝利**：1 Exp × 勝利数
  - **強敵勝利**：+2 Exp
  - **領域主勝利**：+5 Exp
- 全滅した場合も、それまでに**勝利している戦闘ぶんの Exp は獲得**する（負けた戦闘自体には Exp は発生しない）。

### 7.3 必要経験値テーブル（MVP 案）

- レベルアップに必要な経験値は「**レベルに応じて 10 ずつ増える**」シンプルなテーブルとする。
  - Lv1 → Lv2：10 Exp
  - Lv2 → Lv3：20 Exp
  - Lv3 → Lv4：30 Exp
  - …
- 一般式：**Lv n → n+1 に必要な Exp = 10 × n**。  
  → 序盤は「爆速でレベルが上がる」体験を重視し、後半の伸び・上限は別途レベリング仕様で調整する。

## 8. 初期テーマとエリアの役割（案）

### 8.1 テーマ：錆びれた森林地区

- 初期実装テーマの例。以下の 3 エリアを持つ：
  - **遊覧舗装路跡**：難易度低め。**ドロップ枠が多い・技能イベントは少なめ（例: 10%）・資源ドロップ（木材/石材など）の割合が高い**。
  - **監視設備廃墟**：難易度中程度。**ドロップ枠は標準・技能イベントはやや多め（例: 25〜50%）・スキル分析書ドロップの割合が高い**。
  - **森林奥地**：難易度高め。**ドロップ枠・技能イベント率は標準（例: 25%）・遺物ドロップの割合が高い**。
- いずれのエリアにも強敵・領域主を配置し、**領域主撃破がテーマ目標の 1 つ**となり得る。

### 8.2 エリアごとのドロップカテゴリ（例）

錆びれた森林地区では、次のようなカテゴリを組み合わせてドロップテーブルを構成する想定：

- **資源系**：木材 / 石材 など（設備建設用のベーシック資源）
- **育成・戦闘系**：そのエリアに設定されたスキルの**スキル分析書**、鉄系・布系の装備部品
- **解放・拡張系**：水資源探索 α 型設計図 などの設計図
- **遺物系**：遺物シリーズ A / 遺物シリーズ B  
  - シリーズごとに「メイン効果のランダム範囲」や出やすい効果の傾向を変える前提。効果内容は別テーブル・別 spec で管理する。

## 9. テーマ攻略完了（目標）

「テーマ内の**すべての目標**をクリアすると、そのテーマ攻略完了」とする。  
**複数目標は AND**（全部達成でクリア）。

### 目標の種類（多様に設定する想定）

テーマごとに、次のような**複数種類の目標**を組み合わせて設定する。

| 種類 | 例 |
|------|-----|
| **領域主撃破** | このテーマ内の**全エリアの領域主**を撃破済みであること |
| **派生型設計図** | このテーマ内の**いずれかのエリア**から、派生型設計図を1つ以上入手していること |
| **遺物収集** | このテーマ内のエリアから遺物を**〇〇個**入手していること（累計でよい想定） |
| （その他） | 特定マップ初回クリア、マナコア到達など、テーマごとに追加可能 |

- 目標の**組み合わせ・数値（遺物〇〇個など）**はテーマごとにマスタで定義。
- 全部達成した時点でテーマ攻略完了。

### 9.1 テーマ攻略完了の報酬

- **新テーマ解放**：攻略完了したテーマに紐づく「次のテーマ」を解放する（常に想定）。
- **工業拡張**：テーマごとに「工業拡張あり/なし」を設定。ありの場合は**設置枠数アップ**または**コスト上限アップ**（018 の単一プールに対する拡張）。設置エリアを増やすのではなく、枠・コストが増える。
- **称号**：称号機能で「〇〇機工区解放！」などを表示し、工業拡張を世界観的に演出する（実装は称号システムに委ねる）。
- **ストーリー**：現時点ではストーリー進行機能はなし。

---

## 9.2 周回の位置づけ

探索は**周回が大前提**。「ずっと周回するゲーム」ともいえる。

- **メインの周回動機**：遺物周回（ハクスラ）、型・レアドロップの入手。
- **その他の周回動機**：今後も検討する（例：特定素材の効率的入手、仲間/メカの育成検証、スコア・タイムアタック、季節イベント用アイテムなど）。設計・実装の余裕に応じて追加。

## 10. 他仕様との接続

- **018**：テーマ攻略完了時、**そのテーマが工業拡張付きなら**設置枠数・コスト上限をアップ（毎回ではない）。称号で「〇〇機工区解放！」等の演出を出す想定。
- **017**：探索報酬テーブル（雑魚/強敵/領域主・技能判定・マップ別）で希少部品・遺物・派生型設計図を定義。
- **02_domain_model**：Expedition は「1回の出撃単位」、Battle は戦闘結果。探索エリア・テーマ・目標（種類・条件）はマスタ、達成状況は進行データで保持。
- **999**：最終エリアのマナコア（過成長植物園など）は世界観と一致。

## 12. 検討メモ：手軽さ・飽きづらさ

### 想定との対応

| 項目 | 020 との対応 |
|------|----------------|
| テーマ→エリア選択 | §1 の階層で一致。 |
| パーティプリセット | 039 の作戦プリセットをそのまま利用。編成＋作戦をまとめて選ぶだけにし、探索専用の別編成は持たない（手軽）。 |
| 持ち込み消耗品 | 出撃前に「何を何個持っていくか」だけ選択。探索中の使用は「イベント終了後」の1回タイミングに統一すると UX が単純。 |
| 技能イベント | 基礎ステを1つ選んで解決。パーティ内最高値のキャラで自動判定（020 既存）。 |
| 戦闘イベント | エリアごとの敵マスタ＋既存戦闘ロジックで実施。 |
| イベント後消耗品 | 「使う／使わない」を毎回選択可能にすると、HP/MP やドロップ枠・次のステアップなどで裁量が生まれ飽きづらさにつながる。 |
| 強敵→領域主抽選 | 1クリアの締めにランダム要素を入れることで、周回ごとの違いが出る。 |
| ドロップ | エリア別テーブル＋結果に応じたドロップ数で、回ごとの報酬の差が明確。 |

### 手軽にするポイント

- **ステップ数を絞る**：テーマ→エリア→プリセット→消耗品→出撃の 5 段階にまとめ、編成・作戦の細かい編集は「作戦室で事前にやっておく」前提にする。
- **既存システムの流用**：作戦プリセット（039）・戦闘（020/038/039/040）をそのまま使い、探索専用の重い新ルールを増やさない。
- **1本道のループ**：探索中は「イベント→（技能 or 戦闘）→消耗品選択→次へ」の繰り返しだけにし、分岐を少なくする。

### 飽きづらくするポイント（飽きない保証はしない）

- **毎回の選択**：どのステで技能に挑むか・どのイベント後にどの消耗品を使うかで、プレイヤー判断が毎回効く。
- **イベントの混在**：技能と戦闘がランダム（またはウェイト）で混ざることで、回ごとにリズムが変わる。
- **領域主抽選**：強敵後の「領域主が出るか」で、1クリアの緊張感と報酬の差が出る。
- **ドロップの可視化**：ドロップ数が「雑魚/技能/強敵/領域主」の結果でどう増えたかを分かりやすくすると、周回の動機づけになる。
- **テーマ・エリアの差**：エリアごとに敵・技能基準値・ドロップテーブルを変えておき、テーマ攻略や工業拡張（018）と紐づけると、長期的な目標になる。
