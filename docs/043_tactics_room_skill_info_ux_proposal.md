# 作戦室でのスキル情報表示 UX 提案

作戦室（作戦スロット編集画面）において、**スキルの効果・性質が分からない**という課題に対し、どのような情報をどこで見せるかの提案。  
spec/039（作戦室）・spec/038（戦闘スキル）を前提とする。

---

## 1. 現状の課題

- **行動**列のスキル選択は、物理/魔法/補助の種別でグループ化された**スキル名のみ**のプルダウン。
- チャージタイム・クールダウン・対象範囲・説明文などが画面から読み取れず、「どのスキルをどの条件で使うか」の判断がしづらい。
- 倍率・ヒット数・効果の数値などは**意図的に非開示**（ユーザー検証に委ねる）方針とする。

---

## 2. 方針の選択肢

| 案 | 内容 | メリット | デメリット |
|----|------|----------|------------|
| **A. 作戦画面に情報を埋め込む** | スキル選択時にツールチップ／ポップオーバーで説明を表示。または選択中スキルの下に 1 行サマリを表示。 | 画面を切り替えずに確認できる。 | テーブルがごちゃつく。10 スロット×3 キャラ分で情報が散らばる。 |
| **B. スキル一覧を別画面・モーダルで提供** | 「スキル一覧」ボタン／リンクで一覧を開き、種別・CT・CD・説明文を一覧表示。 | 一覧で比較しやすい。作戦テーブルはシンプルに保てる。 | 作戦編集と一覧を往復する手間。 |
| **C. ハイブリッド（推奨）** | 作戦画面のプルダウンでは**短い補足**（例: スキル名の横に CT/CD のバッジ）。詳細は**スキル一覧**（モーダル／ドロワー／別タブ）で表示。 | 選びながら「軽い情報」はその場で見え、深く知りたいときだけ一覧を開く。 | 実装は A+B の両要素が必要。 |

**推奨**: **C. ハイブリッド**  
- 作戦テーブル上: スキル選択肢に「CT0 / CD2」のような短いラベルを付与（任意）。ホバーで説明文のツールチップを出してもよい。  
- 作戦室のヘッダーまたはフッターに **「スキル一覧」** リンク／ボタンを設け、押下で一覧（モーダルまたはスライドパネル）を開く。

---

## 3. スキル一覧で表示する項目

ユーザーが「どのスキルがどんなものか」を判断するために必要な**最小限の項目**。倍率・効果の数値などは開示しない。

### 3.1 必須（提案時点で合意したい項目）

| 項目 | 説明 | データソース |
|------|------|--------------|
| **スキル名** | 表示名 | `Skill.name` |
| **種別** | 物理 / 魔法 / 補助 | `Skill.battleSkillType` → 既存ラベル（BATTLE_SKILL_TYPE_LABELS） |
| **チャージタイム** | 0＝即時、1以上＝N サイクル後に発動 | `Skill.chargeCycles`（サイクル数で表示） |
| **クールダウン** | 0＝なし、1以上＝使用後 N サイクル再使用不可 | `Skill.cooldownCycles`（サイクル数で表示） |
| **説明文** | スキルの効果・性質の文章説明 | `Skill.description`（**`docs/14_初期スキル` の「説明」列の内容**を表示する。seed で DB に投入されている前提。） |

※ 説明文が未設定（null）のスキルは「—」または「説明なし」と表示する。戦闘ログ用の `logMessage` は用途が異なるため、一覧の説明文としては `description`（＝14_初期スキルの説明列）のみを使う。

### 3.2 推奨（あると作戦立案に役立つが必須ではない）

| 項目 | 説明 | データソース |
|------|------|--------------|
| **対象範囲** | 敵単体／敵全体／味方単体／味方全体／自分 | `Skill.targetScope` → 日本語ラベルに変換 |
| **属性** | 無／圧縮／焼損／凍傷など（コンボ理解に有用） | `Skill.attribute` → 既存 ATTR_OPTIONS 等のラベル |

これらは数値ではなく「誰に効くか」「何属性か」の情報のため、バランス検証を損なわずに開示しやすい。

### 3.3 タグ（ハッシュタグ風）表示

一覧でスキルの性質を**短いタグ**でまとめて表示する案。スキャンしやすく、説明文だけでは伝わりにくい「攻撃パターン」「効果のキーワード」を補える。

**表示例**

| スキル名 | タグ例 |
|----------|--------|
| 虎挟ミ | #敵単体 #複数回集中攻撃 #圧縮属性 |
| 血ノ嘆キ | #敵単体 #複数回攻撃 #切創属性 #切創決壊 #出血 |

- **対象**: #敵単体 / #敵全体 / #味方単体 / #味方全体 / #自分
- **攻撃パターン**: #複数回攻撃 / #複数回集中攻撃 など
- **属性**: #圧縮属性 / #切創属性 / #焼損属性 など
- **効果キーワード**: #切創決壊 / #出血 / #列後退 など（状態異常名・effectType に対応する日本語ラベル）

**データの持ち方**

| 方式 | 内容 | メリット | デメリット |
|------|------|----------|------------|
| **displayTags（手動タグ）** | Skill に `displayTags`（文字列配列またはカンマ区切り）を追加し、seed/マスタで各スキルにタグを付与する。 | 表現を自由に揃えられる。効果の「見せ方」を統一しやすい。 | 全スキル分のタグ入力・更新が必要。 |

**実装前の作業**: 本機能実装前に、`docs/14_初期スキル` の全スキルについて Cursor からタグ案を出し、`displayTags` に入れる中身（例: `["#敵単体", "#複数回集中攻撃", "#圧縮属性"]`）を seed 用メモとして用意する。

- 一覧では**説明文の上または下にタグ行**として表示（例: 説明文の直下に `#敵単体 #複数回攻撃 #切創属性 #切創決壊 #出血` を小さめの文字で表示）。
- 作戦テーブルのホバー時にも、タグをツールチップに含めると「一言で性質が分かる」補助になる。

### 3.4 非開示とする項目（一覧には出さない）

- 威力倍率（`powerMultiplier`）
- 攻撃回数（`hitsMin` / `hitsMax`）
- 効果の param（倍率・確率・割合など）
- MP 消費の数値（CAP 係数・固定値）※ただし **作戦テーブル側のチップ／ツールチップでは表示可**（キャラごとの CAP から計算した実 MP 消費を表示する想定。スキル一覧画面には出さない）。

---

## 4. スキル一覧の対象スキル

- **表示対象**: **編成 3 人分のみ**。そのユーザーが現在編成に含めている 3 キャラが習得している戦闘スキルだけを一覧に出す（作戦で選べるスキルと一致させる）。全スキル図鑑は別機能として将来検討する。

---

## 5. 表示形式のイメージ

### 5.1 スキル一覧（モーダル／ドロワー）— 行カード（リスト）形式

- **1 行 = 1 スキル**の「行カード」形式のリストで表示する（テーブルより少し余白を多めに取った行ベース UI）。
- **各行の構成例**  
  - 左側: スキル名＋種別バッジ（物理/魔法/補助）＋CT/CD バッジ  
  - その下に説明文（`docs/14_初期スキル` の「説明」列の内容。2～3 行で折り返し。長文は省略＋ツールチップで全文可）  
  - 説明文の下にタグ行（`#敵単体 #複数回攻撃 #切創属性` などを小さめの文字で表示）
  - 右側（行の右端）に、**習得済みキャラのアイコン**を小さく横並びで表示（そのスキルを習得している編成 3 人のうち誰が持っているかを示す）
- 種別でフィルタまたはグループ表示（物理／魔法／補助）をつけると探しやすい。
- 同じスキルを複数キャラが習得している場合は、行は 1 つだけにし、その右端に習得キャラのアイコンを複数並べる。

### 5.2 作戦テーブル上のスキル選択（ハイブリッド時）

- プルダウンの各オプション: `スキル名 (CT{n}/CD{m})` のように短く表示（CT0/CD0 は「即時」など省略表示してもよい）。
- オプションにホバー／フォーカスしたときに `description`（およびタグがあればタグ）のツールチップを表示すると、一覧を開かずに「このスキルは何か」を確認できる。
- **MP 消費**については、キャラごとに CAP から計算した実 MP 消費値を、作戦テーブル側のチップ／ツールチップ内にのみ表示してよい（「MP 消費: 32」など）。スキル一覧のカードには MP 消費を出さない（キャラごとに値が変わりややこしくなるため）。

## 6. API・データの変更

- **既存**: `getBattleSkillsForCharacters(characterIds)` が `BattleSkillOption { id, name, battleSkillType }` を返している（tactics 用）。
- **スキル一覧用**: 編成 3 人の characterIds に対して、次のような API が必要。
  - 返却: 習得スキル一覧（重複は 1 スキル 1 件にまとめる）＋各スキルに `chargeCycles`, `cooldownCycles`, `description`, `targetScope`, `attribute`、および**タグは自動導出**した配列。さらに**そのスキルを習得しているキャラの一覧**（id およびアイコン用の iconFilename など）を付与する。
  - クライアントでカード描画時、習得済みキャラのアイコンを表示するために、スキルごとに「どのキャラが習得しているか」の情報が必要。
- 一覧表示は「選択中プリセットの 3 キャラが習得しているスキル」を 1 回の取得でまとめて返し、同一スキルは 1 カードにまとめ、そのカード内に習得キャラのアイコンを並べる。

## 7. 他項目の要否（FAQ 想定）

- **効果の「名前」だけ出す？**  
  effectType の日本語名をそのまま列にするより、**タグ**（#切創決壊 #出血 #列後退 など）で効果キーワードを出した方が一覧でスキャンしやすい。説明文で概括しつつ、タグで「何がついているか」を補う形を推奨。
- **スキル一覧は「全スキル」か「編成 3 人のみ」か？**  
  → **編成 3 人分の習得スキルのみ**とする。全スキルは図鑑的な別機能で将来検討する。

---

## 8. まとめ

| 項目 | 内容 |
|------|------|
| **UX** | ハイブリッド: 作戦テーブルではスキル名＋短い CT/CD 表示（とホバーで説明文）、詳細は「スキル一覧」で確認。 |
| **一覧の対象** | **編成 3 人分の習得スキルのみ**。 |
| **一覧の表示形式** | **行カード型のリスト**で一覧。各行の右端に習得済みキャラのアイコンを小さく表示。 |
| **一覧の必須項目** | スキル名・種別・チャージタイム・クールダウン・説明文（`Skill.description` ＝ 14_初期スキルの「説明」列）。 |
| **一覧の推奨項目** | 対象範囲・属性。 |
| **一覧のタグ表示** | ハッシュタグ風で性質を表示。**実装時は自動算出**（targetScope / attribute / hits＋resampleTargetPerHit / effectType マッピング）。 |
| **非開示** | 倍率・ヒット数・効果 param・MP 消費数値。 |
| **実装** | 作戦室に「スキル一覧」入口を追加。一覧 API でスキル＋習得キャラ一覧（アイコン用含む）を返す。タグはサーバまたはクライアントで自動導出。 |

この内容を docs の提案として固定し、spec/039 の「0.3 行動（スキル）選択の UX」を満たす拡張として実装に進める想定。
