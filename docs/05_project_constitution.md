# プロジェクト憲法（設計思想 / ぶれ防止）

この文書は「設計の最優先ルール」を固定するためのもの。
詳細仕様（spec）が迷ったときは、この憲法が最上位の判断基準になる。

------------------------------------------------------------------------

## 1. ゲームの核

-   本作は「探索（PvE）」「訓練（育成）」「工業（生産）」の循環で成立する。
-   戦闘は"操作"ではなく"準備と設計"が主役。結果はログで受け取る。
-   工業は戦闘を代替しない。戦闘は探索・遺物・資材獲得の意味を持ち続ける。

------------------------------------------------------------------------

## 2. 育成哲学：レベル＋CAP採用

-   無限レベル・無限転生は採用しない（レベル帯は将来拡張しつつも有限レンジで設計する）。
-   育成は「レベルアップで増える総合CAP（合計ポイント）＋プレイヤーによる配分調整」で表現する。
    -   レベルアップごとに CAP が増え、その一部は自動で基礎ステに配分され、残りをプレイヤーが自由に割り振る。
    -   各ステータスには CAP に対する割合の下限/上限（例：5〜30%）を設け、極端な偏りだけ制御する。
    -   既存の訓練設備によるステータス変動は廃止し、設備は工業サイクル寄りに役割を集中させる。
-   長期プレイヤー優位は、レベル差そのものよりも「ステ配分・装備・作戦など選択肢の広がり」で表現する。

------------------------------------------------------------------------

## 3. パーティ哲学：三位一体

戦闘パーティは以下の3枠で固定（MVP）：

1.  アカウントキャラ（本人）
2.  仲間（1体）
3.  追従メカ（1体）

-   本人は戦闘アクティブ・戦闘パッシブを持つ。
-   仲間は工業パッシブ・才能パッシブを持ち、厳選要素の核となる。
-   メカは戦術装置として再構築しやすい存在とする。

------------------------------------------------------------------------

## 4. 戦闘哲学：ログのみ / 事前作戦 / 再現性

-   戦闘はログのみ表示。
-   戦闘中の操作は不可。
-   事前に作戦を設定する。
-   行動順は派生値「速度（回避）」ベースの行動抽選方式（基礎 AGI 等を係数で換算した結果）。
-   再現性（seed）を意識する。

------------------------------------------------------------------------

## 5. 工業哲学：基礎値は生産、歪みは遺物

-   生産装備は安定した基礎値上昇。
-   遺物はビルドを歪ませる枠。
-   過度なインフレを避ける設計とする。

------------------------------------------------------------------------

## 6. 実装哲学：チャット依存禁止

-   仕様の正は repo 内の docs/spec。
-   ドメインロジックはUIから分離。
-   MVPでは過剰なリアルタイムを避ける。

------------------------------------------------------------------------

## 7. MVP原則

まず成立させるのは：

-   アカウント
-   キャラ
-   訓練
-   探索PvE
-   工業

市場・ギルド・大規模PvPは後回し。
