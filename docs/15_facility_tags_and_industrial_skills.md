# 工業設備のタグと工業スキル効果

工業設備に**タグ**を付け、**工業スキル**が「そのタグに一致する設備に配備されたときだけ効果を発揮する」形にする設計メモ。  
世界観（スチームパンク × ハイファンタジー）に沿った**初期設備案**と、タグ・スキル効果の例をまとめる。

---

## 1. タグの考え方

### 1.1 設備タグ

- **設備**（FacilityType＝施設の種類、または設置済み設備インスタンス）に、複数の**タグ**を持たせる。
- 例：浄水施設 → タグ `水`, `生活`。鉄鉱山 → タグ `鉱石`。鉄精錬炉 → タグ `精錬`, `金属`。
- 配備（Assignment）時に、「このキャラの工業スキルが参照するタグ」と「配備先設備のタグ」が**1つでも一致**すれば、そのスキル効果が発動する。

### 1.2 工業スキル効果の定義

- 工業スキルは「**〇〇タグの設備に配備されたとき**」に限り効果を持つ。
- 効果の種類（例）：
  - **時間短縮**：その設備の Job 所要時間を X% 短縮。
  - **生産量アップ**：その設備の出力を X% 増加（歩留まり・増産）。
- スキルマスタ（または効果定義）に「対象タグ」と「効果種別・数値」を持たせる。

例：
- スキル「水の心得」：タグ `水` の設備に配備時、**時間短縮 5%**。
- スキル「水質管理」：タグ `水` の設備に配備時、**生産量 3% アップ**。
- スキル「採掘の勘」：タグ `鉱石` の設備に配備時、時間短縮 5%。

## 2. 世界観から見た初期設備案

スチームパンク × ハイファンタジーを前提に、**生活基盤・資源・製造・訓練**がひととおり回るラインを想定した案。  
設備名は仮称。タグはスキル効果用のキーとして使う想定。

### 2.1 水・生活系

| 設備（案） | 役割 | タグ例 | メモ |
|------------|------|--------|------|
| **浄水施設** | 汚水や雨水を飲用・工業用水に。蒸気機関の補給水にも使う。 | `水`, `生活` | 浄水→貯水槽やボイラーへ。 |
| **貯水槽** | 浄水や雨水を貯留。 | `水` | 浄水施設の下流、または単体で「水」資源の安定供給。 |
| **蒸気凝縮器** | 蒸気機関の排気を凝縮して水回収。 | `水`, `蒸気` | スチームパンクらしい循環。 |

### 2.2 食糧系

| 設備（案） | 役割 | タグ例 | メモ |
|------------|------|--------|------|
| **穀物貯蔵庫** | 麦・豆などの保管・乾燥。 | `食糧`, `保存` | 探索や交易で得た穀物のストック。 |
| **乾燥窯** | 肉・魚・野菜の燻製・乾燥。保存食の生産。 | `食糧`, `保存` | 長期保存用。 |
| **調理場** | 食糧を加工し、行動用の携帯食やボイラー用の燃料補助。 | `食糧`, `生活` | 食糧消費＋軽いバフ等に使う想定。 |

### 2.3 採掘・鉱石系

| 設備（案） | 役割 | タグ例 | メモ |
|------------|------|--------|------|
| **鉄鉱山** | 鉄鉱石の採掘。 | `鉱石` | 既存例のまま。装備・メカの素材元。 |
| **石炭坑** | 石炭の採掘。燃料・蒸気機関用。 | `鉱石`, `燃料` | スチームパンクのエネルギー源。 |
| **水晶採掘場** | 魔術用の水晶・魔石を採掘。 | `鉱石`, `魔術` | ファンタジー要素。将来の魔術設備・装備に。 |

### 2.4 精錬系

| 設備（案） | 役割 | タグ例 | メモ |
|------------|------|--------|------|
| **鉄精錬炉** | 鉄鉱石 → 純鉄（既存例）。 | `精錬`, `金属` | 鍛冶・メカの前工程。 |
| **銅精錬炉** | 銅鉱石 → 銅地金。配線・蒸気パイプ等。 | `精錬`, `金属` | 電気・蒸気まわりで使う想定。 |

### 2.5 製造・加工系

| 設備（案） | 役割 | タグ例 | メモ |
|------------|------|--------|------|

| **メカ工房** | メカパーツの組み立て・調整。 | `製造`, `機械` | 既存のメカ頭パーツ機等を内包するイメージ。 |
| **調合室** | 魔術薬・強化剤などの調合。 | `製造`, `魔術` | ファンタジー側の生産。 |

| **読書室** | 知力・集中の訓練。 | `訓練`, `知力` | ステータス変動の方向を分ける。 |

## 3. タグ一覧（初期案）

スキル効果の「対象タグ」として使うキー。設備は上記のとおり複数タグを持てる。

| タグ | 説明 | 主な設備例 |
|------|------|------------|
| `水` | 水の扱い・浄水・貯水・凝縮。 | 浄水施設、貯水槽、蒸気凝縮器 |
| `食糧` | 食糧の生産・加工・保存。 | 穀物貯蔵庫、乾燥窯、調理場 |
| `生活` | 生活基盤・補給。 | 浄水施設、調理場 |
| `保存` | 貯蔵・長期保存。 | 穀物貯蔵庫、乾燥窯 |
| `蒸気` | 蒸気機関・蒸気利用。 | 蒸気凝縮器（将来はボイラー等） |
| `鉱石` | 採掘・未加工鉱石。 | 鉄鉱山、石炭坑、水晶採掘場 |
| `燃料` | 燃料の採掘・供給。 | 石炭坑 |
| `精錬` | 精錬工程。 | 鉄精錬炉、銅精錬炉 |
| `金属` | 金属素材・金属加工。 | 鉄精錬炉、銅精錬炉、鍛冶場 |
| `製造` | 製造・加工全般。 | 鍛冶場、メカ工房、調合室 |
| `機械` | メカ・機械部品。 | メカ工房 |
| `魔術` | 魔術・魔石・調合。 | 水晶採掘場、調合室 |
| `訓練` | 訓練設備。 | 訓練所、読書室 |
| `知力` | 知力系の訓練・研究。 | 読書室 |

- タグは**設備（FacilityType）に紐づける**想定。同一設備は同じタグセットを持つ。
- 新規設備追加時は、上記タグの組み合わせか、必要に応じて新タグを追加する。

## 4. 工業スキル効果の例（タグ一致で発動）

仲間の工業スキルが「〇〇タグの設備に配備されたとき」だけ効く形。

| スキル名（例） | 対象タグ | 効果（例） |
|----------------|----------|------------|
| 水の心得 | `水` | 配備先設備の **時間短縮 5%** |
| 水質管理 | `水` | 配備先設備の **生産量 3% アップ** |
| 採掘の勘 | `鉱石` | 時間短縮 5% |
| 精錬の手慣れ | `精錬` | 時間短縮 5% |
| 鍛冶の才 | `金属` または `製造` | 時間短縮 5% |
| 保存の知恵 | `保存` | 生産量 3% アップ |
| 蒸気いじり | `蒸気` | 時間短縮 5% |
| 魔術の素養 | `魔術` | 時間短縮 5% または 生産量 3% アップ |

- MVP では効果は「時間短縮 X%」「生産量 X% アップ」の 2 種に絞ってもよい。
- スキルマスタ（Skill）に「効果種別」「効果値」「対象タグ（1つまたは複数）」を持たせる。設備側は FacilityType（設備マスタ）にタグリスト（または FacilityTypeTag 中間テーブル）を持つ。

## 5. 工業エリアの設備三種類（入出力の流れ）

工業エリアに配備する設備は**大別して三種類**にする。それぞれ入出力の意味が異なる。

### 5.1 資源探索（resource_exploration）

- **入力**：なし。
- **出力**：その拠点で得られる**資源**（素材・マテリアル）。
- **例**：川探索拠点 → 水。山探索拠点 → 銅。鉄鉱山 → 鉄鉱石。石炭坑 → 石炭。水晶採掘場 → 水晶。
- レシピ的には「input なし → output 資源 1 種」。探索拠点ごとに取得できる資源が決まる。

### 5.2 工業設備（industrial）

- **入力**：あり。資源や中間素材。
- **出力**：加工後の**アイテム**（精錬品・製品など）。
- **例**：浄水設備に**水**（川から取った水など）を input → **飲料水**が output。鉄精錬炉に鉄鉱石 → 純鉄。銅精錬炉に銅鉱石 → 銅地金。鍛冶場に純鉄 → 武器下地。
- レシピ的には「input 1 種以上 → output 加工品」。資源探索で得たものを工業設備に渡す流れ。

### 5.3 訓練設備（training）

- **入力**：なし、またはものによってはあり（消費品で訓練効率アップ等は将来）。
- **出力**：なし（**ステータス変動のみ**が目的）。アイテムは出さない。
- **例**：訓練所（筋力・体力など）、読書室（知力・集中）。Job 完了時にキャラのステータスを変動させる。
- レシピ的には「input 任意 → output なし（効果はステータス変動）」。

### 5.4 流れの例

- **川探索拠点**で **水** を獲得 → **浄水設備**に水を input → **飲料水** が output。
- **山探索拠点**で **銅**（銅鉱石）を獲得 → **銅精錬炉**に input → **銅地金** が output。
- 資源探索で得た資源を工業設備に回し、工業設備の出力をさらに別の工業設備や装備製造に回す。訓練設備はその流れの外でステータス育成用。

### 5.5 データ上の対応（方針）

- **FacilityType**（設備マスタ）に **kind**（`resource_exploration` / `industrial` / `training`）を持たせる。配備先の「種類」判定と、将来の Recipe の input/output 解釈に使う。
- Recipe 実装時：資源探索は input なし・output 資源。工業設備は input あり・output 加工品。訓練設備は output なし・効果はステータス変動。
- 設備の「型」（基本型／派生型）は **docs/017** 参照。

## 6. データ設計の方向性（メモ）

- **FacilityType**（設備マスタ）：id, name, レシピ参照等。タグは別テーブルで多対多。
- **FacilityTag** または **Tag** マスタ ＋ **FacilityTypeTag**：設備 ↔ タグの多対多。
- **Skill**（既存）：工業スキルに「効果種別」「効果値」「対象タグ（tag を 1 つ、または SkillTag で複数）」を追加。
- 配備時：Assignment で「キャラ → 設備インスタンス」が決まる。設備インスタンスの FacilityType のタグと、キャラの習得工業スキルの対象タグを照合し、一致すればそのスキルの効果を Job 計算に反映する。

実装フェーズでは、spec/030 の工業スキル 5 種を「タグ＋効果」で定義し、設備側のタグは設備実装時に合わせて追加する形でよい。

---

## 7. まとめ

- **設備にタグ**をつけ、**工業スキルは「特定タグの設備に配備されたとき」だけ効果**を持つ形にする。
- 初期設備は、**水・食糧・鉱石・精錬・製造・訓練**をカバーしつつ、**スチームパンク（蒸気・石炭・金属）** と **ハイファンタジー（魔術・水晶・調合）** の両方が感じられるラインで提案した。
- タグは `水`, `食糧`, `鉱石`, `精錬`, `製造`, `訓練`, `燃料`, `金属`, `機械`, `魔術`, `生活`, `保存`, `蒸気`, `知力` などを初期セットとして想定。設備追加時にタグを付与し、スキル側で「〇〇タグで時間短縮／生産量アップ」を定義する。
