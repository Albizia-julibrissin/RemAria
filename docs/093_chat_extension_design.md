# チャット機能拡張設計（システムメッセージ・リンク・種別・表示設定）

`spec/037_chat.md` および `docs/022_chat_ui_design.md` を前提に、**システムメッセージの導入**・**メッセージ種別による色分け・リンク**・**ユーザーごとのチャット表示設定**を拡張するための設計メモ。実装時は本 doc を spec に反映してからコードを変更する。

---

## 0. 実装スコープ（本 doc での整理）

| 機能 | スコープ |
|------|----------|
| **任務クリア時のチャット通知**（quest_clear） | **本実装**。acknowledgeQuestReport 内で 1 件投稿。 |
| システムメッセージの kind/systemKind/色分け | 本実装（quest_clear 用）。 |
| 開拓者証（プロフィール）へのリンク | 本実装（任務達成メッセージから）。 |
| **チャットのユーザーごと表示設定**（サイズ等） | **本実装**。ヘッダー「設定」から設定画面へ。 |
| 作戦プリセットシェア（preset_share） | **将来拡張**。 |
| 探索クリア通知（exploration_clear） | **将来拡張**。 |
| その他 systemKind（title_unlock, level_up, system_announce 等） | 必要に応じて追加。 |

---

## 1. 目的

- **ユーザー投稿**と**システム投稿**を区別し、システムメッセージを色分け・アイコンで表示する。
- システムメッセージとして「誰々が任務〇〇を達成しました」を流し、世界の動きを感じさせる（本実装）。
- 将来的に「作戦プリセットのシェア」など、クリック可能な要素をチャットに載せられるようにする。
- **チャットパネルの大きさなどをユーザーごとに変更可能にし**、ヘッダーの「設定」から設定する。

---

## 2. メッセージ種別（kind）

| kind | 説明 | 送信者 | 表示方針 |
|------|------|--------|----------|
| **user** | プレイヤーが送信した通常メッセージ | User（senderName） | 従来どおり。時刻・名前・本文。 |
| **system** | システムが自動投稿するメッセージ | なし（または system） | 背景色・アイコンで区別。送信者名は出さない or 「システム」等。 |

- 既存の `ChatMessage` はすべて **user** として扱う（マイグレーションで `kind = 'user'` を付与）。
- 新規: **system** はサーバー側の処理（任務報告完了・称号解放など）から 1 件 `ChatMessage` を挿入する形で実装する。`userId` は null 許容にするか、**システム用のダミーユーザーを 1 件用意する**かは実装方針に任せる（推奨: null 許容で `userId` を nullable にし、system のときは null）。

---

## 3. システムメッセージのサブタイプ（systemKind）と例

`kind = 'system'` のとき、**systemKind** でさらに種別を分け、表示スタイルやリンクの有無を変える。

| systemKind | トリガー | 表示例 | リンク・付加要素 |
|------------|----------|--------|-------------------|
| **quest_clear** | 誰かが任務のクリア報告をした | 「〇〇が任務「△△」を達成しました。」 | 任意: 開拓者証（プロフィール）へのリンク |
| **title_unlock** | 誰かが新称号を解放した | 「〇〇が称号「△△」を獲得しました。」 | プロフィールへのリンク |
| **level_up** | （将来）誰かがキャラをレベルアップした | 「〇〇の主人公が Lv.N になりました。」 | プロフィールへのリンク |
| **exploration_clear** | （将来拡張）誰かが探索エリアを初クリアした | 「〇〇が「△△」を初制圧しました。」 | プロフィールへのリンク |
| **preset_share** | （将来拡張）誰かが作戦プリセットをシェアした | 「〇〇が作戦プリセット「△△」を共有しました。」 | プリセット閲覧・コピー用リンク |
| **system_announce** | 運営からのお知らせ（管理画面から投稿） | 任意の短文 | 必要に応じて URL やミッションへのリンク |

- **body**: 上記の「表示例」のような文言をサーバーで組み立てて保存する。システムメッセージも `body` に人間が読めるテキストを持たせる（検索・表示の一貫性のため）。
- **payload**（JSON, 任意）: リンク用の id を保持する。例: `{ "userId": "cuid", "questId": "cuid", "questName": "任務名" }`。フロントは payload を見て「開拓者証」リンクを描画する。

---

## 4. 他にありがたいシステムメッセージ案

- **quest_clear** … 誰々が任務〇〇を達成（前述）。世界の進捗が伝わる。
- **title_unlock** … 称号獲得。開拓者証で見せたいので相性が良い。
- **level_up** … 主人公 Lv アップ。頻度は高いので「同一ユーザー N 分に 1 回まで」など抑制してもよい。
- **exploration_clear** … エリア初制圧。マップ開放の共有。
- **preset_share** … 作戦プリセットのシェア。コミュニティでビルド共有。
- **system_announce** … 運営お知らせ・メンテ予告など。
- **facility_built**（将来）… 「〇〇が設備△△を建設しました。」工業の雰囲気。
- **relic_appraised**（将来・任意）… 「〇〇が遺物を鑑定しました。」頻度注意。
- **mail_received** … 個々には流さず「郵便が届いています」はヘッダー等で通知する想定。チャットには不要の可能性大。

優先度の目安: **quest_clear** → **title_unlock** → システムお知らせ → preset_share → その他。

---

## 5. 表示仕様（色分け・UI）

- **user**: 現行どおり。`text-text-primary`、送信者名は `text-text-muted`。
- **system**:  
  - 行全体を薄い背景で区別（例: `bg-base-elevated/60` または `bg-brass/10`）。  
  - 先頭にアイコン（例: 情報アイコン or 旗アイコン）。`GameIcon` で `flag` / `info` 等。  
  - 文言は `text-text-secondary` または `text-text-muted` で「流れ」として目立たせすぎない。  
- **systemKind 別**（任意）:  
  - 任務達成: 旗アイコン +  brass 系の薄い背景。  
  - お知らせ: メガホン or 情報アイコン + 別色。  
  - プリセット共有: 作戦アイコン + リンクボタン。

**リンク**: チャット内の**すべてのプレイヤー名**（ユーザー投稿の送信者・システムメッセージの主体）を `/dashboard/profile/[userId]` へのリンクにする。任務クリア報告に限らない。  
**主人公アイコン**: そのアカウントの主人公アイコン（Character.iconFilename）を取得し、getRecentChatMessages の返却に含める。チャット UI では送信者名／quest_clear のプレイヤー名の横に `/icons/{iconFilename}` で表示する。

---

## 6. データモデル案（ChatMessage 拡張）

| カラム | 型 | 説明 |
|--------|-----|------|
| kind | String | `user` \| `system`。デフォルト `user`。 |
| systemKind | String? | kind=system のときのみ。`quest_clear`, `title_unlock`, `preset_share` 等。 |
| payload | Json? | リンク用 id 等。例: `{ "userId": "...", "questId": "...", "questName": "..." }`。 |

- **userId**: kind=system のとき null 許容にする場合、`User?` のリレーションに変更。既存レコードはすべて userId あり・kind=user。
- **body**: 従来どおり。システムメッセージも本文をここに持つ。

---

## 7. API・処理の変更点

- **getRecentChatMessages**: 返却に `kind`, `systemKind`, `payload` を含める。既存は `kind: 'user'`。
- **sendChatMessage**: 従来どおり userId 付きで作成、kind は `user`（デフォルト）。
- **システムメッセージ挿入**: 新規 Server Action または既存アクション内で `prisma.chatMessage.create({ data: { userId: null, kind: 'system', systemKind: 'quest_clear', body: '...', payload: {...} } })` を呼ぶ。挿入箇所の例:
  - **quest_clear**: `acknowledgeQuestReport` 内で、報告完了・報酬付与後に 1 回だけ投稿。
  - **title_unlock**: 称号解放時に（クエスト報酬や他トリガーから）1 回だけ投稿。
- **運営お知らせ**: 管理画面から「システムメッセージ送信」フォームを用意し、body + systemKind（例: system_announce）で挿入。

---

## 8. リンク・開拓者証

- 表示文の「〇〇」を、payload.userId があれば `Link` で `/dashboard/profile/[accountId]` に飛ばす。accountId は User.id でよい（プロフィール画面が accountId でルーティングしている前提）。
- 「開拓者証を見る」を文末に付ける場合は、同じ profile リンクを 1 本表示する。

## 9. チャット表示設定（ユーザーごと）

チャットパネルの**大きさ**や表示オプションを、**ユーザーごとに保存**し、**ヘッダーの「設定」**から変更できるようにする。

### 9.1 設定の入口

- **ヘッダー「設定」**ドロップダウンに「チャット設定」リンクを追加し、**設定ページ**（例: `/dashboard/settings` または `/dashboard/settings/chat`）へ遷移する。
- または、設定をまとめた「設定」ページを 1 本用意し、その中に「チャット」セクションを設ける（将来、通知ON/OFF やテーマなども同じページに並べやすい）。

### 9.2 変更できる項目（望ましいもの）

| 項目 | 説明 | 保存先 | 備考 |
|------|------|--------|------|
| **パネルサイズ** | 幅・高さを **S / M / L** または **カスタム** で指定 | User 設定 or localStorage | カスタム時は**割合**（幅＝vw 相当、高さ＝vh 相当）を自由に変更。下記 9.2.1 参照。 |
| **デフォルトの開閉** | 画面読み込み時にチャットを開いておくか | 同上 | 現状は localStorage の `remaeria-chat-open` のみ。サーバーに保存すれば端末をまたいで同じ挙動にできる。 |
| **メッセージ文字サイズ** | チャット本文のフォントサイズ | 同上 | 例: 小 / 標準 / 大（text-xs / text-sm / text-base）。可読性のため。 |
| **システムメッセージを表示する** | quest_clear 等のシステム投稿を表示するか | 同上 | ON/OFF。流れが多すぎる場合に OFF でユーザー投稿のみにできる。 |

#### 9.2.1 パネルサイズ：S/M/L ＋ カスタム（割合）

- **プリセット**: **S / M / L** のいずれかを選択。それぞれ幅・高さのペアを固定値で適用する。
  - 例: S＝幅 280px・高さ 30vh、M＝360px・45vh、L＝440px・55vh（値は 022 や実機で調整）。
- **カスタム**: 「カスタム」選択時は、**幅**と**高さを割合で自由に調整**する。
  - **幅**: ビューポート幅に対する割合（実装は `vw` または `%`）。例: 25〜85% の範囲でスライダーまたは数値入力。最小幅 260px・最大 90vw など clamp で安全圏を設けてよい。
  - **高さ**: ビューポート高さに対する割合（実装は `vh`）。例: 25〜70vh の範囲でスライダーまたは数値入力。
  - 保存形式の例: `sizeMode: "custom", widthPercent: 35, heightPercent: 45`（幅 35vw 相当、高さ 45vh）。
- **いけるか**: はい。CSS で `width: min(90vw, max(260px, 35vw))`、`height: min(70vh, max(200px, 45vh))` のように **clamp / min-max** を使えば、極端な値でも崩れない。設定画面では「幅（%）」「高さ（%）」のスライダー 2 本、または S/M/L ラジオ＋「カスタム」選択時にスライダーを表示する形にすればよい。

**将来の候補**

- パネル位置（右下固定のまま / 左下など）。実装コストはやや高め。
- 新着メッセージ時のサウンド・デスクトップ通知（許可が必要）。
- ポーリング間隔（省電 vs リアルタイム寄り）。

### 9.3 保存方針

- **MVP 的には**: 上記を **localStorage** に保存し、同一ブラウザ内でだけ有効にする。DB スキーマ変更なしで実装可能。
- **端末をまたいで同じ設定にしたい場合**: `User` に JSON カラム（例: `preferences` または `chatSettings`）を追加し、設定ページで保存・読み込みする。
- **サイズの保存例**:
  - プリセット時: `{ "chatSizeMode": "preset", "chatSizePreset": "M", ... }`（S/M/L のいずれか）。
  - カスタム時: `{ "chatSizeMode": "custom", "chatWidthPercent": 35, "chatHeightPercent": 45, ... }`（幅％・高さ％）。

### 9.4 UI イメージ（設定ページ）

- **パネルサイズ**: ラジオで「小(S) / 中(M) / 大(L) / カスタム」を選択。**カスタム**選択時のみ、下に「幅（%）」「高さ（%）」のスライダー（または数値入力）を表示。例: 幅 25〜85、高さ 25〜70。プリセット選択時はスライダーを非表示または無効。
- 「起動時にチャットを開く」: チェックボックス。
- 「メッセージの文字サイズ」: 小 / 標準 / 大。
- 「システムメッセージを表示する」: チェックボックス。
- 保存ボタンで Server Action を呼ぶ（DB に保存する場合）または setState + localStorage のみ（クライアントのみの場合）。DB に保存する場合は、チャット開閉時・パネル描画時に設定を取得し、`sizeMode` に応じて幅・高さを px/vh または %/vh で反映する。

## 10. 作戦プリセットシェア・探索クリア（将来拡張）

- **preset_share**: ユーザーが「このプリセットをチャットでシェア」を押す → 専用の share 用 API で、`kind=system, systemKind=preset_share` または user メッセージの payload に presetId を載せる。受信側は「コピーして自分のプリセットに追加」可能に。spec/063 に依存。
- **exploration_clear**: 誰かが探索エリアを初クリアしたときに「〇〇が「△△」を初制圧しました。」を流す。トリガーは探索 finish 時の初回クリア判定。いずれも将来拡張とする。

## 11. まとめ

| 項目 | 方針 |
|------|------|
| 種別 | kind: user / system。system は systemKind で細分化。 |

| 任務達成（本実装） | acknowledgeQuestReport 内で quest_clear を 1 件投稿。body に「〇〇が任務「△△」を達成しました。」。 |
| 開拓者証リンク | payload に userId を入れ、表示側で profile への Link を出す。 |

| プリセットシェア・探索クリア | 将来拡張。 |
| その他 systemKind | title_unlock, level_up, system_announce, facility_built 等を必要に応じて追加。 |
