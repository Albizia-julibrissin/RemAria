# 経験値とレベルアップ

レベル1スタートで「Lv N → N+1 に必要な経験値 = N × 10」とする仕様と、探索以外の入手手段も想定した表現・実装方針。

---

## 1. 必要経験値の式

- **Lv 1 → 2**: 10
- **Lv 2 → 3**: 20
- **Lv N → N+1**: **N × 10**

累計で見ると、Lv M に到達するために必要な累計経験値は:

- Lv 1: 0
- Lv 2: 10
- Lv 3: 10 + 20 = 30
- Lv 4: 30 + 30 = 60
- Lv N: **10 × (1 + 2 + … + (N-1)) = 10 × (N-1)×N / 2**

---

## 2. データの持ち方

- **Character に「累計経験値」だけ持たせる**  
  - カラム例: `experiencePoints` (Int, default 0)  
  - レベルはこの累計から計算できるが、**表示・戦闘用には `level` と CAP を DB に持っておく**（048 と同じ。経験値付与時に level / CAP を更新する）。

- **レベル・CAP の更新**  
  - 経験値付与時に「累計経験値 → 新しいレベル」を計算。  
  - 現在レベルより上がっていれば、`level` と `CAP = 560 + 60×(level-1)` を更新（048・docs/09 準拠）。  
- **レベルアップ時の 7 ステ自動加算（下限 10% を満たすため）**  
  - 各ステは **CAP の 10〜30%** の範囲（048）。レベルアップで CAP が増えたとき、下限 10% を満たすために、**各ステに floor(deltaCAP × 0.10) を加算**する（7 ステで deltaCAP の 70%）。  
  - 残り **deltaCAP の 30%** は「自由分」として、いずれのステも 30% 上限を超えないように 1 ポイントずつ配分する。  
  - これでレベルアップ直後も「合計＝CAP」「各ステ 10〜30%」が成り立ち、以降はステ再配分で 30% 分の内訳を変更できる。

---

## 3. 経験値付与の表現（拡張しやすくする）

- **「誰に・いくつ」をまとめて処理する共通の入口を用意する**  
  - 例: `grantCharacterExp(characterIds: string[], amount: number)`  
  - 探索報酬では「参加キャラ全員に同量」でこの関数を 1 回呼ぶ。  
  - 将来はクエスト・イベントなども、同じ関数に「対象キャラ ID と量」を渡すだけで済む。

- **対象キャラ**  
  - 主人公・仲間（レベルを持つキャラ）のみ。メカはレベルを持たないので経験値付与対象外（049 と同じ前提）。

---

## 4. 実装メモ

- **累計 → レベル計算**  
  - `experiencePoints` から「その時点のレベル」を求める関数を 1 箇所（例: `lib/level.ts` や server 内）に用意。  
  - 上記の累計式の逆: 満たす最大 N を求める。

- **付与フロー**  
  1. 対象キャラの `experiencePoints` に加算。  
  2. 加算後の累計から新しいレベルを計算。  
  3. 現在の `level` より大きければ、`level` と `CAP` を更新。

- **探索との接続**  
  - `finishExploration` 内で、報酬確定時に「参加キャラ（主人公＋仲間 2 体）の ID 一覧」と `totalExpGained` を取得し、`grantCharacterExp(characterIds, totalExpGained)` を呼ぶ。

---

## 5. spec との対応

- **048**: レベルと CAP の関係はそのまま。経験値・レベルアップ処理は「別扱い」としていた部分を、本 doc ＋ 実装で満たす。
- **049**: 探索報酬の経験値は、上記の共通付与 API を呼ぶ形で実装する。
