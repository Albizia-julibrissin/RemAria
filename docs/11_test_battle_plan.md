# 仮戦闘（テスト用）の進め方

`docs/10_battle_calc_formulas.md` と `10_battle_status.csv` を前提に、**テスト用の仮戦闘**を実装するための手順とスコープをまとめる。

---

## 1. 目的・スコープ

- **目的**: 戦闘計算式が意図どおり動くか検証するための「仮の戦闘」ができるようにする。
- **スコープ（テスト用に絞る）**:
  - **1v1**: 主人公1体 vs 敵1体（列・パーティは考えない）。
  - **通常攻撃のみ**: スキルは使わない。命中・直撃・致命・ダメージ・MP回復だけ実装。
  - **装備なし**: 基礎ステータスのみ（Character の STR/INT/DEX/VIT/SPD/LUK/CAP）。
  - **敵**: 固定のモック敵（ステータスをコードまたは定数で定義。DB には持たない）。
  - **永続化**: 戦闘結果の保存はしない（または「テスト用にログだけ保存」は後で検討）。表示はログ＋勝敗で十分。

---

## 2. 進め方（推奨ステップ）

### Step 1: 仕様の固定（spec 化）

- **spec/020_test_battle.md** を新規作成する。
  - 入力: 主人公（現在ログインユーザーの Character 1体・category=protagonist）、敵（固定定義1種）。
  - 出力: 戦闘ログ（ターンごとの行動・命中/直撃/致命・与ダメ・MP変動）、勝敗。
  - ルール: `10_battle_calc_formulas.md` の「通常攻撃のみ」部分（行動順・命中・直撃/致命・ダメージ・乱数・疲労・MP回復）を参照とし、パラメータ（α, β, D, K 等）を spec に明記する。
  - スコープ: 1v1・通常攻撃のみ・装備なし・敵は固定1種。

### Step 2: 派生ステータス算出

- **基礎ステータスを係数表で二次解釈し、戦闘用ステータス（派生値）を算出する**。`10_battle_status.csv` がその係数表。
  - 基礎（7 種）は DB にのみ保存。**AGI は速度そのものではなく**、係数計算の入力であり、結果として派生値「速度（回避）」が得られる。
  - CSV の列1: 戦闘用ステータス（HP, MP, 物理攻撃, 魔法攻撃, 物理防御, 魔法防御, 命中力, 速度（回避）, 運）。
  - 式: `DerivedStat = Σ(BaseStat[s] * Coef[DerivedStat][s])`。係数は CSV の行2〜10 に対応。
  - **DB と CSV 一致**：Character の基礎ステータスは STR, INT, VIT, WIS, DEX, AGI, LUK（7 種）＋ CAP。CSV の列順と同じ（docs/16 で A 案採用済み）。
  - 実装: `src/lib/battle/derived-stats.ts` で基礎ステータスを渡すと派生値（HP, MaxMP, PATK, MATK, PDEF, MDEF, HIT, EVA, LuckPoint）を返す。CAP は MP 消費等で別途使用。

### Step 3: 戦闘ループ（1サイクル = 2ターン）

- 1v1 なので「1サイクル = 自分ターン + 敵ターン」。行動順は**派生値 EVA（速度（回避））**＋ LUK の重みで「先攻」を決める（EVA は基礎 AGI 等を係数で換算した結果）。
- 各ターン:
  1. ターゲットは相手1体で固定。
  2. 命中判定 → 直撃/致命判定 → 乱数（通常攻撃は攻撃側 1.0）→ 防御減衰 → ダメージ計算 → 直撃/致命倍率適用。
  3. 通常攻撃なので MP 回復（攻撃側のみ。MaxMP の 2％＋6〜12）。
- サイクルごとに疲労を更新（5サイクルまでは 1.0、以降は 10 の式）。
- どちらか HP≤0 になるか、最大サイクル数（例: 30）で終了。勝敗を返す。

### Step 4: API（Server Action）

- **runTestBattle()**: 現在ログインユーザーの主人公を取得し、固定敵と 1v1 で戦闘実行。戦闘ログ（配列）と勝敗（"player" | "enemy" | "draw" 等）を返す。DB には保存しない。

### Step 5: テスト用画面

- **`/battle/test`**（または `/dashboard/battle-test`）を用意。
  - ボタン「戦闘開始」で `runTestBattle()` を呼ぶ。
  - 結果として「勝敗」と「ログ」（ターンごとのテキスト）を表示する。
  - 保護画面として、ログイン＋主人公作成済みであること前提でよい。

---

## 3. 敵の定義（固定1種）

- 名前: 例「スライム」。
- ステータス: 10_battle_status の「物理型B」を参考に、CAP=350 想定で STR/INT/VIT/WIS/DEX/AGI/LUK を定数オブジェクトで持つ。**主人公と同構造**（STR, INT, VIT, WIS, DEX, AGI, LUK, CAP）で、派生値は同じ `derived-stats` で計算する。

---

## 4. マッピング整理（DB と CSV）

DB と CSV は一致（docs/16 A 案採用）。Character のカラムは CSV の列順と同じ。

| CSV（係数側） | キャラ（Character） |
|---------------|---------------------------|
| STR           | STR                       |
| INT           | INT                       |
| VIT           | VIT                       |
| WIS           | WIS                       |
| DEX           | DEX                       |
| AGI           | AGI                       |
| LUK           | LUK                       |

- 派生値「運」は `12 * LUK` で LuckPoint として別計算（10 の式）。CAP は K の計算等で使用。

## 5. このあとやること（チェックリスト）

1. [ ] spec/020_test_battle.md を作成する（上記 Step 1 の内容）。
2. [ ] 派生ステータス計算モジュールを実装する（CSV 係数の読み込み or ハードコード）。
3. [ ] 1v1 戦闘ループ（行動順・命中・直撃/致命・ダメージ・疲労・MP回復）を実装する。
4. [ ] runTestBattle の Server Action を実装する。
5. [ ] `/battle/test` のテスト用画面を実装する（戦闘開始ボタン・ログ・勝敗表示）。

---

## 6. スコープ外（仮戦闘ではやらない）

- スキル・消費 MP。
- 3体以上のパーティ・列・ターゲット抽選。
- 戦闘結果の DB 保存・報酬。
- 敵の種類複数・マスタ化。

これらは「仮戦闘が動いたあと」の本実装で扱う想定。
