# 特別アイテムと設備加速（開拓任務連動・HowTo スムーズ化）

**目的**: バッグの「課金」タブを「特別」にし、設備加速券を**特別アイテム**として実装する。開拓任務ベースで HowTo を進め、任務クリアで設備加速券を付与して「工業で生産→受け取り→もう一度探索」を待ち時間なしで体験できるようにする。

- **参照**: `docs/026_user_inventory_and_items.md`（設備加速券の位置づけ）、`manage/ROADMAP_1_3_MONTHS.md`（直近TODO 4・設備加速券）、`docs/056_how_to_play_guide.md`（HowTo）
- **世界観**: スチームパンク × ハイファンタジー。開拓・機工区・工房。

---

## 1. アイデアの評価

- **特別タブ＋特別アイテム**
  - バッグのタブ名を「課金」→「**特別**」にし、設備加速券などを「特別アイテム」として扱う。
  - 将来的に課金アイテムとして販売する予定でも、いまは**報酬・配布**で付与するため、「特別」のほうが世界観にも合いやすく、課金前提を出しすぎない。
- **開拓任務ベースで HowTo**
  - 「はじめにやること 3 ステップ」を**開拓任務のクリア条件**に紐づける（例：任務1＝主人公作成、任務2＝探索1回、任務3＝工業で受け取り→探索）。
  - 任務クリア報酬で**設備加速券**を付与すれば、ステップ 3 の生産待ちを解消できる。
  - プレイの導線が「ガイドを読む」だけでなく「**任務をこなす**」になり、報酬も明確でやりやすい。
- **結論**: この方向で設計・実装してよい。特別タブ名と開拓任務連動は、HowTo の体験と将来の課金設計の両方に効く。

---

## 2. 方針の整理

| 項目 | 内容 |
|------|------|
| **バッグのタブ** | 従来の「課金」を「**特別**」に変更。category は `paid` のままでも可（表示名のみ「特別」）。または category を `special` にし、アイテムを「特別アイテム」として扱う。 |
| **設備加速の効果** | 設備画面で消費し、**対象設備の生産完了を即時**（または残り時間を短縮）。1 枚 = 1 設備 1 回分を想定。 |
| **付与** | 開拓任務のクリア報酬として付与。初回 HowTo 用に「工業で受け取り→探索」に必要な分を任務で配る。 |
| **将来** | 課金アイテムとして同じアイテムを販売する想定。`docs/026` の「課金アイテム」と整合させる。 |

## 3. 設備加速券の名称案（世界観に合わせる）

世界観は**スチームパンク × ハイファンタジー**。開拓・機工区・工房がキーワード。

| 案 | 読み・備考 |
|----|------------|
| **機工特例券** | きこうとくれいけん。機工区に由来。「特例」で「通常より早く」を表現。短くてタブ・リスト向き。 |
| **緊急製造指示書** | きんきゅうせいぞうしじしょ。書類っぽくて世界観に合う。やや長い。 |
| **生産短縮券** | せいさんたんしゅくけん。効果が一目で分かる。やや無機質。 |
| **工房の時短符** | こうぼうのじたんふ。工房＋時短。「符」でお札・特例のイメージ。 |
| **即応製造券** | そくおうせいぞうけん。即応＝すぐに対応。軍事的・組織的なニュアンス。 |
| **優先手配書** | ゆうせんてはいしょ。優先して手配する＝早く回す。書類系。 |
| **蒸気注入券** | じょうきちゅうにゅうけん。蒸気を注入して工程を短縮、というスチームパンク寄り。 |
| **開拓団配給券（加速用）** | 長い。開拓任務の「配給」とストーリーで繋げるならあり。 |

**採用**: **緊急製造指示書**（きんきゅうせいぞうしじしょ）。書類っぽく世界観に合い、効果も伝わりやすい。

## 4. 通知機能と任務・HowTo の連携

- **方針**: 通知機能を完成させ、**新しい任務を受注したら通知に出す**。プレイヤーが通知を見て任務画面に飛び、そこで「今やること」と説明を読む流れにする。
- **任務が HowTo を兼ねる**
  - 任務ひとつひとつの内容を HowTo のステップに対応させる（例：任務1＝主人公作成、任務2＝探索1回、任務3＝工業で受け取り→探索）。
  - **毎回任務画面を見に来てもらえる**ので、その画面で「この任務では〇〇をしよう。そのためには△△画面で……」と説明を書ける。ガイドを別で読ませるより、**やるべきことと説明が同じ場所**になり迷いにくい。
- **評価**: この流れでよい。通知で「新しい任務がある」と知らせ → 任務画面を開く → 任務の説明欄に HowTo 的な文言を載せる、で一連の導線がつながる。実装範囲は広がるが、通知基盤と任務受注の連携は HowTo の定着にも効く。
- **実装に含めるもの（メモ）**
  - 通知機能の完成（ヘッダー「通知を表示（準備中）」を有効化。通知一覧・既読・種別）。
  - 通知種別のひとつとして「新規任務受注」（開拓任務を 1 件受注したときに 1 件追加。クリックで `/dashboard/quests` へ）。
  - 任務画面: 各任務の説明・達成条件・報酬に加え、必要なら「ここでやること」の短い HowTo 文を表示できるようにする。

---

## 5. 実装時に決めること（メモ）

- Item マスタ: 設備加速アイテムの名称は **緊急製造指示書**。category = `paid` のまま「特別」タブに出すか、`special` を新設して特別アイテム専用にするか。
- 開拓任務の報酬型: 報酬に「アイテム付与（itemId + quantity）」を追加し、緊急製造指示書を指定できるようにする（spec/054 拡張）。
- 設備画面での消費: 「この設備の生産を今すぐ完了させる」API と、消費時に UserInventory から 1 枚減算。対象は「進行中で未受け取りの 1 件」を選ぶ or 設備指定。
- シード・初回: 開拓任務の報酬テーブルに「〇〇任務クリアで緊急製造指示書 N 枚」を登録する。
