# 仲間キャラクター雇用 設計メモ

仲間（companion）を「雇用斡旋所」でゲーム内通貨または課金通貨を払って獲得する仕組みの設計。  
`docs/02_domain_model.md` の仲間・工業スキル方針に沿う。  
ヒアリング結果を反映し、決定事項として整理する。

---

## 1. 決定事項（概要）

### 1.1 通貨・価格

| 項目 | 決定内容 |
|------|----------|
| 支払い方法 | **ゲーム内通貨** または **課金通貨** のどちらでも可。 |
| 価格（暫定） | ゲーム内通貨 **1000** または 課金通貨 **100**。そのうち正式にバランス調整する。 |
| 課金通貨の消費順 | **無償分 → 有償分** の順で消費（docs/08 方針どおり）。 |
| 価格パターン | **1種類**でよい。初回割引・ランク別は今回スコープ外。 |

### 1.2 雇用の流れ（支払いと作成の分離）

**「正式作成できなかった場合に決済したくない」** ため、**雇用回数（チケット）の購入** と **名前・アイコン決定＋キャラ作成** を分離する。

- **購入**：ゲーム通貨 1000 または 課金通貨 100 を支払うと、**「仲間雇用可能回数」が 1 増える**。この時点ではキャラは作られない。
- **作成**：プレイヤーが「雇用を実行」する画面で名前・アイコンを入力し確定すると、**雇用可能回数を 1 消費**して仲間（Character category=companion）を 1 体作成する。
- こうすることで、**現金で課金したあと名前入力で離脱しても「回数」は残っており、後で名前・アイコンを決めて作成できる**。決済と作成のずれで発生するトラブルを避ける。

**必要なデータ**：ユーザーごとに「仲間雇用可能回数」を保持する。例：`User.companionHireCount`（Int, 0 以上）。購入時に +1、作成完了時に -1。履歴は CurrencyTransaction（購入時）と作成処理の参照で残す。

### 1.3 画面・入口

| 項目 | 決定内容 |
|------|----------|
| 入口 | ダッシュボードに **「雇用斡旋所」** という名前のカードを追加。クリックで雇用斡旋所画面へ。 |
| 残高不足時 | **「〇〇が足りません」** と必要数（あといくつ必要か）を表示する。 |

### 1.4 雇用実行フロー（名前・アイコン確定時）

1. 雇用斡旋所で「雇用可能回数」が 1 以上あることを確認。
2. 「仲間を雇う」等の操作で、名前入力・アイコン選択画面へ。
3. 名前・アイコンを確定 → **雇用可能回数を 1 消費** し、仲間（Character category=companion）を 1 体作成。
   - 初期基礎ステータス：主人公と同じ（オール10、CAP=60 等、定数で統一）。
   - **工業スキル**：スキルマスタから **1 つをランダムに選び** 習得として紐づける。雇用時点ではプレイヤーには見せない。
4. 作成完了後、**その仲間のキャラ詳細画面（/dashboard/characters/[id]）へリダイレクト**。ここで初めて付与された工業スキルが表示される（一石二鳥）。

### 1.5 工業スキル

| 項目 | 決定内容 |
|------|----------|
| 定義の持ち方 | **スキルマスタ**（DB）に持つ。まだ未作成。実装時に Skill テーブル等を用意する。 |
| MVP の種類数 | 工業スキルを **5 個** 用意する。 |
| 付与タイミング | 雇用実行（名前・アイコン確定）時に 1 つランダムで付与。表示は **雇用直後のキャラ詳細画面** で行う。 |

### 1.6 仲間の上限・解雇

| 項目 | 決定内容 |
|------|----------|
| 所持上限 | 仲間は **最大 10 体** まで。上限に達している場合は新規雇用（回数購入・作成）はできないようにする。 |
| 解雇 | **同じスコープで実装**する。 |
| 解雇の流れ | **キャラ一覧 → キャラ詳細** から操作。キャラ詳細に **削除（解雇）ボタン** を設ける。押したら **キャラ名の入力** で確認し、一致した場合のみ削除する「ちゃんと確認して削除する形式」にする。 |

### 1.7 アイコン

- 仲間のアイコンは **主人公作成と同じリスト**（例：public/icons の .gif）でよい。

## 2. フロー整理（確定）

### 2.1 雇用回数の購入（決済）

1. プレイヤーが雇用斡旋所画面で「ゲーム通貨で購入」または「課金通貨で購入」を選択。
2. 残高チェック（ゲーム 1000 以上 または 課金 100 以上）。不足なら「〇〇が足りません（あと X 必要）」を表示。
3. 支払い実行：該当残高を減算し、`User.companionHireCount` を +1。CurrencyTransaction に記録（reason: `companion_hire_purchase` 等）。
4. 課金通貨の場合は **無償 → 有償** の順で消費。

### 2.2 仲間の作成（雇用実行）

1. 雇用可能回数が 1 以上であること、かつ仲間所持数が 10 未満であることを確認。
2. 名前入力・アイコン選択画面で内容を確定。
3. `User.companionHireCount` を -1。Character（category=companion）を 1 件作成（名前・アイコン・初期ステータス）。工業スキルをマスタからランダム 1 つ選び習得テーブルに登録。
4. 作成したキャラの **詳細画面（/dashboard/characters/[id]）へリダイレクト**。ここで工業スキルを表示。

### 2.3 解雇

1. キャラ一覧 → 対象仲間のキャラ詳細を開く。
2. 詳細画面の「解雇する」ボタンを押す。
3. 確認 UI で **そのキャラの表示名（displayName）を入力** させる。入力が一致した場合のみ削除実行（Character 削除、紐づく習得スキルも削除）。不一致の場合は削除しない。
4. 解雇後はキャラ一覧等へ戻す。

## 3. 既存仕様との対応・必要な追加

| 項目 | 内容 |
|------|------|
| 通貨 | User の gameCurrencyBalance / premiumCurrencyFreeBalance / premiumCurrencyPaidBalance。消費時は CurrencyTransaction に履歴。 |
| 仲間の格納 | 既存の Character テーブル。category = "companion"。 |
| **雇用可能回数** | **新規**：User に `companionHireCount`（Int, default 0）のようなカラムを追加する想定。購入で +1、作成で -1。 |
| 工業スキル | **新規**：スキルマスタ（Skill 等）と習得テーブル（CharacterSkill：characterId, skillId）を用意。MVP では工業スキル 5 種類をマスタに登録。 |
| 名前・アイコン | 主人公作成（spec/015）と同様。表示名＋アイコン（public/icons 等）。 |
| 仲間上限 | 作成前に「現在の仲間数（category=companion の件数）が 10 未満」であることをチェック。購入時は上限チェック不要（回数だけ増える）。作成時に上限チェックする。 |

---

## 4. 技術メモ

- **トランザクション**  
  - 購入：残高減算 ＋ companionHireCount +1 ＋ CurrencyTransaction を同一トランザクションで実行。  
  - 作成：companionHireCount -1 ＋ Character 作成 ＋ CharacterSkill 登録 を同一トランザクションで実行。失敗時はロールバック。
- **解雇**  
  Character 削除時は onDelete: Cascade 等で習得スキル（CharacterSkill）も削除する想定。
- **スキルマスタ**  
  未実装のため、spec または別 doc で Skill テーブル・工業スキル 5 種の定義と CharacterSkill のスキーマを記載する。
