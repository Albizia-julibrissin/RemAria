# 闇市・黒市設計（GRA ⇔ システム アイテム取引）

**目的**: 課金系要素を**すべてアイテム経由**にし、GRA で「システムからアイテムを購入する」専用画面を設ける。  
開拓拠点に**闇市メニュー**を追加し、タブで**闇市**（無償 GRA で購入）と**黒市**（有償 GRA 専用アイテム）を切り替える。**システム買取は存在しない**（ユーザーがアイテムをシステムに売却する機能はなし）。

---

## 1. 背景・方針

### 1.1 課金系をアイテム経由にする

- 現状: 仲間雇用は **GRA を直接消費**して回数購入（`companionHireCount` 加算）している。
- **方針**: **GRA でシステムと取引 → アイテムを入手 → 各機能画面でそのアイテムを消費**する形に統一する。
  - 例: 「雇用権利書」を闇市／黒市で購入 → 人材局で権利書を消費して仲間を雇用。
- 効果: 課金アイテム・報酬アイテムの扱いが一本化し、バッグ・履歴・運営設定が整理しやすい。人材局の全面変更（後述）とも整合する。

### 1.2 本画面の位置づけ

| 項目 | 内容 |
|------|------|
| **名前（案）** | 闇市メニュー（開拓拠点の入口表示）。画面内はタブで「闇市」「黒市」を選択。 |
| **通貨** | **GRA**（無償: `premiumCurrencyFreeBalance` / 有償: `premiumCurrencyPaidBalance`）。消費順は既存どおり無償→有償。 |
| **取引相手** | **システム（NPC）のみ**。プレイヤー間取引は既存の「市場」のまま。 |
| **取引内容** | **GRA → アイテム**。ユーザーが GRA を支払い、システムからアイテムを**購入**するのみ。買取（ユーザー→システムへの売却）は行わない。 |

---

## 2. 画面構成

### 2.1 開拓拠点への追加

- **開拓拠点（ダッシュボード）**のメニューに **「闇市」** を追加する。
- クリックで **闇市メニュー画面**（`/dashboard/underground-market` など）に遷移。
- 画面内は **タブ** で以下を切り替え:
  - **闇市**: 無償 GRA で購入するアイテム。
  - **黒市**: 有償 GRA 専用で購入するアイテム。

### 2.2 闇市タブ

| 項目 | 内容 |
|------|------|
| **通貨** | **無償を先に消費し、足りなければ有償も使用**（市場と同じ。`premiumCurrencyFreeBalance` → `premiumCurrencyPaidBalance` の順で消費）。 |
| **購入** | システムが販売するアイテムを一覧表示。ユーザーが GRA を支払い、アイテムを所持（UserInventory に加算）する。 |
| **想定品目** | 雇用権利書・設備加速券・基本資源パックなど、**無償 GRA で回せる**アイテム。ログボ・イベントで付与する GRA の消化先としても使う。 |

### 2.3 黒市タブ

| 項目 | 内容 |
|------|------|
| **通貨** | **有償 GRA のみ**で購入（`premiumCurrencyPaidBalance` を消費）。無償では購入不可。 |
| **購入** | システムが販売する「有償専用」アイテムを一覧表示。ユーザーが有償 GRA を支払い、アイテムを所持する。 |
| **想定品目** | 課金で入手する権利書・限定アイテム・有償専用パックなど。 |

## 3. データ・マスタの方向性

### 3.1 システム販売品の定義

- **新規マスタ（案）**: 「システムが売る」品目を 1 行で表現するテーブルを用意する。
  - 例: `SystemShopItem` または `UndergroundMarketListing`（市場の Listing と名前が被るなら別名）。
  - 項目（案）:
    - 店種別: `tab` または `market_type`（`"underground"` = 闇市 / `"black"` = 黒市）
    - アイテム ID（Item.id）
    - 販売単価（GRA／1 個）— ユーザーが購入するときの価格
    - 表示順・在庫上限（任意）・販売開始/終了日時（任意）
  - 同一アイテムを闇市・黒市のどちらか一方にのみ出現させるか、役割で分ける（闇市＝無償用、黒市＝有償用）。

### 3.2 通貨履歴

- 既存の **CurrencyTransaction** を流用する。
  - 闇市購入: `reason` = 例 `"underground_market_purchase"`、無償 GRA の消費を記録。
  - 黒市購入: `reason` = 例 `"black_market_purchase"`、有償 GRA の消費を記録。
- `beforeBalance` / `afterBalance` は既存方針（docs/076 等）に合わせる。

### 3.3 アイテム側

- 既存の **Item** と **UserInventory** のまま。
- 闇市・黒市で「購入」した結果は **UserInventory に加算**するのみ。
- 課金系アイテム（雇用権利書・加速券など）は **Item マスタで category 等** を用意し、人材局・設備画面などで「そのアイテムを消費」する実装に寄せる。

---

## 4. API・画面の責務（案）

| 役割 | 内容 |
|------|------|
| **一覧取得** | 闇市タブ用・黒市タブ用に「システム販売品目」を返す。ユーザーの所持 GRA（無償/有償）も返す。 |
| **購入** | 指定した system_shop_item_id（または itemId + tab）と数量で、GRA を減算し UserInventory を加算。闇市なら無償のみ、黒市なら有償のみ消費。 |

---

## 5. 人材局全面変更との関係

- **人材局**を全面的に変える前に、**課金系をアイテム経由**にすることが前提。
- 闇市・黒市で「雇用権利書」などを販売し、**人材局では GRA 直接課金ではなく「権利書アイテムを消費して雇用」**する形にすると、人材局の役割が「仲間の雇用・解雇・一覧」に整理できる。
- 本 doc は**闇市・黒市の画面設計とデータの方向性**まで。人材局の新仕様・画面は別 doc で記載する。
