# チャット UI 設計案

## 1. 仕様の前提（docs より）

- **00_overview.md**: 全画面共通の常駐チャット（開閉可能）。全体チャット・ミュート/ブロック・通報。永続ログは最小限・直近ログのみ。「世界が同時に動いている感覚」を供給。
- **07_ui_guidelines.md**: 深い青黒ダーク ＋ ティール/銅アクセント。Tailwind（base / base-elevated / base-border / text-text-primary / text-text-muted / brass）。

---

## 2. 推奨スタイル・UI パターン

### 2.1 配置：右下フロート + ドロワー

| 案 | 説明 | メリット | デメリット |
|----|------|----------|------------|
| **A. 右下フロートパネル（推奨）** | 画面右下に「チャット」ボタン／タブを常駐。クリックでパネルが上方向に開く（高さ 40〜50vh 程度）。 | どの画面でも同じ位置で開閉でき、ヘッダーやメインコンテンツを隠さない。ゲーム画面を邪魔しにくい。 | 小さい画面ではパネルがややかさばる |
| B. 右端スライドサイドバー | 右端に細いタブだけ出し、クリックで右から幅 320px 程度のパネルがスライドイン。 | メッセージリストを広く表示できる。 | 横スペースを取る。レイアウトが左右で変わる |
| C. ヘッダー内タブ | ヘッダーに「チャット」リンクを置き、クリックで下またはモーダルで開く。 | ナビと一体で分かりやすい。 | 「常にどの画面でも見える」には別途フロートが必要になる |

**推奨**: **A. 右下フロートパネル**。横断表示・閉じておくことも可能、という要件に最も合う。

---

### 2.2 閉じた状態（デフォルト）

- **右下に小さなフロート要素を 1 つ**
  - 例: 角丸のピル型ボタン `[💬 チャット]` またはアイコン + 「チャット」ラベル
  - スタイル: `bg-base-elevated border border-base-border text-text-primary hover:border-brass hover:text-brass`（07 に準拠）
  - 未読がある場合はラベルやバッジで「N 件」を表示できる（将来用）
- クリックで「開いた状態」に切り替え

---

### 2.3 開いた状態

- **同じ右下を起点に、上方向へ伸びるパネル**
  - 幅: 320px〜400px（モバイルでは `min(100vw - 16px, 400px)` など）
  - 高さ: 40vh〜50vh（最大 60vh）。スクロールはメッセージエリアのみ
- **構成（上から）**
  1. **ヘッダー**: 「全体チャット」タイトル + 閉じるボタン（×）
  2. **メッセージリスト**: 縦スクロール、直近ログのみ。吹き出しまたは行形式。送信者・時刻・本文。
  3. **入力欄**: 1 行入力 + 送信ボタン（brass アクセント）
- **下部にオプション**: ミュート／ブロック／通報は「⋯」メニューにまとめておく（MVP では省略可）

---

### 2.4 スタイル詳細（07 準拠）

- パネル外枠: `bg-base-elevated border border-base-border rounded-t-lg shadow-lg`
- ヘッダー: `border-b border-base-border px-3 py-2`
- メッセージエリア: `bg-base`（やや暗くしてパネルと差をつける）、`text-text-primary`、`font-sans`
- 入力欄: `bg-base-elevated border-t border-base-border`、入力は `ring-brass` フォーカス
- 送信ボタン: `bg-brass hover:bg-brass-hover text-text-primary`
- 閉じるボタン: `text-text-muted hover:text-brass`

---

### 2.5 開閉状態の保持

- **クライアント状態**: 開閉は React state（useState）で管理
- **永続化（任意）**: `localStorage` の `remaeria-chat-open` などで「最後に開いていたか」を保存し、再読み込み時に復元すると UX が良い
- ルートレイアウト（`app/layout.tsx`）の `children` の外、かつヘッダーと同じ階層で「チャット用のクライアントラッパー」を置く（後述）

---

## 3. 実装上の配置

- **表示位置**: 全画面で見せるため、**ルートレイアウト**（`app/layout.tsx`）にチャット UI を置く。
- **クライアントコンポーネント**: 開閉・入力・送信はクライアントなので、例: `components/chat/ChatFloating.tsx` のようなクライアントコンポーネントで実装し、ルートレイアウトで `<ChatFloating />` をレンダリングする。
- **認証**: ログイン時のみチャットを表示（または未ログインは「ログインすると利用できます」のメッセージのみ表示）すると、仕様と一致する。

---

## 4. まとめ

| 項目 | 方針 |
|------|------|
| 配置 | 右下フロート + 上方向に開くパネル |
| 閉じた状態 | 右下の「チャット」ボタン／タブ（角丸・base-elevated・brass ホバー） |
| 開いた状態 | 幅 320〜400px、高さ 40〜50vh のパネル（ヘッダー・リスト・入力） |
| スタイル | docs/07 の base / base-elevated / base-border / brass / text-text-primary / text-text-muted に統一 |
| 開閉の保持 | クライアント state + 任意で localStorage |
| 配置場所 | ルート layout にチャット用クライアントコンポーネントを追加 |

この方針で実装すれば、「横断的で、常にどの画面でも見られるが、閉じておくことも可能」なチャット UI を、既存の世界観・UI ガイドラインと揃えられます。
