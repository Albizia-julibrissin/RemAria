# RE:mAria 利用規約（設計・精査メモ）

本ドキュメントは利用規約の内容精査と、登録時同意フローの設計をまとめたもの。表示文面の正本は `/terms` ページ（`src/app/terms/page.tsx`）とする。

## 1. 内容精査の結果

提供いただいた規約案をベースに、以下の点を整理・補足した。

### 1.1 表記・表現の統一

- 「ユーザー」と「ユーザ」：UI・規約文面では「ユーザー」で統一。
- 「運営」：主語として一貫して「運営」を使用（「当運営」は使用しない）。

### 1.2 追加を推奨した項目

- **施行日**：規約の効力発生日を明示。初回公開日を記載する。
- **お問い合わせ**：争議・問い合わせの窓口を一文で記載（例：「お問い合わせは運営が指定する方法で受け付けます」）。具体的な連絡先は運営側で後から差し替え可能な表現にした。
- **第12条（規約変更）**：重要な変更時にサービス内で通知することがある旨を追記。利用継続＝変更後規約への同意とする現行の趣旨は維持。

### 1.3 そのまま採用した項目

- 第1条〜第11条：趣旨を変えず、読みやすさのため句読点・改行のみ調整。
- 禁止事項（第3条）：不正ツール・BOT・RMT・嫌がらせ・過負荷・バグ悪用・不適切行為で網羅的。
- ゲームデータの権利帰属（第4条）、データ消失の非保証（第6条）、課金返金（第8条）、テストサービス・リセットの可能性（第10条）：一般的なオンラインゲーム規約として妥当。

### 1.4 任意（将来拡張）

- 準拠法・管轄裁判所：日本法・東京地方裁判所など、必要に応じて別条で追加可能。
- 個人情報の取扱い：別途プライバシーポリシーを設ける場合は、規約からは「個人情報の取扱いについてはプライバシーポリシーに従う」と参照のみとする形がよい。

## 2. 登録時同意フロー

- アカウント登録画面に「利用規約に同意する」チェックボックスを設置する。
- チェックなしでは送信不可（クライアントで必須表示、サーバーで検証して `TERMS_NOT_AGREED` を返す）。
- 同意した場合に限り登録を完了し、`User.termsAgreedAt` に同意日時を記録する（証跡として保持）。
- 規約本文はフッターの「利用規約」リンクおよび登録画面のリンクから `/terms` で表示する。
