# スキル設計：草案から見る必要項目とシステム

本書は **スキル草案** を題材に、スキルに必要になりそうな**項目（フィールド・テーブル）**と、それを実現するための**システム**を洗い出す。  
前提: `14_battle_core_spec.md`, `14_battle_attributes_tactics.md`, `14_battle_tactics_impl_draft.md`。

※ 属性名は既存仕様に合わせて「焼損」で統一。草案中の「熱損」は焼損と同じ扱いとする。

## 2. 草案から抽出した「スキルに必要そうな項目」

以下は、上記 4 案を満たすために必要な**スキル側のフィールド（または関連テーブル）**の整理。

### 2.1 基本（既存・拡張）

| 項目 | 説明 | 草案での使用例 |
|------|------|----------------|
| name | 表示名 | 全スキル |
| category | 工業 / 戦闘アクティブ / 戦闘パッシブ 等 | 戦闘用は battle_active 等 |
| **battleSkillType** | 戦闘時の種別: **物理 / 魔法 / 補助** | メテオ=魔法、強行突撃=物理、癒しの光=魔法、研ぎ澄まされた感覚=補助 |
| **mpCostFormula** | MP 消費式。例: `CAP*0.1+30` を係数で持つ（CAP係数＋固定値） | メテオ: 0.1 と 30、強行突撃: 0.05 と 15 |
| **chargeCycles** | 溜めサイクル数。0 なら即時発動。**サイクルで管理**する | メテオ=2、他=0 |
| **damageMultiplier** | ダメージ（または回復）倍率。補助は 0 や NULL | メテオ=2.2、強行突撃=2.0、癒しの光=0.5（回復係数） |
| **battleAttribute** | 無・圧縮・切創・穿孔・焼損・凍傷・侵食・極性。補助は無 | メテオ=焼損、強行突撃=穿孔、癒しの光=無、研ぎ澄まされた感覚=なし |
| battleLogMessage / battleLogMessageOnCondition | 戦闘ログ用（既出） | 全スキルで未定だが項目として必要 |

### 2.2 ターゲット・攻撃回数

| 項目 | 説明 | 草案での使用例 |
|------|------|----------------|
| **targetScope** | 対象範囲: 敵単体 / 敵全体 / 味方単体 / 味方全体 / 自分 | メテオ・強行突撃=敵単体（ただしメテオは複数回）、癒しの光=味方全体、研ぎ澄まされた感覚=自分 |
| **hitCountMin / hitCountMax** | 攻撃回数。1 固定なら両方 1。乱数なら 3～5 など | メテオ=3～5、他=1 |
| **resampleTargetPerHit** | 攻撃ごとにターゲットを再抽選するか | メテオ=true、他=false |
| **targetWeightModifier** | このスキルだけの列ウェイト加算。例: 後列 +2.0 | 強行突撃: 後列 +2.0 |

### 2.3 スキル効果（複数持ちうる）

既存の **SkillBattleEffect** の拡張＋新種効果として、以下が草案から必要になる。

| 効果の種類 | 説明 | 草案での使用例 |
|------------|------|----------------|
| 対象が属性状態 X のとき 倍率＋消費 | 既出（ハイスピア型） | - |
| **対象が属性状態 X のとき 確率で状態異常 Y 付与**（属性状態は消費） | ヒット時、対象が焼損なら 50％で燃焼付与、焼損消費 | メテオ |
| **ターゲットが後列だったとき 与ダメの一定割合を敵全体に** | 単体与ダメを算出後、その 50％を敵全員に追加 | 強行突撃 |
| **味方全体回復** | 回復量 = f(MATK, 倍率) を味方全員に | 癒しの光 |
| **特定状態異常を解消** | 対象から指定デバフを除去 | 癒しの光 |
| **自身にバフ付与**（ステータス上昇・N サイクル） | PATK +50％、HIT +25％、3 サイクル | 研ぎ澄まされた感覚 |
| **自身に属性状態を付与（代償）** | 使用時に自分に凍傷状態 | 研ぎ澄まされた感覚 |
| **対象の列を移動** | ヒットした相手を指定列へ、または「1段階後退/前進」で移動 | 新規 |
| **自分の列を移動** | 使用時に自分を指定列へ、または1段階前進/後退 | 新規 |

これらを「スキル効果」としてどう持つかは、**効果種別コード ＋ パラメータ（JSON または個別カラム）** の形が拡張しやすい（既存 SkillBattleEffect の triggerType を広げるか、別テーブルで効果タイプごとに持つか）。

## 3. 必要なシステム（処理・データ）

草案を実現するために、**戦闘ループ側やマスタ側で必要な仕組み**をまとめる。

### 3.1 溜め（チャージ）システム

- **概要**：スキル使用宣言時に「溜め N **サイクル**」なら、そのターンには何もせず「N **サイクル**後に発動」を予約する。**溜めはサイクルで管理**する。  
- **必要なもの**  
  - 各キャラごとに「発動予定スキル / 残り溜めサイクル数」を戦闘コンテキストで保持。  
  - 自分のターン開始時、溜め中なら残りサイクルを 1 減らし、0 になったらそのターンにスキルを実行。  
  - 溜め中に行動順が回ってきたときは「何もしない（通常攻撃にも行かない）」か、または「溜め中はスキップして次のキャラへ」か仕様で決定。  
- **考慮**：溜め中に死亡・行動不能になった場合の扱い（予約キャンセルするか）。

### 3.2 状態異常（デバフ）と DoT

- **燃焼**  
  - 付与時に「そのヒットで与えたダメージ」を記録。  
  - **自分のターン開始時**に、燃焼状態なら「記録ダメの 20％」を自分に与え、持続を 1 減らす（2 ターンなら 2 回発動して解除）。  
- **必要なもの**  
  - **Debuff マスタ**に、code（例: burning）、name、**効果種別**（DoT / その他）、**発動タイミング**（ターン開始時など）、持続、DoT の場合は「元ダメージの何％」などを定義。  
  - 戦闘中は「誰がどのデバフを、いつまで・付与時のメタ情報（記録ダメ）」を持つ。  
  - ターン開始処理で「燃焼持ちなら DoT 適用」を実行。

### 3.3 ターゲット範囲と列ウェイト修飾

- **敵単体**：既存の列ウェイト抽選で 1 体。  
- **敵全体**：列ウェイトは使わず、敵 3 体全員に効果（または攻撃回数分の対象リスト）。  
- **味方全体**：味方 3 体全員に回復・デバフ解除等。  
- **自分**：対象は行動者本人のみ。  
- **ターゲットウェイト修飾**：スキルごとに「後列 +2.0」などを指定し、**そのスキル使用時だけ**列ウェイトに加算してから抽選。  
  - 既存 COLUMN_WEIGHTS に上乗せする形で実装可能。

### 3.4 多ヒット＋ヒットごとターゲット再抽選

- **メテオ**：1 回のスキル実行で 3～5 ヒット。各ヒットごとに「敵 3 体から列ウェイトで 1 体」を抽選し、その 1 体にダメージ＋属性状態付与＋「焼損なら 50％で燃焼」を判定。  
- **必要なもの**  
  - スキルに hitCountMin, hitCountMax と resampleTargetPerHit を持たせる。  
  - ループで「ヒット回数だけ」ターゲット抽選 → 命中判定 → ダメージ・効果・属性状態・状態異常付与を実行。  
  - 同一体に複数ヒットした場合、2 ヒット目で「1 ヒット目で付いた焼損を消費して 50％で燃焼」ができるように、**ヒット順**で処理する。

### 3.5 回復とデバフ解除

- **癒しの光**  
  - 対象：味方全体。  
  - 回復量：MATK（または別式）× 倍率（0.5）を味方 3 体に加算。上限は各キャラの MaxHP。  
  - 解除：指定状態異常を味方全員から除去。  
- **必要なもの**  
  - スキルに「回復スキル」フラグまたは targetScope=味方全体＋効果種別=回復。  
  - 回復式（MATK ベース等）を 1 つ決める。  
  - 「解消する状態異常」のリストをスキルに持つ（デバフ code の配列または別テーブル）。

### 3.6 バフ（ステータス上昇・時限）

- **研ぎ澄まされた感覚**  
  - 自分に PATK +50％、HIT +25％を **3 サイクル**（発動サイクル含む）付与。  
  - 付与は「バフ」として保持し、ダメージ計算時や命中計算時に「現在のバフ込み PATK/HIT」を使う。  
- **必要なもの**  
  - 戦闘コンテキストで「誰がどのバフを、あと何サイクル持っているか」を保持。  
  - バフ定義：どのステータスを何％上げるか、持続サイクル。  
  - スキルに「付与するバフ」を 1 つ以上紐づけ（バフマスタ or スキル効果テーブルで「stat_modifier, PATK, 50, 3 サイクル」など）。

### 3.7 代償（自身への属性状態付与）

- **研ぎ澄まされた感覚**  
  - スキル使用時に、自分に**凍傷状態**を付与（発動サイクル含めて 2 サイクル）。  
- **必要なもの**  
  - スキル効果として「使用時に自分に属性状態 X を付与」を持つ。  
  - 既存の属性状態の付与ロジックを「対象 = 自分」でも実行する。

### 3.8 列移動（相手・自分）

- **対象の列を移動**  
  - スキルヒット時、対象（敵 1 体）の**列**を変更する。例：前列→中列に後退、後列→中列に前進。  
  - 列は 1＝前列・2＝中列・3＝後列。移動は「指定列へ直接」か「1 段階後退（前→中→後）／前進（後→中→前）」で指定。  
- **自分の列を移動**  
  - スキル使用時、**自分**の列を変更する。前進（後→中→前）で突撃、後退（前→中→後）で引きなど。  
- **必要なもの**  
  - 戦闘コンテキストで「各ユニットの現在列」を更新可能にすること（既存の BattlePosition.col）。  
  - スキル効果として `move_target_column` / `move_self_column` を持ち、param で「toColumn」または「direction: back / forward」を指定。  
  - 同一行内で「列だけ変える」ので、行の衝突は発生しない（仕様どおり）。

## 4. スキル効果のテーブル案（拡張イメージ）

既存 **SkillBattleEffect** は「対象が属性状態のとき 倍率＋消費」のみ。草案をカバーするには**効果種別**を増やす必要がある。

- **案 A**：SkillBattleEffect に `effectType` を追加し、同じテーブルで「attr_state_trigger_damage」「attr_state_chance_debuff」「column_splash」「heal_all」「dispel」「self_buff」「self_attr_state_cost」「**move_target_column**」「**move_self_column**」などを区別。パラメータは JSON で持つ。  
- **案 B**：効果タイプごとに別テーブル（SkillEffectAttrTrigger, SkillEffectSplash, SkillEffectHeal, ...）を用意し、スキル ID で紐づける。  
- **案 C**：Skill に「効果は JSON 配列で 1 カラム」持たせ、アプリで解釈。柔軟だがバリデーションや一覧性は弱い。

**推奨**：拡張性と「主語・条件の追加」と揃えるなら、**効果種別をコードで持ち、1 テーブルで effectType ＋ param（JSON）** の案 A が扱いやすい。主語・条件の追加と同様、新しい効果タイプは「コード追加＋アプリの処理分岐」で済む。

## 5. まとめ：項目とシステムの対応

| 草案で使う機能 | スキル側の項目 | 必要なシステム |
|----------------|----------------|----------------|
| メテオ | battleSkillType=魔法, chargeCycles=2, hitCount 3～5, resampleTargetPerHit, 効果「焼損→50％で燃焼」 | 溜め（サイクル）、多ヒット＋毎回ターゲット抽選、状態異常「燃焼」と DoT（ターン開始時） |
| 強行突撃 | targetWeightModifier 後列+2.0, 効果「後列ヒット時 与ダメ50％を敵全体に」 | 列ウェイト修飾、単体与ダメのスプラッシュ（敵全体への割合ダメ） |
| 癒しの光 | targetScope=味方全体, damageMultiplier=0.5（回復係数）, 効果「回復＋指定デバフ解除」 | 回復式、味方全体対象、デバフ解除リスト |
| 研ぎ澄まされた感覚 | battleSkillType=補助, 効果「自身バフ 3 サイクル」「自身に凍傷」 | バフの時限付与と計算への組み込み、自身への属性状態付与 |

この一覧をベースに、スキルマスタのカラム追加と SkillBattleEffect（または SkillEffect）の effectType 拡張、および戦闘ループ側の「溜め（サイクル）・DoT・ターゲット範囲・多ヒット・回復・バフ・代償・列移動」の実装順を検討できる。

### 6.4 新規項目・システムの整理（30 種で必要な追加）

| 項目／システム | 説明 | 使うスキル例 |
|----------------|------|--------------|
| **chargeCycles** | 溜めを**サイクル**で管理。0で即時、1以上で発動予約 | メテオ(2)、極意・貫(1)、稲光(1)、極大光(2)、鼓舞(1) |
| **move_target_column** | ヒット時に対象の列を変更。param: toColumn または direction: back/forward | 圧縮の一撃、氷結の矢、退散の呪文 |
| **move_self_column** | 使用時に自分の列を変更。param: toColumn または direction | 前進突撃、捨て身、鉄壁の構え、転移、反転の盾 |
| **ally_buff** | 味方単体／味方全体にバフ付与（誰を対象にするか） | 守りの祈り、鼓舞 |
| **状態異常「毒」** | 持続中、ターン開始時に現在HPの○％ダメ | 毒霧 |
| **状態異常「萎縮」** | 持続中、PATK・MATK を○％ダウン | 萎縮の呪い |
| **heal_single** | 味方単体回復（targetScope=味方単体） | 応急手当 |
| **dispel(1つ)** | 指定または「1つ」解除 | 浄化 |

これらを既存の「2. 必要項目」「3. 必要なシステム」にマージし、effectType に `move_target_column` / `move_self_column` を追加すれば、30 種をすべて表現できる。
