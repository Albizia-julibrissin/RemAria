# 郵便機能（運営→プレイヤー）設計メモ

**目的**: 運営からプレイヤーへ「お知らせ」と「付与物（アイテム・通貨など）」を届ける「郵便」機能の仕様相談用メモ。プレイヤー同士のメールは対象外。

---

## 1. やりたいこと（整理）

| 項目 | 内容 |
|------|------|
| **送り手** | 運営（管理画面から送信） |
| **受け手** | プレイヤー（特定ユーザー or 一斉） |
| **内容** | お知らせ文（タイトル・本文）＋ 付与可能なもの |
| **付与** | 無償 GRA・有償 GRA・アイテム（複数）・研究記録書・称号。§2 の通りすべて対応。 |

---

## 2. 付与物（確定）

運営郵便で付与できるものは以下すべて対応する。

| 付与物 | 説明 |
|--------|------|
| **無償 GRA** | User.premiumCurrencyFreeBalance に加算。お詫び・イベント付与など。 |
| **有償 GRA** | User.premiumCurrencyPaidBalance に加算。不具合補填・サポート返却用。 |
| **アイテム（複数）** | 任務報酬と同じ形（itemId + amount のリスト）。**バッグ上限を超えて付与**（郵便受取時は上限チェックしない）。 |
| **研究記録書** | User.researchPoint に加算。 |
| **称号** | UserTitleUnlock に 1 件追加（titleId）。複数指定可とするかは spec で定義。 |

詳細は **spec/090_mail.md** を参照。

---

## 3. 挙動・制約

| 項目 | 内容 |
|------|------|
| **受取のタイミング** | 「郵便を開いて『受け取る』を押した時点」で付与。未受取の間は付与されない。 |
| **重複受取** | 1 通につき 1 回だけ受取可能。受取済みフラグで制御。 |
| **有効期限** | **採用**。Mail に `expiresAt` を設ける。null のときは無期限。有効期限を過ぎたら受取不可（一覧では「期限切れ」表示など）。 |
| **バッグ上限** | **郵便受取時は上限を超えて付与する**。アイテム付与で所持数上限があっても、郵便経由では超過を許容し、必ず付与する。 |
| **お知らせのみ** | 付与がすべて 0 の郵便も可。その場合は「開封＝既読」だけで「受け取る」は不要（またはボタン非表示）。 |
| **既存の通知との違い** | 既存の `Notification` は付与なしのシステム通知。郵便は付与付き配布が主目的で、データモデルも別テーブル。 |

---

## 4. データモデル案

- **Mail（郵便マスタ・1通の定義）**  
  - 運営が「1通」作るときの内容。  
  - 項目: タイトル、本文、無償 GRA、有償 GRA、研究記録書、アイテム（JSON: `[{ itemId, amount }]`）、称号（rewardTitleIds: 複数可）、有効期限（expiresAt。null で無期限）、作成日時、送信対象の扱い（下記）。
- **送信対象**  
  - **案 A**: Mail に「全員向け」フラグ + 「個別 userId リスト」を持たせる（送信時に「全員」なら全 User に UserMail を一括作成、そうでなければリストのユーザーにだけ作成）。  
  - **案 B**: 「配信」を別テーブルにして、1 Mail に対して「全員配信」1件 or 「ユーザー別配信」複数件を紐づける。  
  - 運用の単純さでは **案 A** で十分なことが多い（「全員」か「指定ユーザーだけ」の 2 パターン）。
- **UserMail（ユーザーごとの受信箱レコード）**  
  - 項目: userId, mailId, **receivedAt**（受取日時。null ＝ 未受取）, readAt（開封＝既読。任意）、createdAt（届いた日時）。  
  - 受取時に receivedAt を立て、そのタイミングで 無償/有償 GRA・研究記録書・アイテム・称号を付与。

**正本は `prisma/schema.prisma`**。実装時に 08_database_schema と本 doc を同期する。

---

## 5. 監査・履歴

- **通貨**: 郵便受取で GRA を増やすときは、既存の `CurrencyTransaction` に 1 件ずつ記録。`reason` は新規コード（例: `mail_reward`）を `currency-transaction-reasons.ts` に追加。`referenceType: "mail"`, `referenceId: userMailId` など。
- **研究記録書**: User の `researchPoint` 加算のみでよいか、履歴テーブルが別途あればそこに理由付きで記録。
- **アイテム**: 既存の UserInventory 増加で足りる。必要なら「どこから増えたか」を reference で残す（郵便なら mailId / userMailId）。特別アイテムを郵便で配る場合は `ItemUsageLog` は**消費時**用なので、配布は別途「配布履歴」が欲しければ Mail 配信ログで代替可能。

## 6. 管理画面

- **送信**  
  - タイトル・本文・無償 GRA・有償 GRA・研究記録書・アイテム一覧・称号（複数可）・有効期限を入力。  
  - 送信先: 「全員」 or 「指定ユーザー」（accountId / userId を複数指定 or ファイルアップロード）。  
  - 送信実行で Mail 1 件作成 ＋ 対象ユーザー分の UserMail を作成。
- **履歴**  
  - 過去に送った Mail 一覧（タイトル・送信日・対象件数・有効期限・受取済み数など）。必要なら「未受取ユーザー一覧」も出せるようにする。

## 7. プレイヤー側 UI（確定）

### 7.1 入口（ヘッダー）

- **画面上部ヘッダー**の「通知」（ringing-bell）の**隣**に、**郵便**用アイコンを置く。
- アイコン: **Game Icons の `mailbox`**（`<GameIcon name="mailbox" />`）。AGENTS.md §5 のファンタジーアイコン方針に準拠。
- **クリックで郵便画面を開く**（ドロップダウンではなく、別画面へ遷移）。URL 例: `/dashboard/mail`。
- 未読または未受取の郵便が 1 通以上あるとき、通知と同様に**バッジ**（件数または「未読」インジケータ）を出してもよい（任意。実装時に決める）。

### 7.2 郵便画面レイアウト（メール風 2 カラム）

- **左側: 郵便選択リスト**
  - 自分宛ての郵便一覧。1 行 = 1 通。
  - 表示: タイトル・届いた日時・未読/既読・付与あり/なし・有効期限（または期限切れ）が分かる程度。
  - **クリックでその郵便を選択**し、右側に内容を表示。
  - 並び順: 未読を上、その後届いた日時降順など。
- **右側: 選択した郵便の内容**
  - 何も選択されていないときは「郵便を選択してください」などのプレースホルダー。
  - 選択時: **タイトル・本文**を表示し、その下に**添付（付与物）**を一覧表示。
    - 無償 GRA・有償 GRA・研究記録書・アイテム（名前＋数量）・称号（名前）を、付与があるものだけ表示。
  - **付与がある場合**: 「受け取る」ボタンを表示。押下で受取 API を呼び、成功したら右側の表示を更新（受取済みにし、ボタンを非表示または「受取済み」に）。
  - **付与がない場合（お知らせのみ）**: 「受け取る」は出さず、開封時に既読にする（「閉じる」や選択変更で既読でも可）。
  - 有効期限切れの場合は「受取期限を過ぎています」などと表示し、「受け取る」は出さない。
- **レスポンシブ**: 狭い画面ではリストと詳細を縦積みやタブ切り替えにするかは実装時に判断。

### 7.3 受取

- 「受け取る」で Server Action を呼び、receivedAt を設定し、無償 GRA・有償 GRA・研究記録書・アイテム・称号を一括付与。通貨は CurrencyTransaction に記録。
- **アイテムはバッグ上限を超えて付与する**（郵便経由では上限チェックを行わない）。

詳細な API・データ仕様は **spec/090_mail.md** を参照。

---

## 8. 決定済み（合意内容）

- 付与物: **無償 GRA・有償 GRA・アイテム・研究記録書・称号** をすべて対応。
- 有効期限: **採用**。expiresAt を設け、null で無期限可。
- バッグ上限: **郵便受取時は上限を超えて付与**。

実装仕様は **spec/090_mail.md** を参照。
