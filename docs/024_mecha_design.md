# メカ設計メモ（パーツ・スキル・ステータス）

キャラ（主人公・仲間）とメカの違い、メカパーツの部位・役割、スキル方針、ステータス計算をまとめる。**実装の正本は `spec/044_mecha_parts_and_stats.md`。**

---

## 1. キャラとメカの違い

| 項目 | キャラ（主人公・仲間） | メカ |
|------|------------------------|------|
| **装備で上がるもの** | 二層解釈後のステータス（物理攻撃力など） | **基礎ステータス**（STR, DEX 等） |
| **装備なし時** | キャラごとの基礎ステ | **オール10**で統一 |
| **スキル** | 習得する（スキル一覧から覚える） | **習得しない**。装備パーツに紐づくスキルのみ使用可能 |

剣などの装備は「基礎→派生」の二層のうち派生側を伸ばす。メカパーツは基礎側を直接伸ばし、フレームで傾向補正（倍率）をかける。

---

## 2. メカスキル方針

- **メカは専用スキルのみ**を使用する（キャラスキルと別カタログ or 同一 Skill で category 等で区別）。
- **メカスキル数は少なめに保つ**  
  - 例: キャラスキルが10個増えるころにメカスキル1個追加、フレーム追加もそのペース程度。
- **スキルはパーツに持たせる**  
  - **基本 1 パーツ 1 スキル**。拡張のため「1パーツ複数スキル」もデータ構造では許容し、初期実装は 1:1。
- **パーツ追加時**  
  - 既存のメカスキルから選んで割り当てるか、必要なら新スキルを1つ作る。  
  - 上位パーツは同じスキルを流用（例: ガンアーム系は同じ射撃スキル、ステータス差で格付け）。
- **クラフト**  
  - 装備・メカパーツは「同じものを何度も作って良い性能が出るまでサイクルする」仕組みを検討済み。

---

## 3. パーツ部位（6スロット）

| 部位 | ステータス役割 | スキル傾向 | 備考 |
|------|----------------|------------|------|
| **フレーム** | **傾向補正（倍率）** | なし | STR 1.2倍・DEX 0.8倍など |
| **コア** | フラット加算 | 任意（回復・バフ・攻撃など） | 種類を問わない |
| **頭部** | フラット加算 | デバフ系 | |
| **右腕** | フラット加算 | 攻撃スキル 1 つ | |
| **左腕** | フラット加算 | 攻撃スキル 1 つ | |
| **脚部** | フラット加算 | バフ系 | |

- **フレームのみ**が倍率（例: STR 1.2, DEX 0.8）。他部位は **STR+100, DEX+80** のような加算。
- スキルは「1パーツ複数」をデータ上許容し、**初期は 1:1**。部位ごとの傾向は上表のとおり。

---

## 4. ステータス計算の流れ

1. **ベース**  
   装備なしなら全基礎ステ **10**。
2. **フラット加算**  
   コア・頭部・右腕・左腕・脚部の加算値を合算 → 仮の STR, DEX, ... が決まる。
3. **フレーム倍率**  
   フレームの係数（1.2, 0.8 など）をかける → 最終的な基礎ステータス。

派生ステ（HP・攻撃力・速度など）は、この最終基礎ステから既存の戦闘計算式で算出する。

---

## 5. 運用・拡張のメモ

- **初期実装**  
  パーツはおおよそ10個程度、スキル重複ありでよい（同一スキルを複数パーツで共有可）。
- **スキルが少ないこと**  
  メカはスキル数を意図的に絞る前提。攻撃は両腕で2系統、バフ・デバフ・コアで数個あれば、作戦・ターン運用で差が出る想定。必要なら「1パーツ複数スキル」を解禁して選択肢を増やせる。
- **データ**  
  - 部位は enum/定数で `Frame | Core | Head | RightArm | LeftArm | Legs` を定義。  
  - パーツマスタ: 部位、ステ加算（またはフレーム用は倍率）、スキルIDは配列で保持（初期は要素1つ）。  
  - メカスキルは既存 `Skill` に category/usage で区別するか、別テーブルかは spec で決定。
