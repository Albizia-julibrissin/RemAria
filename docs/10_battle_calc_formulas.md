# 戦闘計算式（暫定・Cursor投入用）

本書は **RE:mAriaーResurgent Resonance-** の戦闘における「基礎ステータス → 派生値 → 戦闘処理（命中/直撃/致命/ダメージ/疲労）」を、実装へ落とすための計算式メモ。

> 方針：古のCGI系。戦闘はオート、表示はログ。射程は扱わない（全員どこでも攻撃可能）。

**補足資料**：`docs/10_battle_status.csv` … 派生ステータスごとの係数表。**列1（1列目）**に戦闘で使うステータス名が並ぶ。本書で「SPD」と書いているものは、CSV の **速度（回避）** に対応する。

---

## 1. 前提

### 1.1 基礎ステータス
- STR / INT / VIT / WIS / DEX / AGI / LUK（DB に保存する唯一のステータス。7 種＋CAP）
- 合計：`CAP`
- 単体上限（仮）：`CAP * 0.3`
- **AGI は「速度」そのものではない**。基礎 AGI は係数表への**入力**の一つであり、この表による**二次解釈**の結果が戦闘用ステータス「速度（回避）」になる。

### 1.2 派生ステータス（戦闘で使うもの）

戦闘時は基礎ステータスを**係数表で二次解釈**した値（派生ステータス）を用いる。派生値は **`docs/10_battle_status.csv`** の係数表に従い、次の形で算出する。

- `DerivedStat = Σ( BaseStat[s] * Coef[DerivedStat][s] )`

**CSV の列1**に並ぶ戦闘用ステータスは次のとおり（係数は CSV を参照）：
- HP, MP, 物理攻撃, 魔法攻撃, 物理防御, 魔法防御, 命中力, **速度（回避）**, 運

本書では「速度（回避）」を **SPD** または **EVA** と表記する。係数は **10_battle_status.csv を唯一の正**とする。

---

## 2. ターゲット選択（列ウェイト）

3x3の見た目は持つが、判定は **前/中/後** 列のみ。
射程は扱わず、列は **ターゲット確率**にのみ影響。

- 前列ウェイト：`1.5`
- 中列ウェイト：`1.0`
- 後列ウェイト：`0.5`

ターゲット確率：
- `P(target i) = w_i / Σ(w_all)`

> スキルの「前列優先/後列優先」などは、`w_i`に乗算・加算で実装する（例：前列対象へ×1.3）。

---

## 3. 行動順（サイクル制・重み付き抽選）

### 3.1 サイクル
- **1サイクル = 生存者が全員1回ずつ行動**したまとまり。
- サイクル開始時に行動順を生成する。

### 3.2 行動順の抽選
サイクル開始時、参加者全員から **重み付き抽選で順番を1人ずつ確定**していく（確定した者は除外して次の抽選）。

抽選重み：
- `W_turn = sqrt(SPD) * (1 + α * (LUK/7))`

ここで **SPD** = 派生値「速度（回避）」（`10_battle_status.csv` 列1）。

定数：
- `α = 0.2`

> 目的：SPD（速度/回避）優位は残しつつ「ワンチャン（運による割り込み）」を薄く混ぜる。

---

## 4. 命中判定（回避の減衰あり）

回避には派生値 **速度（回避）**（本書では EVA）を使用する。いずれも `10_battle_status.csv` 列1の項目。

- `β = 0.5`

命中率：
- `P(hit) = HIT / ( HIT + EVA * β )`

ここで：
- `HIT` = 派生「命中力」
- `EVA` = 派生「速度（回避）」（= SPD）

---

## 5. 直撃 / 致命（運による上振れ）

### 5.1 運ポイント
新式：
- `LuckPoint = 12 * LUK`

### 5.2 K（CAP追従）
- `K = CAP * 0.3 * 12`

例：CAP=700 → K=2520

### 5.3 直撃率（同値25%を基準）
- `Δ = LuckPoint_attacker - LuckPoint_defender`

- `P(direct) = clamp( 0.25 + D * ( Δ / ( |Δ| + K ) ), 0, 0.95 )`

`D`：影響力（デフォルト）
- `D = 0.5`

> パッシブ等で「運による直撃/致命影響力があがる」を付与する場合、`D`を増やす（例：`D=1.0`）。

### 5.4 致命率（直撃の上で判定／上限50%）
直撃が出た場合のみ、致命を判定する。

- `P(fatal | direct) = clamp( 0.25 + ( P(direct) - 0.25 ) / 3, 0, 0.50 )`

性質：
- 同値（P(direct)=0.25）なら致命も25%
- 直撃率が+1%上がると、致命率が+0.333…%上がる

### 5.5 ダメージ修正
- 直撃：ダメージ×`1.2`
- 致命：さらにダメージ×`1.2`、かつ **防御を半分扱い**
  - `DEF_effective = DEF * 0.5`

> 致命は「直撃の上位」として扱う（当たり→直撃→致命）。

---

## 6. ダメージ計算

### 6.1 スキル倍率（暫定）
- 通常攻撃：`m = 1.0`
- スキル：`m = 1.7`
- 条件達成：`m = 2.4`

（想定）
- 条件達成は **3ターンに1回程度**（例：`1.7, 1.7, 2.4`）

### 6.2 バフ（暫定）
- 物理攻撃バフ例：`BuffATK = 1.3`（該当ターンのみ等、運用はスキル側で管理）

### 6.3 乱数（ダメージ計算前）

攻撃側・防御側ともに、**ダメージ計算に使う前の値**に **0.8 ～ 1.0** の乱数倍率を掛ける。

- **攻撃側**：攻撃力（PATK または MATK）に `Rand_attacker`（0.8～1.0 の一様乱数）を掛けた値をダメージ計算に使う。
- **防御側**：防御（DEF_effective の元になる DEF）に `Rand_defender`（0.8～1.0 の一様乱数）を掛けた値をダメージ計算に使う。

**特例**：**通常攻撃**のときは、**攻撃側には乱数を適用しない**（攻撃側は 1.0 固定）。
- 通常攻撃：`Rand_attacker = 1.0`、`Rand_defender` は 0.8～1.0
- スキル攻撃：`Rand_attacker` も `Rand_defender` も 0.8～1.0

### 6.4 使用する防御（物理・魔法の別）

攻撃種別に応じて、**対象のどちらの防御を使うか**を切り替える。

| 攻撃種別 | 使用する防御（派生値） | 備考 |
|----------|------------------------|------|
| **物理** | 対象の **PDEF**（物理防御） | 通常攻撃・物理スキル |
| **魔法** | 対象の **MDEF**（魔法防御） | 魔法スキル |

- 派生値は `10_battle_status.csv` の通り（物理防御は VIT 寄り、魔法防御は WIS 寄り）。
- 致命時の「防御半分」は、**その攻撃で使っている DEF（PDEF または MDEF）** に対して適用する。

### 6.5 防御の減衰（暫定・物理・魔法共通式）

- `Mitigation = Denom / (Denom + DEF_effective)`

ここで：
- **DEF_effective**：上記 6.4 で決めた DEF（PDEF または MDEF）に、致命時は ×0.5、さらに 6.3 の防御側乱数 `Rand_defender`（0.8～1.0）を掛けた値。
- **Denom**：減衰の傾き用定数。**案A（現行）** では物理・魔法とも `Denom = 1000` で統一。

**別案（バランス用）**：物理と魔法で傾きを変えたい場合の例。
- 案B：`Denom_physical = 1000`、`Denom_magic = 800` → 同値の防御なら魔法の方が少し通しやすい（魔法ダメをやや強くしたいとき）。
- 案C：`Denom_physical = 1200`、`Denom_magic = 1000` → 物理を通しやすくしたいとき。

実装時はまず **案A（共通 1000）** でよい。必要に応じて定数だけ分離する。

### 6.6 基本ダメージ（命中時）

物理攻撃（通常攻撃・物理スキル）：
- `DEF = 対象の PDEF`（6.4）
- `Dmg_onHit = (PATK * Rand_attacker) * m * BuffATK * Mitigation * Fatigue`

魔法攻撃（魔法スキル）：
- `DEF = 対象の MDEF`（6.4）
- `Dmg_onHit = (MATK * Rand_attacker) * m * BuffATK * Mitigation * Fatigue`

> Mitigation は 6.5 の式で、DEF_effective に上記 DEF（乱数・致命反映済み）を用いる。

### 6.7 ダメージ確定手順（1行動）
1. ターゲット抽選（列ウェイト）
2. 命中判定（P(hit)）
3. 命中した場合：直撃判定（P(direct)）
4. 直撃した場合：致命判定（P(fatal|direct)）
5. **使用する DEF を決定**：攻撃種別が物理なら対象の PDEF、魔法なら MDEF（6.4）。
6. `DEF_effective` を決定（通常=DEF、致命=DEF*0.5）。乱数は 6.3 に従い DEF に適用。
7. **乱数**：攻撃側・防御側に 0.8～1.0 を適用（通常攻撃の攻撃側は 1.0 固定）。
8. `Dmg_onHit` を計算（上記乱数済みの値を使用）。攻撃種別に応じて PATK と PDEF、または MATK と MDEF の組で計算（6.6 参照）。
9. 直撃/致命の倍率（×1.2、致命ならさらに×1.2）を反映。
10. **MP 変動**：通常攻撃なら MP 回復（下記）。スキルなら消費（スキルごとの設定に従う）。

### 6.8 MP の回復と消費

#### 回復（通常攻撃時）

通常攻撃を行ったとき、**MP 回復**する。

- `MP_recovery = floor( MaxMP * 0.02 ) + Random(6, 12)`
- `Random(6, 12)`：6 以上 12 以下の整数乱数（一様）。端の扱いは実装で統一（6～12 のいずれか）。
- 現在 MP は `min( 現在MP + MP_recovery, MaxMP )` に更新。

#### 消費（スキル使用時）

スキル使用時は **MP を消費**する。消費量は**スキルごとに**次の形で設定する。

- `MP_cost = floor( CAP * (スキルごとの％) ) + (スキルごとの固定値)`

例：CAP=700、スキルが「CAP の 1% ＋ 固定 5」なら `7 + 5 = 12` 消費。

- 消費は**行動を選択した時点**で判定する。MP が足りない場合はそのスキルは選択不可（作戦・AI で別行動を選ぶ想定）。
- 足りている場合のみ消費し、`現在MP = 現在MP - MP_cost` とする。

---

## 7. 疲労（決着促進）

- サイクル1〜5：`Fatigue = 1.0`
- サイクル6以降：
  - `Fatigue = 1 + 0.04 * (cycle - 5)`

性質：
- 30サイクルで Fatigue=2.0（被ダメ2倍）
- 目的：回復・耐久の永久ループを防ぎ、30以内に収束

---

## 8. 調整パラメータ一覧（暫定）

- 行動順：`α = 0.2`
- 命中：`β = 0.5`
- 直撃/致命：`D = 0.5`（パッシブで増やす）
- 直撃/致命：`K = CAP * 0.3 * 12`
- 防御減衰：`Denom / (Denom + DEF_effective)`。案A では物理・魔法とも Denom=1000。DEF は攻撃種別で PDEF または MDEF。
- 疲労：5サイクル猶予、以降 `+4%/cycle`
- 乱数（ダメージ前）：攻撃側・防御側とも 0.8～1.0。通常攻撃の攻撃側のみ 1.0 固定。
- MP 回復（通常攻撃）：MaxMP の 2％ ＋ 6～12 のランダム。
- MP 消費（スキル）：スキルごとに「CAP の ○○％ ＋ 固定」で設定。

---

## 9. メモ（実装上の注意）
- clampの上限下限は、確率が0〜1に収まるように（直撃は0.95上限、致命は0.50上限）。
- ランダムは **サイクル順生成**・**戦闘判定（命中/直撃/致命）**・**ダメージ計算前（攻撃側 0.8～1.0、防御側 0.8～1.0。通常攻撃は攻撃側 1.0 固定）** に使用。
- 射程なしのため、バランス差は「列ウェイト」「命中」「疲労」「条件倍率」で作る。
