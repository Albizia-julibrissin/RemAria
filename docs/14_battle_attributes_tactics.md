# 戦闘：属性・耐性・状態・作戦スロット

本書は **RE:mAria** の戦闘における「攻撃種別・属性・耐性」「属性状態とコンボ」「作戦スロット」の仕様をまとめる。  
前提は `14_battle_core_spec.md` および `10_battle_calc_formulas.md`。  
**テーブル設計とアプリ処理の噛み合わせ**（作戦スロットをどう永続し、どう評価・実行するか）は `14_battle_tactics_impl_draft.md` に仮案をまとめている。

---

## 1. 攻撃種別と防御

### 1.1 大別

- **攻撃種別**は **物理** と **魔法** の 2 種類。
- **防御**はそれぞれに対応する。
  - 物理攻撃 → **物理防御（PDEF）** で軽減。
  - 魔法攻撃 → **魔法防御（MDEF）** で軽減。

現在の計算式（`10_battle_calc_formulas.md`）では、物理用の PATK/PDEF によるダメージ計算は実装されている。魔法用は MATK/MDEF で同様の構造を組む想定。  
**属性**による追加処理（後述）は、この「種別ごとのダメージ」を算出した**後**に適用する。

---

## 2. 属性（Attribute）

### 2.1 定義

各攻撃は **攻撃種別（物理 or 魔法）** に加え、**属性**を 1 つ持つ。  
組み合わせは「2 種別 × 8 属性」を想定（物理＋切創、魔法＋穿孔など）。

| 属性 | 読み／備考 |
|------|------------|
| **無** | む。通常攻撃はこれ。 |
| **圧縮** | あっしゅく |
| **切創** | せっそう |
| **穿孔** | せんこう |
| **焼損** | しょうそん |
| **凍傷** | とうしょう |
| **侵食** | しんしょく |
| **極性** | きょくせい |

- **極性**  
  - システム上は他属性と同じ（耐性・状態の扱いも同一）。  
  - ゲーム上は「特別な属性」として扱い、**ほとんどのスキルでは実装しない**方針。

### 2.2 ダメージへの適用順

1. **種別に応じたダメージ計算**  
   - 物理：PATK と PDEF（と Mitigation 等）から物理ダメージを算出。  
   - 魔法：MATK と MDEF から魔法ダメージを算出。
2. **そのダメージに対して、属性ごとの耐性を割合で適用**（後述）。

---

## 3. 属性耐性（割合軽減）

### 3.1 考え方

防御側は **属性ごとに耐性値**（0～1 の割合、または 0～100% で管理）を持つ。  
「物理ダメージ 500 − 物理防御で 300 軽減 → 200」のように**種別ダメージ**を出したあと、その 200 に対して**属性耐性**をかける。

### 3.2 計算例

- 物理攻撃（切創属性）で、種別計算後のダメージが **200**。
- 相手の **切創耐性 20%** とする。
- 最終ダメージ:  
  \( 200 \times (1 - 0.20) = 160 \)

一般形:

- \( D_{\text{final}} = D_{\text{afterDef}} \times (1 - R_{\text{attr}}) \)
  - \( D_{\text{afterDef}} \)：種別（物理/魔法）と防御から求めたダメージ。
  - \( R_{\text{attr}} \)：その属性に対する耐性（0～1）。

耐性は「無」にも設定可能（通常攻撃用）。極性も他と同様に耐性で軽減する。

## 4. 属性状態（状態としての起点）

### 4.1 付与ルール

- **属性攻撃を当てた場合、相手にその属性の「状態」が付与される。**
  - 例：圧縮属性の攻撃を当てる → 相手は **圧縮状態** になる。
- この状態は **状態異常（デバフ）ではない**。  
  **他スキルの条件やコンボの起点**として使う。

### 4.2 仕様（暫定）

- **持続**：一律 **2 サイクル**。**発動したサイクルを含めて** 2 サイクル持つ。
  - 例：1 サイクル目で先手を取って相手に属性状態を付与 → そのサイクルと次の 2 サイクル目まで有効。後続の味方が同じ 1 サイクル内で起爆することも、2 サイクル目で自分が起爆することも想定。
- **複数状態の併存**：可能。  
  例：圧縮状態と穿孔状態が同時についている。
- **スタックなし**：同じ属性は「ついている／ついていない」のみ。  
  「圧縮×2」などにはしない。

### 4.3 消費

スキル効果で「〇〇状態を消費する」と定義されている場合、その状態を 1 つ解除する。  
スタックしないため、消費 = その属性の状態を消す。

---

## 5. コンボ例（状態の連鎖）

- **魔法スキル：ブラックホール**  
  - 属性：**圧縮**。  
  - 命中時、相手に **圧縮状態** を付与。
- **物理スキル：ハイスピア**  
  - 属性：**穿孔**。  
  - 効果：**対象が圧縮状態のとき、ダメージ 1.5 倍かつ圧縮状態を消費**。  
  - 攻撃後、相手は **穿孔状態** になる（圧縮は消費されて消える）。

このように「状態の付与 → 別スキルで状態を条件に倍率・消費」を組み合わせてコンボを表現する。

## 6. 作戦スロット（戦闘前設定）

### 6.1 概要

- 戦闘前に **作戦スロットを 10 個**用意する。
- 各スロットは **「主語 − 条件 − 行動」** の 1 セット。
  - **主語**：誰の状態を見るか（例：自分、味方全体、特定枠など）。
  - **条件**：いつ発動するか（例：自分の HP が 40% 以下、など）。
  - **行動**：そのときに取る行動（例：スキル〇〇を使う、通常攻撃、など）。

例（自然言語）:  
「**自分の HP が 40% 以下だったら**、**スキル〇〇を使う**」。

### 6.2 判定ルール

- 自分のターンが来たとき、**スロット 1 から順に**条件を評価する。
- **最初に条件を満たしたスロットの「行動」を実行**する。
- **クールタイム（CT）**：条件を満たしたスロットの行動がスキルで、そのスキルが**CT 中**（使用後 N サイクル経過前）のときは、そのスロットは**採用しない**。次のスロットの条件を評価する（フォールバック）。詳細は `docs/023_skill_cooldown_ct_design.md`。
- **いずれも満たさない場合**、または採用したスロットがなかった場合は **通常攻撃** を行う。
- 行動に「通常攻撃」を明示して入れれば、条件付きで通常攻撃を優先させることもできる。

### 6.3 主語（見る対象）

条件評価時に「誰の状態を見るか」を表す。MVP で用意する主語は次のとおり。

| 主語 | 意味 |
|------|------|
| **自分** | 行動者本人。 |
| **味方のいずれか** | 自パーティ 3 人のうち、誰か 1 人でも条件を満たせば true。 |
| **相手のいずれか** | 敵 3 体のうち、誰か 1 人でも条件を満たせば true。 |
| **正面の相手** | 自分の行（row）にいる敵のうち、一番前列にいる生存者。いなければ該当なし（条件不成立とするか、別規定）。 |
| **サイクル** | 現在サイクル番号（1 始まり）を条件の対象とする。偶数／奇数／N の倍数／N 以上 など。隔ターン（2 サイクルに 1 回など）もここで表現可能。 |
| **ターン** | 本サイクル内の行動順（1～6）を条件の対象とする。例：1～2 なら「先手を取れた場合だけ発動」。 |

将来の拡張候補：主人公枠／仲間枠／メカ枠の指定、味方全員／敵全員。

### 6.4 条件の種類とパラメータ

各条件は **条件種別（kind）** と、種別に応じた **パラメータ** で表現する。

#### HP・MP 閾値

- **対象**：自分 / 味方のいずれか / 相手のいずれか / 正面の相手（HP のみ。MP は自分・味方のいずれかのみ想定）。
- **種別**：`hpBelowPercent`, `hpAbovePercent`, `mpBelowPercent`, `mpAbovePercent`。
- **パラメータ**：閾値 **10％刻み**（10, 20, …, 90）。
- 例：「味方のいずれかの HP が 40％以下だったら」「自分の MP が 20％以上だったら」。

#### 自分が状態異常

- **種別**：`selfHasDebuff`。
- **意味**：自分が状態異常（デバフ効果を持つ状態）を 1 つでも持っていれば true。主語は常に自分。
- 状態異常の定義・一覧は別途詰める。

#### 主語が特定の属性状態

- **種別**：`subjectHasAttrState`。
- **主語**：自分 / 味方のいずれか / 相手のいずれか / 正面の相手。
- **パラメータ**：属性（無・圧縮・切創・穿孔・焼損・凍傷・侵食・極性のいずれか）。
- 例：「相手のいずれかが圧縮状態だったら」「正面の相手が凍傷状態だったら」。

#### サイクル（主語「サイクル」または主語なし）

- **種別**とパラメータ：
  - `cycleIsEven`：現在サイクルが偶数なら true。
  - `cycleIsOdd`：現在サイクルが奇数なら true。
  - `cycleIsMultipleOf`：パラメータ N（例: 2 で隔ターン、3 で 3 サイクルに 1 回）で、サイクルが N の倍数なら true。
  - `cycleAtLeast`：パラメータ N で、サイクルが N 以上なら true（終盤専用作戦など）。

#### 本サイクルの行動順（主語「ターン」または主語なし）

- **種別**：`turnOrderInRange`。
- **パラメータ**：`turnIndexMin`, `turnIndexMax`（1 始まり。そのサイクル内で何番目に行動するか。1～6）。
- 例：「1～2 番目なら発動」→ 先手を取れた場合だけ発動。「5～6 番目ならフィニッシュ技」。

### 6.5 行動

- **通常攻撃**：`action = normalAttack`。ターゲットは後述の「ターゲット抽選」に委ねる。
- **スキル**：`action = skill`, `skillId = スキルID`。同上、ターゲットは抽選。

行動で「誰を狙うか」は指定しない。あくまで「何をするか」だけをスロットで決める。

### 6.6 条件判定とターゲット抽選の関係

- **条件で見る対象**（主語）と、**実際に攻撃が当たる相手**は別である。
- 行動が決まったあと、**ターゲットは従来どおり列ウェイトによる抽選**で 1 体選ぶ。
- そのため、「相手のいずれかが圧縮状態だったらハイスピアを使う」と設定しても、**必ずしも圧縮状態の相手に当たるとは限らない**。この「狙いたい相手に当たる保証はない」点を、ゲーム上のいやらしさ・駆け引きとして設計に含める。

### 6.7 設計上の位置づけ

- プレイヤーは 10 個の「主語−条件−行動」を組み、**コンボや状況に応じた選択**を事前にプログラミングする形になる。
- 敵側の AI は別仕様（ターゲット抽選・スキル選択などは後で定義）。

## 7. 他ドキュメントとの対応

- **ダメージ計算の流れ（拡張後）**  
  1. 攻撃種別（物理/魔法）で PATK/PDEF または MATK/MDEF を用いて \( D_{\text{afterDef}} \) を算出。  
  2. 直撃・致命の有無で倍率・防御半減を適用（`14_battle_core_spec.md` の通り）。  
  3. **属性耐性**で \( D_{\text{final}} = D_{\text{afterDef}} \times (1 - R_{\text{attr}}) \)。  
  4. 必要ならスキル効果（「〇〇状態のとき 1.5 倍」など）をこの前後で反映。  
  5. 命中時に、攻撃の属性に応じて相手に**属性状態**を付与。  
  6. 「〇〇状態を消費」する効果があれば状態を解除。

- **作戦スロット**  
  - 味方の「行動選択」は、本スロットの評価結果（またはデフォルトの通常攻撃）で決める。
- **戦闘ログ・スキル演出メッセージ**  
  - スキルごとに「通常メッセージ」と「条件達成時の追加メッセージ」を持たせ、戦闘ログに表示する。条件達成時の追加文は **文字色を変えて** 表示する。詳細は `14_battle_tactics_impl_draft.md` の Skill 戦闘用拡張を参照。
