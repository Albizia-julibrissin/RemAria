# ステータス振り直し機能（再構築アンプルα）設計

個室（キャラ詳細）のステータス配分画面で、アイテム「再構築アンプルα」（`reconstitution_ampoule_alpha`）を使ってステータスを振り戻し・振り直しする機能の設計。  
実装の正本は **spec/092_stat_reconstitution.md**。

---

## 1. 前提・依存

- **アイテム**:  
  - **再構築アンプルα**: `Item.code = "reconstitution_ampoule_alpha"`。定数は `src/lib/constants/level.ts` の `LEVEL_CAP_REWARD_ITEM_CODE` と同一（spec/074）。部分再構築・完全再構築で消費。  
  - **再構築アンプルβ**: `Item.code = "reconstitution_ampoule_beta"`（完全再構築βで消費）。
- **ステータス・CAP**: spec/048（レベルとCAP、10%下限・30%上限）、docs/09。
- **対象**: 主人公・仲間のみ。メカは対象外。
- **配置**: キャラ詳細 `/dashboard/characters/[id]` のステータスブロック。「配分」を選択したときに「再構築」用ボタンを設置。

---

## 2. 三方式（すべて実装・プレイヤーが選択）

施術名称を **案A = 部分再構築**、**案B = 完全再構築**、**完全再構築β** とする。三つとも実装し、プレイヤーがどれを実行するか選ぶ。

### 2.1 部分再構築（案A）

- **消費**: 再構築アンプルα **N 個**（N はプレイヤーが選択、1〜所持数）。
- **効果**: 「1レベル分」＝ **18 ポイント**（60×0.30。spec/048 §2.6 の自由配分分）を振り戻して未割り振りにし、再配分する。
- **手順**: **1回の送信にまとめ、状態を持たない**。配分画面で「部分再構築」→ 使用個数 N 選択 → 振り戻し量（合計 N×18、各ステは CAP の 10% を下回らない）と **再配分後の7ステ** を同時に入力して送信。サーバでアイテム N 個消費 ＋ ステータスを再配分後の値で更新。
- **制約**: 振り戻し後も各ステ ≥ floor(CAP×0.10)、合計 ≤ CAP。振り戻し可能な最大量は「現在の合計 − 7×floor(CAP×0.10)」まで。

### 2.2 完全再構築（案B）

- **消費**: 再構築アンプルα **1 個**。
- **コスト**: レベルを **5 下げる**（経験値は newLevel の累計にリセット。戻したレベル分の経験値は消失）。実行時に注意を表示する。
- **効果**: 新しい CAP に対して **下限（10%）のみ** を自動配分し、残りをすべて未割り振りにする。プレイヤーが一から配分し直す。
- **手順**: 配分画面で「完全再構築」→ 確認モーダル（5レベルダウン・経験値消失・ステは下限のみになる旨の**注意**）→ 実行。level ≤ 5 のときは**使用不可**。表示文言: 「身体への負担が大きく施術不可能」。
- **制約**: **現在レベルが 5 以下の場合は実行不可**（身体への負担が大きく施術不可能と表示）。

### 2.3 完全再構築β

- **消費**: 再構築アンプルβ（`reconstitution_ampoule_beta`）**1 個**。
- **デメリット**: **レベルダウンなし**。経験値も減らさない。
- **効果**: 現在の **レベル・CAP はそのまま**で、7 ステのみ **下限（10%）** に振り直し、残りをすべて未割り振りにする。プレイヤーが一から配分し直す。
- **手順**: 配分画面で「完全再構築β」→ 確認モーダル（ステが下限のみになり未割り振りになる旨）→ 実行。レベル制限なし（レベル 1 でも実行可）。
- **制約**: 特になし（所持アイテム ≥ 1 のみ）。

---

## 3. 共通で決めること

- **実行権**: 対象キャラのオーナー（session.userId === character.userId）のみ。
- **履歴**: アイテム消費時は docs/081 に従い ItemUsageLog に記録する（理由コードは item-usage-reasons に追加）。
- **UI**: 再構築ボタンは「配分」表示中のみ表示。メカの場合は配分自体が出ないため、再構築も出さない。

---

## 4. 決定済み事項（ヒアリング結果）

- **施術名称**: 案A = **部分再構築**、案B = **完全再構築**、第三の施術 = **完全再構築β**（アンプルβ使用・レベルダウンなし）。三つとも実装。
- **案A**: 1レベル分 = **18pt**。振り戻しと再配分は **1回の送信にまとめる**（状態を持たない）。
- **案B**: アイテム **1個** 消費。レベル 5 以下のときは **使用不可**。「身体への負担が大きく施術不可能」と表示。経験値はレベルダウンで消失 → **実行時に注意を表示**。
- **共通**: 再構築後、**未割り振りのまま保存してよい**（合計 ≤ CAP でよい）。
